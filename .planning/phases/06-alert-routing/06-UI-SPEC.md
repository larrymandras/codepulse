| Severity | Text | Background | Border |

- Failed / retrying: `--status-error` (`oklch(0.65 0.18 27)`) — red dot

## Interaction Patterns

### Condition Builder (Claude's Discretion — designed here)

The compound condition builder uses a **flat grouped list** pattern (not a tree). Layout:

1. Each condition row: `[metric selector] [operator] [threshold value] [lookback window]`
2. Between rows: `AND` / `OR` operator pill — clicking toggles between AND and OR for the group
3. "Add condition" text button appends a new empty row
4. "Add group" text button wraps a new indented sub-group (one level of nesting only — no recursive nesting in v1)
5. Remove (×) button on each row

Metric selector uses shadcn `Select` component populated from the existing rule catalog categories. Lookback window uses `Select` with options: 5m, 15m, 30m, 1h, 24h.

### Alert Rule Form (Sheet)

- Triggered by "Edit" or "New Custom Rule" button
- Opens as a right-side `Sheet` (480px wide on desktop)
- Header: rule name (editable input for custom rules, read-only for static), severity badge
- Body: threshold override fields first, then condition builder (custom rules only)
- Footer: "Save Rule" primary button + "Cancel" ghost button + "Delete Rule" destructive link (custom rules only)
- Unsaved changes show a `•` indicator in the Sheet header

### Mute Duration Picker

- Triggered by "Mute" button on alert row or rule row
- Renders as shadcn `Popover` anchored to the Mute button
- Single-click selection — selecting a duration immediately applies and closes the popover
- Muted item shows clock icon + remaining duration in `text-muted-foreground`

### Webhook Test Flow

- "Send Test" button in Notification Channels section
- Button enters loading state (`Loader2` spin) during HTTP call
- On success: shows `--status-ok` dot + "Test delivered" for 3 seconds, then resets
- On failure: shows `--status-error` dot + "Test failed — check URL" persistently until URL is changed

### Escalate to Task

- "Escalate" button on alert row
- Opens shadcn `Dialog` (not Sheet) pre-filled with alert title as task name, severity as priority
- User can edit title and description before confirming
- On confirm: creates Kanban task, shows success toast, alert row gains "Linked to task" badge

### Acknowledge

- Single-click action — no confirmation dialog
- Row transitions to `opacity-60` immediately (optimistic UI)
- Alert state badge changes from "Active" to "Acknowledged" inline

## Page-Level Layout Notes

### Alerts Page additions

Existing tabs (All / Critical / Error / Warning / Info) are preserved. Each alert row gains a right-side action cluster:
- `[Acknowledge]` button — ghost, text-sm
- `[Mute ▾]` button — ghost, text-sm, opens MuteDurationPicker popover
- `[Escalate]` button — ghost, text-sm, opens Escalate dialog
- `[delivery status badge]` — WebhookStatusBadge, right-aligned

### Settings Page additions

Two new sections appended after the existing sections in `Settings.tsx`:

**Section: Notification Channels**
- Section header: "NOTIFICATION CHANNELS" (uppercase tracking-wide muted-foreground, source: UI-03)
- Discord webhook URL: label + monospace `Input` + status dot + "Send Test" button
- Slack webhook URL: label + monospace `Input` + status dot + "Send Test" button
- Layout: each channel in its own `div` with `gap-sm` between label and input row

**Section: Notification Preferences**
- Section header: "NOTIFICATION PREFERENCES"
- 4 rows (one per severity): severity badge + `Select` with 4 delivery mode options
- Save button: "Save Preferences" primary button, full-width on mobile

### Alert Rules Engine additions

Two new columns per rule row:
- Threshold override input (numeric, shows current default as placeholder, appears on row hover)
- Mute toggle (clock icon button)

New header button: "+ New Custom Rule" — triggers AlertRuleForm Sheet
