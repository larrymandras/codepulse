**Backend additions to `convex/webhookDelivery.ts`:**

Add public queries/mutations for Settings page consumption:

**`getChannels` query** (public version of getNotificationChannels):
- Queries `agentConfigs` for keys `"webhook-discord-url"` and `"webhook-slack-url"`
- Returns `{ discordUrl: string | null, slackUrl: string | null }`

**`setChannel` mutation**:
- Args: `channel: v.string(), url: v.string()`  (channel is "discord" or "slack")
- Validates `url` starts with `https://` — throw if not
- Key: `"webhook-{channel}-url"`
- Upserts into `agentConfigs`

**`removeChannel` mutation**:
- Args: `channel: v.string()`
- Deletes the `agentConfigs` record for `"webhook-{channel}-url"`

**`testWebhook` action** (public action):
- Args: `channel: v.string()`
- Reads the stored URL from agentConfigs
- Sends a test message: Discord embed with title "CodePulse Test Alert" / Slack block with "CodePulse Test Alert"
- Returns `{ success: boolean, error?: string }`
