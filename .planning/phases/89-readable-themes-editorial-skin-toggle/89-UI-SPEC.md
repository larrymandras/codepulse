## Token Architecture (TH-01)

All themes share the same token names. A theme = a `[data-theme="slug"]` block that overrides the full token set defined in `.dark`. No token is hardcoded inside a component тАФ every color/glow/contrast value reads from `var(--token)`.

### Canonical token set (every theme must define all of these)

**Surface tokens:**
- `--background` тАФ dominant surface (body/page)
- `--card` / `--popover` тАФ elevated surface
- `--foreground` / `--card-foreground` / `--popover-foreground` тАФ primary text on those surfaces
- `--secondary` / `--muted` тАФ recessed surface (sidebar wells, input bg)
- `--secondary-foreground` / `--muted-foreground` тАФ text on recessed surfaces
- `--border` / `--input` тАФ hairline dividers and form outlines
- `--ring` тАФ focus ring

**Brand tokens:**
- `--primary` тАФ accent fill (buttons, active nav indicator, chart-1)
- `--primary-foreground` тАФ text atop `--primary` fills
- `--accent` тАФ secondary accent (hover state, tooltips)
- `--accent-foreground`

**Status tokens (must all be WCAG-AA on `--background`):**
- `--status-ok` / `--metric-ok`
- `--status-warn` / `--metric-warn`
- `--status-error` / `--metric-error`
- `--status-info` / `--metric-info` / `--info`
- `--destructive` / `--destructive-foreground`

**Chart tokens:**
- `--chart-1` through `--chart-5`
- `--chart-bar` / `--chart-bar-accent`
- `--chart-p50` / `--chart-p95` / `--chart-p99`

**Effect tokens (set to `none` / `0` to disable effects in readable/aubergine):**
- `--glow-xs` / `--glow-sm` / `--glow-md` / `--glow-lg` тАФ box-shadow glow values
- `--glass-bg` / `--glass-border` / `--glass-blur` тАФ glassmorphism

**War Room / speaking indicator:**
- `--speaking-ring` / `--speaking-ring-glow`

**Sidebar:**
- `--sidebar` / `--sidebar-foreground` / `--sidebar-primary` / `--sidebar-primary-foreground`
- `--sidebar-accent` / `--sidebar-accent-foreground`
- `--sidebar-border` / `--sidebar-ring`
- `--sidebar-active-bar`

**Canvas resolver (TH-01):**
JS themes must also be readable by canvas-rendered graphs (`ForceGraphCanvas`, `CodeVaultGraph`, KG Explorer). These do not read CSS variables natively тАФ a `useThemeColors()` hook resolves the current theme's named colors to hex/rgba by reading `getComputedStyle(document.documentElement)`. The hook must cache on `data-theme` change. Consumed by G3D (Phase 91) тАФ this is the hard dependency.

## Theme Switcher тАФ Interaction Contract (TH-05)

### Widget
- Location: `DashboardLayout.tsx` header chrome тАФ existing position at line 708, between `PrivacyShield` and `CrtToggle`.
- Component: extend the existing `ThemeSwitcher.tsx` (already a shadcn `Select` wrapping 3 items). Add `readable` and `aubergine` options; remove `amber` from user-facing list.
- Width: `w-[160px]` (extend from current `w-[140px]` to accommodate longer labels).
- Height: `h-8` тАФ unchanged.
- Icon: `Paintbrush` from Lucide тАФ unchanged.

### localStorage key consolidation
Two keys currently exist: `"theme"` (dark/light toggle, `DashboardLayout.tsx:566`) and `"codepulse-theme"` (accent selector, `ThemeSwitcher.tsx:17`). These must be consolidated to one key: `"codepulse-theme"`. The `"theme"` key and its dark/light toggle logic in `DashboardLayout.tsx:220-231` are removed тАФ `class="dark"` is now permanent (all themes are dark variants). Migration: on init, if `"theme"` key is `"light"`, treat as `"readable"` (closest equivalent), then delete `"theme"` key.

### No-flash pre-paint script (TH-05)
A blocking inline `<script>` in `index.html` (before `main.tsx`) reads `localStorage.getItem("codepulse-theme")` and sets `document.documentElement.setAttribute("data-theme", ...)` synchronously, before the React bundle loads. This eliminates FOUC (flash of un-themed content). The script must:
1. Read `"codepulse-theme"` key (not `"theme"`).
2. Default to `"cyan"` if absent or invalid.
3. Set `data-theme` attribute immediately.
4. Set/preserve `class="dark"` unconditionally.
5. Be < 200 bytes minified тАФ no external dependencies.

### Transition behavior
- Theme switch applies by setting `data-theme` attribute тАФ all CSS custom properties update via the cascade immediately.
- No JavaScript-driven color interpolation тАФ pure CSS cascade transition.
- Optional: `transition: background-color 200ms ease, color 200ms ease` on `body` for smooth theme change. This transition is suppressed under `prefers-reduced-motion: reduce` by the existing global rule in `src/index.css:441-444`.

### prefers-reduced-motion (TH-05)
- Existing global rule at `src/index.css:441-444` suppresses all animations. This is sufficient.
- Additional per-theme: in `[data-theme="readable"]` and `[data-theme="aubergine"]`, the `body::before` paper-grain and `body::after` ambient gradient pseudo-elements are also disabled under `@media (prefers-reduced-motion: reduce)`.
- CRT scanline (`CrtToggle`) respects the existing global rule independently.
