## Component Inventory

### SDKSpendGuard card (replaces `SDKSpendCapGauge` in-place)

Behavior spec (from CONTEXT.md D-01, D-02, D-03):
- Single card in the Analytics page, same slot as current `SDKSpendCapGauge`
- Export contract preserved: `classifyCapStatus()` and `DAILY_CAP` / `ALERT_THRESHOLD` remain exported
- Structure top-to-bottom:
  1. Section heading: `text-xs font-mono tracking-widest text-primary uppercase` — "SDK DAILY CAP"
  2. Metric row: spend value (`text-xl font-semibold tabular-nums`) + " of " + cap (`text-sm text-muted-foreground`) — e.g. "$2.31 of $5.00 today"
  3. Gauge bar: full-width `h-2 bg-muted rounded-none` with fill using `--status-ok/warn/error` based on `classifyCapStatus()`. 80% threshold tick mark at `left: 80%` using `bg-[--status-warn]` (preserved from existing component)
  4. Sparkline: `Sparkline` component (existing `src/components/Sparkline.tsx`), width = container width, height = 40px, color = `#10b981` when OK, `#eab308` when warning, `#ef4444` when exceeded. Renders hourly spend buckets from `costByPeriod` with `billingType='api'` and `period='hourly'`
  5. Projection row: `text-sm text-muted-foreground` — "Projected: $X.XX today" — shown when projection differs from current spend by >5%. If projection exceeds cap: `text-[--status-warn]` + clock icon (Lucide `Clock`) + copy per Copywriting Contract
  6. Status badge: `Badge` component — variants per `classifyCapStatus()` output
- Loading state: skeleton rows matching the 6-element structure above
- Data source: `costByPeriod` with `billingType='api'`, `period='hourly'`, `lookbackDays=1`. Client-side extrapolation for projection (CONTEXT.md D-03).

### ProviderControls panel (new section on Settings page)

Behavior spec (from CONTEXT.md D-04, D-05, D-06, D-07):
- Settings page section heading pattern: matches existing `NotificationChannels` / `LLMProviderConfig` sections — `text-xs font-mono tracking-widest text-primary uppercase` heading, `border-b border-border pb-2` separator
- Section label: "GATEWAY PROVIDERS"
- Per-provider row (drag-to-reorder via `@dnd-kit`):
  - Drag handle: `GripVertical` icon (Lucide, 16px, `text-muted-foreground`), 44px row height
  - Provider color dot: 8px circle, color from provider family table above
  - Provider display name: `text-sm text-foreground`
  - Billing type badge: `bg-yellow-500/20 text-yellow-400 text-[10px] font-mono uppercase px-1.5 py-0.5` for API-billed; `bg-gray-700/50 text-gray-400` for subscription (matches `ProviderHealthPanel` pattern)
  - Enable/disable toggle: shadcn `Switch` component, checked=green ring (`ring-[--status-ok]`), unchecked=default. Sends real WebSocket command (CONTEXT.md D-04)
  - Priority number: `text-xs text-muted-foreground tabular-nums` — shown as 1-indexed ordinal in list position
- Disabled provider visual: row opacity-50, strike-through on name using `line-through text-muted-foreground`
- Saving indicator: `Loader2` icon spinning at 16px on the row being changed (optimistic UI per Phase 2 WebSocket pattern)
- Empty state (no providers): "No gateway providers configured." `text-sm text-muted-foreground`

### Provider badges (session views)

Behavior spec (from CONTEXT.md D-08, D-09, D-10):
- Component: shadcn `Badge` with custom className overrides — no new component
- Session timeline tool call events: append badge immediately after event icon, before event type label
- Badge structure: `text-[10px] font-mono uppercase px-1.5 py-0.5 rounded-none` (respects `--radius: 0`)

- Always rendered — not conditional on mixed-provider sessions (CONTEXT.md D-09)

### RoutingAuditTable (upgrade of `RoutingDecisionsTable`)

Behavior spec (from CONTEXT.md D-13):
- Upgrade existing `RoutingDecisionsTable.tsx` in-place — same file, same export
- New columns added: "Score" summary column (shows `finalScore` as a compact `0.000` value inline) and "Reason" column (shows "FALLBACK" badge or provider match icon)
- Expanded row content upgrade: show all 4 scores in labeled pill chips (`bg-muted text-muted-foreground text-xs font-mono`) rather than inline `span` grid — `quota`, `latency`, `cost`, `final`

- Loading state: 3 skeleton rows matching table structure

## Interaction Contracts

### Spend projection algorithm (client-side)

- Fetch hourly buckets for today (hours 0–current hour)
- Sum spend so far → `todaySpend`
- Compute hourly average: `todaySpend / elapsedHours`
- Projected end-of-day: `todaySpend + (hourlyAvg * remainingHours)`
- Show projection only when `remainingHours > 0` and `elapsedHours >= 2` (at least 2 data points)
- "At current rate, you'll hit $5 by {time}" — compute time = `now + (DAILY_CAP - todaySpend) / hourlyAvg` expressed as local time (`HH:MM AM/PM`)

### Provider enable/disable interaction

- Optimistic UI: toggle flips immediately in UI
- WebSocket command sent via existing Phase 2 command sender
- Persisted to Convex `providerConfig` table on ack
- On error: toggle reverts, toast error message: "Failed to update {Provider Name}. Try again."
- Disabled state blocks new task routing immediately (CONTEXT.md D-04)

### Drag-to-reorder priority

- Uses `@dnd-kit/core` + `@dnd-kit/sortable` (already installed from Phase 4 Kanban)
- `SortableContext` wrapping provider rows
- `DragOverlay` shows ghost row at 80% opacity during drag
- Priority order persisted to Convex on `dragEnd`, not on every `dragMove`
- Visual feedback: active drag row gets `opacity-50 bg-muted` background

### Session timeline badge rendering

- Badge appended to each `ToolUse` event row in `SessionTimeline`
- Badge reads `event.provider` field (added in Phase 66)
- If `event.provider` is absent: badge is omitted silently — no placeholder
- Badge is always rendered when provider field is present (not conditional on session having multiple providers)
