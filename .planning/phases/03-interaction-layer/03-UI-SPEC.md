## Spacing Scale

Standard 8-point scale. All spacing must be multiples of 4px.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding between icon + label |
| sm | 8px | Compact element padding (badge padding, tight rows) |
| md | 16px | Default element spacing, chat bubble padding |
| lg | 24px | Section padding, panel internal padding |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Command palette inner padding: 8px vertical, 16px horizontal per `CommandItem`
- Keyboard focus ring: 2px offset (not a spacing value — ring only)
- Approval block left stripe: 4px width (`border-l-4`) — structural, not spacing
- Touch targets minimum: 44px height for approve/reject buttons in ApprovalBlock

## Typography

All sizes from the established Phase 1 system. Geist Variable for all UI text; JetBrains Mono for code/diff blocks only.

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px | 400 (regular) | 1.5 | Chat messages, inbox item body, run block text, insights responses. Code and diff blocks use JetBrains Mono at this same 14px size, weight 400, line-height 1.6. |
| Label | 12px | 400 (regular) | 1.4 | Metric labels (`uppercase tracking-wide`), section headers, timestamps, keyboard shortcut hints |
| Heading | 16px | 600 (semibold) | 1.3 | Panel titles, command palette group headings, round accordion headers |
| Display | 24px | 600 (semibold) | 1.2 | MetricBlock values (`tabular-nums`), large count displays |

Declared sizes: 12, 14, 16, 24. Maximum of 4 — no exceptions.

| Secondary (30%) | `oklch(0.985 0 0)` — off-white | `oklch(0.205 0 0)` — dark gray | Sidebar, card surfaces, command palette background, inbox row hover, run accordion headers |
| Muted | `oklch(0.97 0 0)` | `oklch(0.269 0 0)` — charcoal | Input backgrounds, empty state backgrounds, inactive round accordion |
| Muted foreground | `oklch(0.556 0 0)` | `oklch(0.708 0 0)` | Secondary text, timestamps, keyboard hints, placeholder text |
| Primary / Accent (10%) | `oklch(0.205 0 0)` | `oklch(0.922 0 0)` | Reserved for: active nav item highlight, approve button background, command palette selected item, streaming pulse dot |
| Status OK | `oklch(0.65 0.15 142)` | `oklch(0.72 0.17 142)` — green | Approve button, success toast, live connection indicator |
| Status Error | `oklch(0.65 0.18 27)` | `oklch(0.72 0.20 27)` — red | Reject button, error state, alert inbox stripe, destructive actions |
| Status Warn | `oklch(0.75 0.12 85)` | `oklch(0.80 0.14 85)` — amber | Approval pending inbox stripe, high-risk approval block stripe |
| Destructive | `oklch(0.577 0.245 27.325)` | `oklch(0.704 0.191 22.216)` | Reject/cancel buttons and confirmations only |

Accent reserved for (explicit list — these elements only):
1. Active sidebar nav item background
2. Approve button (primary action) in ApprovalBlock and InboxCard
3. Command palette `CommandItem` when keyboard-focused/hovered
4. Streaming pulse dot on the active Live Run round
5. "Send" button in ChatInput
6. Primary CTA button in InsightsChat

Do NOT use accent on: table row hovers, general hover states, metadata text, section dividers.

### Approval Block Left Stripe
Mirrors InboxCard exactly:
- `border-l-4 border-(--status-warn)` — approval pending
- `border-l-4 border-(--status-error)` — alert, high-risk
- `border-l-4 border-(--primary)` — notification / read items have no stripe

## Focal Points

Primary visual anchor per screen. Executor must ensure each screen has exactly one dominant focal element.

| Screen | Primary Focal Point |
|--------|---------------------|
| Command Palette (overlay) | The `CommandInput` search field — full-width, cursor auto-focused, visually isolated in the modal header |
| Agent Chat | The assistant message bubble stream — left-aligned, fills the chat column; the most recent assistant turn is the natural eye anchor |
| Inbox | The leftmost 4px status stripe on the top-most unread `InboxCard` — color signals urgency before any text is read |
| Live Run | The active (expanded) round in the accordion — amber left stripe + streaming content draws the eye to the current execution state |
| Insights Chat | The `ChatInput` at the bottom of the page — blank slate on first load, invites interaction as the single interactive element visible |

## Interaction Contracts

### Command Palette (IL-01)

- **Trigger:** `Cmd+K` (macOS) / `Ctrl+K` (Windows/Linux) from any page
- **Close:** `Esc`, click outside, or selecting an item
- **Layout:** Full-width `CommandDialog` modal, centered, max-width 560px
- **Search behavior:** Filters all groups simultaneously as user types. Client-side filter across pre-loaded Convex data.
- **Result grouping:** Four `CommandGroup` sections with `CommandSeparator` between: `Agents` | `Sessions` | `Alerts` | `Quick Actions`
- **Item selection:** `Enter` or click navigates to entity page and closes palette
- **Empty state:** `CommandEmpty` renders "No results found." in muted-foreground at 14px
- **Quick actions list (exact labels):**
  - "Send task to agent" → navigates to `/chat`
  - "View Unified Inbox" → navigates to `/inbox`
  - "Mute all alerts" → dispatches mute command, shows toast confirmation
  - "Navigate to Insights Chat" → navigates to `/insights`

### Inbox Keyboard Navigation (IL-02)

- **Arrow Up / Arrow Down:** Moves focus between `InboxCard` items. Focus ring: `ring-2 ring-ring ring-offset-1`
- **Enter:** Expands or collapses the focused item
- **A (when focused on approval item):** Triggers approve action — same as clicking Approve button
- **R (when focused on approval item):** Opens reject text input inline
- **Escape:** Clears keyboard focus, returns to mouse mode

### Generative UI Blocks in Chat (IL-03 / IL-04)

- **User message bubbles:** Right-aligned, `bg-primary text-primary-foreground`, 14px body
- **Assistant message bubbles:** Left-aligned, `bg-muted` background, 14px body
- **Block rendering:** Blocks render inline within the assistant message flow — no separate "block area"
- **Block borders:** Each block within an assistant turn is separated by an 8px vertical gap only. No card borders, no shadow.
- **Streaming state:** Active turn shows a blinking cursor `|` at 1Hz. Pulse animation via existing `live-update-pulse` keyframes.
- **Approval blocks in chat:** Full `ApprovalBlock` renders inline. After approve/reject, the block collapses to a one-line confirmation ("Approved — sent to Ástríðr" or "Rejected") in muted-foreground.
- **Unknown block fallback (D-06):** Renders the block's `content` field as markdown using the standard markdown renderer — graceful degradation, always shows something.

### Live Run Widget (IL-05)

- **Accordion state:** One `shadcn Accordion` (use CSS-only, no shadcn install needed — build with `<details>`/`<summary>` or Radix primitive). Completed rounds: collapsed, show round index + tool call count summary. Active round: expanded, shows streaming content.
- **Round header format:** `Round {N}` + muted tool call count (e.g., "3 tool calls") + completion checkmark when done
- **Active round indicator:** Amber `--status-warn` left stripe on the active round header row (4px)
- **Stop button:** Top-right of Live Run panel header. Icon: Lucide `Square` (16px). Label: "Stop". Calls `sendCommand({action: "run.stop"})`. Disabled when no active run.
- **Flow tab:** Rendered via `@xyflow/react`. Node style: `bg-secondary border border-border text-foreground`, 12px label. Edge style: `stroke: var(--muted-foreground)`. Layout: dagre top-to-bottom directed graph.

### Insights Chat (IL-06)

- **Page location:** Sidebar under `INSIGHTS` section. Nav label: "Insights". Icon: Lucide `MessageSquare` (16px).
- **Input placeholder:** "Ask about cost, errors, sessions..."
- **Responses:** Same `ChatBubble` + `BlockRenderer` as Agent Chat. Blocks render natively (MetricCard for numbers, TableBlock for lists).
- **Thinking state:** "Querying..." loading indicator — `Loader2` spinning icon (16px) + "Querying your data..." in muted-foreground
- **No streaming:** Insights responses arrive complete (not streamed). Show loader until full response received.

## Wire Protocol

The Generative UI Block JSON schema defines the contract between Ástríðr's WebSocket events and the `BlockRenderer`. This is the shared protocol for both Agent Chat and Insights Chat.

```typescript
// GenerativeBlock — wire format for run.block events
type GenerativeBlock =
  | { type: "metric"; label: string; value: string | number; trend?: "up" | "down" | "neutral" }
  | { type: "table"; columns: string[]; rows: (string | number)[][] }
  | { type: "chart"; data: { label: string; value: number }[]; title?: string }
  | { type: "code"; language: string; content: string }
  | { type: "diff"; before: string; after: string; language?: string }
  | { type: "approval"; requestId: string; action: string; details: Record<string, unknown>; riskLevel: "high" | "medium" | "low"; agentName?: string }
  | { type: "markdown"; content: string }
  | { type: string; [key: string]: unknown }  // fallback → render content field as markdown (D-06)

// WebSocket event envelope — subscribe via subscribeEvent("run.block", cb)
interface RunBlockEvent {
  event_type: "run.block"
  session_id: string
  block: GenerativeBlock
}
```

Source: RESEARCH.md Pattern 2 (wire protocol proposed). Confirmed with Ástríðr v4.0 Phase 48 before coding the Chat WebSocket subscriber.
