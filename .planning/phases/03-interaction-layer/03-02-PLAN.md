**Step 2: Create sub-components** (all in `src/components/blocks/`):

`MetricBlock.tsx`:
- Import MetricCard from `@/components/MetricCard`
- Accept `{ block: MetricBlockData }` prop
- Render `<MetricCard label={block.label} value={block.value} trend={block.trend} />`
- Minimal wrapper — no extra styling

`TableBlock.tsx`:
- Accept `{ block: TableBlockData }` prop
- Render an HTML `<table>` with `<thead>` from `block.columns` and `<tbody>` from `block.rows`
- Add local state for sort column and direction
- Clicking a column header toggles sort. Sort rows by that column.
- Style: `text-sm`, `border-collapse`, `border-(--border)` for cells, `bg-(--secondary)` for header row
- Each header cell: `cursor-pointer select-none uppercase tracking-wide text-xs text-(--muted-foreground)` (matches Phase 1 SectionHeader pattern)

`ChartBlock.tsx`:
- Import FlexBarChart from `@/components/FlexBarChart`
- Accept `{ block: ChartBlockData }` prop
- Render optional `block.title` as a 12px uppercase muted label above the chart
- Map `block.data` to FlexBarSegment array (label + value, no color override)
- Render `<FlexBarChart data={segments} />`

`CodeBlock.tsx`:
- Import `Prism as SyntaxHighlighter` from `react-syntax-highlighter` and `oneDark` theme (same as ChatBubble.tsx)
- Accept `{ block: CodeBlockData | DiffBlockData; diff?: boolean }` props
- For `code` type: render `<SyntaxHighlighter language={block.language} style={oneDark}>{block.content}</SyntaxHighlighter>`
- For `diff` type: render two panels side-by-side — `before` on the left labeled "Before", `after` on the right labeled "After", each in SyntaxHighlighter. Use `grid grid-cols-2 gap-2`. Labels are 12px uppercase muted.
- Font: JetBrains Mono 14px (per UI-SPEC typography)

`ApprovalBlock.tsx`:
- Accept `{ block: ApprovalBlockData; onApprove?: (requestId: string) => void; onReject?: (requestId: string, reason?: string) => void }` props
- Local state: `status: "pending" | "approved" | "rejected"` starts as `"pending"`
- When `status === "pending"`: render card with left stripe based on riskLevel:
  - `"high"` → `border-l-4 border-(--status-error)`
  - `"medium"` → `border-l-4 border-(--status-warn)`
  - `"low"` → `border-l-4 border-(--primary)`
- Card content: `block.action` as title (16px semibold), `block.agentName` as subtitle (12px muted), `block.details` rendered as key-value pairs in 14px
- Two buttons at bottom: "Approve" (bg-[--status-ok] text-white, min-h 44px) and "Reject Request" (bg-[--destructive] text-white, min-h 44px)
- On approve click: call `onApprove(block.requestId)`, set status to `"approved"`
- On reject click: show inline textarea with placeholder "Optional: explain rejection...", submit button calls `onReject(block.requestId, reason)`, set status to `"rejected"`
- When `status === "approved"`: collapse to single line `"Approved — sent to Ástríðr"` in muted-foreground 14px
- When `status === "rejected"`: collapse to single line `"Rejected"` in muted-foreground 14px

**Step 3: Create BlockRenderer.tsx**:
- Import GenerativeBlock from `@/types/generative-blocks`
- Import all 5 block components from `@/components/blocks/*`
- Import ReactMarkdown from `react-markdown` and remarkGfm
- Export `BlockRenderer({ block, onApprove, onReject }: BlockRendererProps)`
- Switch on `block.type`:
  - `"metric"` → `<MetricBlock block={block} />`
  - `"table"` → `<TableBlock block={block} />`
  - `"chart"` → `<ChartBlock block={block} />`
  - `"code"` → `<CodeBlock block={block} />`
  - `"diff"` → `<CodeBlock block={block} diff />`
  - `"approval"` → `<ApprovalBlock block={block} onApprove={onApprove} onReject={onReject} />`
  - `"markdown"` → `<ReactMarkdown remarkPlugins={[remarkGfm]}>{block.content}</ReactMarkdown>`
  - `default` → `<ReactMarkdown remarkPlugins={[remarkGfm]}>{"```json\n" + JSON.stringify(block, null, 2) + "\n```"}</ReactMarkdown>` (D-06 fallback)
- Each block separated by 8px vertical gap when rendered in sequence (caller handles via `space-y-2` or `gap-2`)
