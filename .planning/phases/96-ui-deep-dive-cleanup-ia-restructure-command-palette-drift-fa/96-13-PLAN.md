<objective>
Close the two CodePulse-side gaps found in live UAT (96-HUMAN-UAT.md):

- **Gap 1 (primary — confirmed live false success):** An Inbox approval that the server rejected still flipped the card to "Approved". The shared `useApprovalActions` hook already returns a boolean and already toasts the error correctly, but `Inbox.tsx` swallows that boolean (returns `Promise<void>`) and `InboxCard.tsx` calls `setApproved(true)` unconditionally. The T-96-03-01 fix landed in Chat's `ApprovalBlock` (which gates on the boolean at ApprovalBlock.tsx:51) but never reached `InboxCard`. Wire the boolean end-to-end so the card only commits state on a server ack.
- **Gap 2 (event-name drift):** `Chat.tsx:198` subscribes to `run.block` (singular), which the Ástríðr backend never emits — it emits `run.blocks` (plural, a `blocks` array; loop.py:1440, post_turn_pipeline.py:437, ws_telemetry.py live-runs topic). The Chat block-rendering path is therefore dead code, exercised only by tests. Align the subscription to the real event so Chat block rendering has a live path.

Purpose: This is the same access-control repudiation class as 96-03 (V4/T-96-03-01) — the second HITL consumer (Inbox) must not claim success the server never gave. Gap 2 removes dead drift so the live block path actually fires.

Output: Boolean-gated InboxCard, boolean-returning Inbox handlers, Inbox regression specs, and a Chat subscription aligned to `run.blocks`.
</objective>

The CORRECT gating pattern to mirror is already in the repo — ApprovalBlock.tsx:44-59:
```typescript
const handleApprove = async () => {
  if (!onApprove) { setStatus("approved"); return; }
  setBusy(true);
  try {
    if (await onApprove(block.requestId)) setStatus("approved");
  } catch {
    // stay pending on a throwing callback — never render a false "Approved"
  } finally {
    setBusy(false);
  }
};
```
