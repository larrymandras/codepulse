## Spacing Scale

Declared values (must be multiples of 4) — unchanged from the rest of the app, this phase does not touch the spacing primitive:

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding (nav icon-to-label gap) |
| sm | 8px | Compact element spacing (badge/pill padding, gap-2) |
| md | 16px | Default element spacing (page content padding `p-6` region is `main` — component-level default stays 16px) |
| lg | 24px | Section padding (card padding, PageHeader vertical rhythm) |
| xl | 32px | Layout gaps (between stacked sections on a page) |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (rare; not used inside dashboard shell) |

**Exceptions for this phase:**
- **F8 mobile collapse controls** (ForgePage master-detail toggle, WarRoom sidebar toggle): the toggle button hit target is **44×44px minimum** regardless of visual icon size (touch-target accessibility exception, not a new spacing token — implement via padding on a `size-11` / `min-h-11 min-w-11` button wrapping a smaller Lucide icon). These buttons are icon-only, so each MUST carry an explicit `aria-label` describing its action (e.g. `aria-label="Show job list"` / `aria-label="Collapse sidebar"`), consistent with the F10 ThemeSwitcher accessibility fix.
- No other exceptions. The existing `max-h-[500px]` caps on Chat/LiveRun/Inbox/Tasks (F7) are being **removed**, not preserved — they were never a sanctioned spacing exception, just leftover promoted-widget styling (`Tasks.tsx:98`).

## Typography

This phase's headline typography deliverable is the shared `<PageHeader>` component (F7) — the ONE standard 4/35 pages already follow, evidenced at `BuildProgress.tsx:24`, `Analytics.tsx:86`, `ForgePage.tsx:145`. All 35 pages migrate to it; no page keeps a bespoke header size/weight after this phase.

| Role | Size | Weight | Line Height | Evidence / Usage |
|------|------|--------|-------------|-------------------|
| Body | 14px (`text-sm`) | 400 (regular) | 1.5 | Descriptions, empty-state body copy, muted helper text (e.g. `Tasks.tsx:109` `text-base text-muted-foreground` — standardize to `text-sm` per this contract) |
| Label | 12px (`text-xs`) | 500 (medium) | 1.4 | Nav items, telemetry readouts, badges, uppercase-mono micro-labels (`DashboardLayout.tsx:704` `text-xs font-mono tracking-widest`) |
| Heading (PageHeader h1) | 24px (`text-2xl`) | 700 (bold) | 1.2 | Page title — THE standard per F7: `text-2xl font-bold text-foreground`. Replaces all divergent forms: `font-semibold` (Executions, Ideation), `text-xl` (Dreaming, MeetingBot, MissionControl→Tasks-merged, WhatsApp, WarRoom, LiveRun, Inbox, hr/*), `text-lg` (ConfigPage, **Tasks.tsx:101**), `text-base` (Chat, InsightsChat), terminal micro-headers (Hive, GraphsHub, Skills), and the missing h1 on DocComments |
| Display | 20px (`text-xl`) | 700 (bold) | 1.3 | Reserved for dialog/modal titles only (e.g. `AlertDialogTitle`, ForgeStopConfirmDialog "Stop this job?") — NOT for page headers |

**Exactly 2 weights declared: 400 (regular) and 700 (bold).**

**Pre-existing exception (not to be changed by this phase):** shadcn `Button` primitives ship `font-medium` (500) and `text-base` by default (`src/components/ui/button.tsx:8`) — this is baseline shadcn styling, out of scope for this cleanup phase. ## Color

Theme is token-driven and runtime-switchable (`data-theme="cyan|emerald|readable|aubergine"`, default `cyan`) — this phase must **never hardcode hex**, only reference existing CSS custom properties. No new color tokens are introduced.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `var(--background)` | Page background, `main` content area |
| Secondary (30%) | `var(--card)` / `var(--secondary)` | Cards, sidebar, nav, panels, PageHeader container |
| Accent (10%) | `var(--primary)` | Active nav indicator, primary buttons, live-pulse indicator dot, focus rings, links |
| Destructive | `var(--destructive)` | Destructive actions only (Stop job, Delete room, Delete skill if wired) |

**Accent reserved for (explicit list — this phase touches these, nothing else):**
- The header telemetry live-pulse dot (`DashboardLayout.tsx:703`) — ONLY rendered when real data exists (see Copywriting Contract, honesty rule); no accent styling on a hidden/absent readout.
- `<PageHeader>` active-state affordances if any (e.g. active view-toggle segment for the merged Tasks board).
- Nav active-row indicator (`.nav-active-shadow`, already shipped — unchanged by this phase).
- Primary CTA buttons (shadcn `Button variant="default"`, already `bg-primary`) — no new accent surfaces introduced.

**Second semantic color:** `var(--destructive)` — reserved for destructive actions only, per the existing `ForgeStopConfirmDialog.tsx` pattern (two-step `AlertDialog` confirm, destructive-variant trigger button). This phase adds/preserves destructive actions honestly rather than introducing new red surfaces elsewhere (e.g. removing the fake "Valid" Security badge is a REMOVAL, not a new destructive-colored element).

## Copywriting Contract

**Governing rule for this phase (from CONTEXT.md discussion):** never render a fabricated number or status styled as live data. Wire it to real data, or remove/hide the readout. This rule generalizes to every honesty fix in F3/F4 and any similar case surfaced during execution.

| Element | Copy |
|---------|------|
| Primary CTA (merged Tasks board, D-01/D-02) | View toggle labels: **"By Status"** (default segment) / **"By Agent"** (second segment, deep-linkable via `?view=agent`) — not a modal CTA, a persistent segmented control in the PageHeader area |
| Header telemetry — data present | `SYS: {n}%` / `LAT: {n}ms` — real values from `systemResources` (Convex), same visual treatment as today (`text-primary font-bold`) |
| Header telemetry — no recent data (empty state) | **Hide the entire readout** (both `SYS:` and `LAT:` spans) — do not render "SYS: —" or "LAT: N/A"; absence of the element IS the honest state, per the no-fabricated-number rule |
| Security page — Audit Chain (after D-05 fix) | Remove "Valid" badge entirely. Replace the misleading "Entry count" label with: **"{n} events loaded"** (honest proxy label, not a chain-integrity claim) |
| Automation page — cron catalog (after D-06 fix) | Section label: **"Configured Schedules"** (not "Active Jobs" or similar live-implying label). Drop `enabled: true` badges and the `totalJobs ?? 12` fallback — show the real count of configured entries only, no fallback number |
| Empty state heading | Established app-wide pattern (evidenced `Tasks.tsx:108`, `Infrastructure.tsx:170`, `Memory.tsx:735`, `Dreaming.tsx:102`): **"No {noun} yet."** |
| Empty state body | Established pattern: **"{No-noun-yet sentence}. {Concrete next step}."** — e.g. `Tasks.tsx:109-111`: "Create a task with the + button in any column, or convert an ideation finding." Apply this exact two-part shape to any new empty state this phase touches (merged Tasks board keeps its existing copy unchanged; FactsTable shared component inherits whichever parent-page empty copy already exists on Memory/Dreaming — do not invent new wording) |
| Error state | Established pattern (evidenced `ConfigPage.tsx:93`, `ProviderControls.tsx:173,200`, `Roster.tsx:93`): **"Failed to {action}[, {context clause}]. Try again."** — problem description + explicit retry instruction. Use for any new error path this phase introduces (e.g. F6 approval-send failure after payload-contract fix) |
| Destructive confirmation | Established pattern (evidenced `ForgeStopConfirmDialog.tsx`): shadcn two-step `AlertDialog`. Title: **"{Action} this {noun}?"** (e.g. "Stop this job?", "Delete this skill?"). Description: state the irreversible consequence in plain language ("This cannot be undone."). Footer: `AlertDialogCancel` labeled **"Cancel"**, `AlertDialogAction` labeled **"Yes, {action} the {noun}"** (e.g. "Yes, stop the job", "Yes, delete the skill"). Apply this exact shape if F9's Skills `onDelete` no-op (`Skills.tsx`) is wired to a real handler rather than dropped |
