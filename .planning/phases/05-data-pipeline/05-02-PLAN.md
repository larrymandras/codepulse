<objective>
Add archived-row filters to all existing queries that touch the four high-volume tables, preventing stale data from polluting active query results once the archival cron runs.

Purpose: Per Pitfall 2 from RESEARCH.md, failing to add archived filters simultaneously with the schema change causes silent data corruption — cost totals inflate, "recent" lists show old data. This plan audits every query in events.ts, llm.ts, and analytics.ts and adds the filter. Per D-11, the Analytics page query swap to aggregates happens incrementally — this plan prepares the raw queries to be archive-aware while the aggregate queries (already created in Plan 01) accumulate data.
Output: All three Convex domain files updated with archived filters.
</objective>
