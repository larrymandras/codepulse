## Spacing Scale

Standard 8-point scale. All spacing values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, label-to-icon inline padding |
| sm | 8px | Tile internal gap between label and value row |
| md | 16px | Tile padding (px-2 py-1.5 translates to 8/6px — tile is compact) |
| lg | 24px | Grid gap between HeroStatsBar and adjacent sections |
| xl | 32px | Section padding |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- KPI tile `min-h-[72px]` — not a spacing token, it is a layout constraint for sparkline breathing room (D-11)
- Grid gap between tiles is `gap-3` (12px) — preserved from current implementation, not changed in this phase
- Tile has negative margin offset `-mx-2` (preserved from current layout)

### Phase 04 Tone Color Map

`thresholdTone()` returns one of four tones. Each tone maps to a specific OKLCH accent hue for the three-layer styling.

| Tone | Hue | Base Color | bg opacity | border opacity | text opacity |
|------|-----|-----------|-----------|---------------|-------------|
| `good` | 142° (health-green) | `oklch(0.70 0.15 142)` | 8% | 15% | 100% |
| `warn` | 80° (cost-amber) | `oklch(0.70 0.15 80)` | 8% | 20% | 100% |
| `danger` | 27° (alerts-red) | `oklch(0.70 0.18 27)` | 8% | 20% | 100% |
| `default` | per-tile accent hue | tile's assigned `--accent-{category}` | 8% | 15% | 100% |

> Source: `04-CONTEXT.md` D-05, D-06, D-07

Three-layer CSS pattern per tile:

```
background-color: color-mix(in oklch, var(--tone-color) 8%, transparent)
border: 1px solid color-mix(in oklch, var(--tone-color) 15%, transparent)
/* warn/danger border bumps to 20% */
color: var(--tone-color)  /* applied only to the stat value */
```

Accent reserved for:
- KPI stat value text (tone color at 100% opacity)
- KPI tile border (tone color at 15% or 20% opacity)
- KPI tile background fill (tone color at 8% opacity via `color-mix`)
- SVG sparkline stroke (accent at 20% opacity)
- SVG sparkline area gradient fill (accent at 8% opacity, bottom-anchored)
- `data-accent` radial gradient (Phase 03 existing, layer 2 behind sparkline layer 1)

Accent is NOT applied to: label text, sub-label text, the outer HeroStatsBar container border, grid gaps.

## Interaction Contracts

### Draw-In Animation (mount)

- Trigger: component mount
- Technique: SVG `strokeDasharray` / `strokeDashoffset` trick
- Duration: 600ms ease-out
- Direction: left-to-right curve draw
- Simultaneous: filled area fades in from `opacity: 0` to `opacity: 1` over same 600ms
- Stagger: tiles animate in sequence, each delayed by `tileIndex * 100ms` (0ms, 100ms, 200ms … 600ms for 7 tiles)
- Reduced motion: `@media (prefers-reduced-motion: reduce)` — instant render, no animation, no stagger (existing rule in `src/index.css` covers all animations via `animation-duration: 0ms`)

### Live Morph Animation (data update)

- Trigger: `useHeroStats` poll returns new data (every 5s)
- Technique: CSS `transition` on SVG `d` attribute OR Motion `animate` on path `d` prop — executor discretion (CONTEXT.md explicitly deferred this choice)
- Duration: 400ms ease-in-out
- Fill gradient: stays consistent during morph (gradient definition unchanged, only path changes)
- Reduced motion: disabled by existing `prefers-reduced-motion` rule

### Tile Click Navigation

## SVG Sparkline Background Specification

### BackgroundSparkline geometry

- SVG fills 100% width and 100% height of tile (positioned absolute, `inset-0`)
- `z-index`: 0 (behind radial gradient layer and text layer)
- Curve: cubic Bézier smooth polyline (Catmull-Rom or standard CP algorithm — executor discretion per CONTEXT.md)
- Padding: 4px top, 4px bottom, 8px left, 8px right (keeps curve from clipping tile edges)
- Gradient fill: `linearGradient` from accent color at 8% opacity (bottom) to transparent (top), anchored to SVG bottom edge
- Stroke: accent color at 20% opacity, stroke-width 1.5px
- Overflow: hidden on tile container (`overflow-hidden` on the tile div)

### Sparkline data mapping (7 tiles)

| Tile | Sparkline Source | Type |
|------|-----------------|------|
| Sessions | `stats.eventSparkline` (12 buckets × 5min) | Real |
| Error Rate | Derived: per-bucket error% from `stats.eventSparkline` counts — use error event proportion (executor discretion on exact derivation per CONTEXT.md) | Derived |
| Alerts | `[stats.activeAlerts]` → flat horizontal band (single value repeated 12×) | Synthetic flat |
| Security | `[stats.securityEvents]` → flat horizontal band | Synthetic flat |
| Memory Hit Rate | `[hitRateValue ?? 0]` → flat horizontal band | Synthetic flat |
| Durable Facts | `[durableFactsCount ?? 0]` → flat horizontal band | Synthetic flat |
| Advisor Savings | `stats.costSparkline` (12 buckets × 5min) | Real |

> Source: `04-CONTEXT.md` D-09, D-10; `convex/heroStats.ts`

## New CSS Tokens (Phase 04 additions to src/index.css)

Add under `/* Phase 04: Tone tokens */` comment block in `.dark {}` (and mirrored in `:root` at lower chroma):

```css
/* Phase 04: Tone tokens */
--tone-good: oklch(0.70 0.15 142);       /* green — same as accent-health dark */
--tone-warn: oklch(0.70 0.15 80);        /* amber — same as accent-cost dark */
--tone-danger: oklch(0.70 0.18 27);      /* red — same as accent-alerts dark */
/* default tone = per-tile var(--accent-{category}) */
```

`data-tone` attribute drives tone CSS custom property override on the tile element:

```css
/* Phase 04: data-tone attribute selectors */
[data-tone="good"]   { --tile-tone: var(--tone-good); }
[data-tone="warn"]   { --tile-tone: var(--tone-warn); }
[data-tone="danger"] { --tile-tone: var(--tone-danger); }
/* default: no data-tone attribute → --tile-tone falls through to tile's --accent-{category} */
```

The three-layer tile styling uses `var(--tile-tone)` as the base color for `color-mix` calls.

## Layer Architecture (tile visual stack)

From bottom to top (z-order):

| Layer | Element | z-index | Description |
|-------|---------|---------|-------------|
| 1 | `BackgroundSparkline` SVG | 0 (absolute, inset-0) | Cubic Bézier curve + gradient fill area |
| 2 | `data-accent` radial gradient | CSS background-image (Phase 03) | Left-edge color glow from Phase 03 |
| 3 | Tile content (label, value, sub) | 1 (relative) | Text and AnimatedNumber on top |

The two background layers (sparkline + radial gradient) are visually additive — sparkline texture + left-edge glow coexist without conflict.
