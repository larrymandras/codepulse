## Spacing Scale

Declared values (all multiples of 4px):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, chip inner padding, tight row gaps |
| sm | 8px | Compact element spacing, tab internal padding |
| md | 16px | Default card padding, form field spacing |
| lg | 24px | Section padding, Sheet content padding |
| xl | 32px | Split-panel gap between editor and preview iframe |
| 2xl | 48px | Page-level vertical rhythm between major sections |
| 3xl | 64px | Not used in this phase |

Exceptions:
- Monaco Editor container: min-height 320px (editor usability floor, not a spacing token)
- Asset thumbnail cells: 96px × 96px (fixed aspect square for image grid)
- Dropzone height: 80px (compact inline, not a spacing token)
- Touch targets for icon-only buttons: 40px minimum (accessibility floor)

1. Active nav indicator bar in sidebar (left border on current route)
2. Primary action buttons: "New Layout", "New Template", "Save", "Upload"

6. Sheet save button (`bg-primary text-primary-foreground`)

Note: indigo-600 (`#4f46e5`) mentioned in CLAUDE.md is the semantic accent from the v4.0 era. The Paperclip design language uses `--primary` (monochromatic oklch near-white in dark mode, near-black in light mode). Executor must use `--primary` tokens, NOT hardcoded indigo hex values.

## Page Structure Contract

### Route: `/email-templates`

```
DashboardLayout (existing)
└── EmailTemplatesPage
    ├── h1: "Email Templates"  [Cinzel, text-2xl, font-bold]
    ├── Page-level action row  [flex justify-between items-center mb-6]
    │   └── "New [Tab-specific entity]" Button  [primary, right-aligned]
    └── Tabs (shadcn/ui Tabs, 4 triggers)
        ├── Tab: Layouts
        ├── Tab: Templates
        ├── Tab: Agent Defaults
        └── Tab: Assets
```

Nav entry: add `{ to: "/email-templates", label: "Email Templates", icon: "mail", group: "OVERVIEW" }` to `overviewNavItems` in `DashboardLayout.tsx`. Icon: `Mail` from Lucide.

### Tab: Layouts

List of email layout records. Each row displays:
- Layout name (text-sm, font-medium)
- Slug (text-xs, font-mono, muted)
- Description (text-xs, muted, truncated to 1 line)
- "Edit" button → opens Sheet (D-02)
- "Delete" icon button → AlertDialog confirmation (destructive)

Use vertical list, not grid. Each item is a card row (`bg-card border-b border-border px-4 py-3`).

### Tab: Templates

Identical list pattern to Layouts. Each row adds:
- Associated layout name as a badge (outline variant)
- Variable count badge (e.g., "3 vars")

### Tab: Agent Defaults

Card grid, 3 columns on lg, 2 on md, 1 on sm (D-03).

Each card contains:
- Avatar thumbnail (48px × 48px, `object-cover`, or placeholder initials)
- Agent signature name (text-sm, font-semibold)
- Signature title (text-xs, muted)
- Assigned layout name (text-xs, muted, "Layout: Default")
- "Edit" button → opens Sheet

### Tab: Assets

Thumbnail grid, 4 columns on lg, 3 on md, 2 on sm (D-04, D-13).

Each cell: 96px × 96px, `object-cover`, dark background, filename below (text-xs, truncated, muted).

Toolbar above grid:
- Upload button (primary)
- Filter pills: All | Avatars | Logos

## Interaction Contracts

### Create/Edit Flow

1. User clicks "New Layout" (or "Edit" on a row).
2. Sheet slides in from right with 200ms ease-out transition (shadcn default).
3. For create: all fields pre-filled with empty defaults (name: "", slug: "", body: "").
4. For edit: all fields populated from API response on Sheet open.
5. User edits fields. "Save" button: `disabled` until form is dirty AND name is non-empty.
6. On save: show spinner in button label ("Saving…"), call API, close Sheet on success, show Sonner toast "Layout saved".
7. On API error: keep Sheet open, show inline error below the save button (text-sm text-destructive).

### Debounced Preview (D-10)

- Debounce delay: 500ms after last keystroke in any editor field.
- During debounce window: show a `Loader2` spinner (12px) in the preview panel header.
- On preview API call: replace spinner with subtle "Updating…" text (text-xs muted).
- On preview success: update iframe `srcdoc` with rendered HTML.
- On preview error: show error banner inside preview panel ("Preview failed — check template syntax").
- Preview does NOT auto-update on variable schema changes, only on body/layout field changes.

### Asset Dropzone (D-12)

- Idle state: dashed border (`border-dashed border-border`), gray background (`bg-muted/30`), 80px height, "Drop image here or click to browse" (text-xs muted, centered).
- Drag-over state: border transitions to `border-primary` with `bg-primary/5` background fill.
- Uploaded state: show thumbnail (full width, 80px height, `object-cover`) with "Replace" overlay on hover (semi-transparent dark overlay with "Replace" text in 12px white).
- File validation: client-side, before upload — max 5MB, accept `image/png,image/jpeg,image/webp`.
- On invalid file: inline error below dropzone (text-xs destructive).

### Asset Picker (D-14)

Rendered as a Dialog (not Sheet). Contains:
1. Existing assets grid (same thumbnail pattern as Assets tab)
2. "Upload New" button at top-right
3. Selection state: selected cell gets `ring-2 ring-primary` outline
4. "Use Selected" button: enabled only when a cell is selected

### Delete Confirmation Pattern

All destructive actions use `shadcn AlertDialog` (not browser `confirm()`):
- Title: "Delete [entity name]?"
- Description: exact copy per entity (see Copywriting Contract below)
- Cancel button: outline variant, no destructive styling
- Confirm button: `variant="destructive"` — red background

## Accessibility Contract

- All form inputs must have associated `<label>` elements (use shadcn `Label` component).
- Sheet panels must trap focus when open (`shadcn Sheet` handles this via Radix Dialog).
- AlertDialog must be keyboard-dismissible (Escape key) — shadcn default behavior.
- Asset thumbnail grid cells must have `alt` text from filename.
- Variable chip buttons: `aria-label="Insert {{variable_name}}"`.
- Monaco Editor: not fully accessible — acceptable for power-user developer tool context.
- Dropzone: `<input type="file" />` hidden behind dropzone, clickable for keyboard users.
- Minimum touch target for icon-only buttons: 40px × 40px (use `p-2` on h-4 w-4 icons).
