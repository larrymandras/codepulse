## Typography

Inherited from Phase 71 ¬ß2.7 de-facto scale. This phase introduces no new roles. The active type scale below is
restricted to what Phase 86 actually authors. (The page H1 `text-2xl font-bold` is an inherited Phase 71 component ‚Äî
already shipped on `KnowledgeGraph.tsx:196`, referenced in prose, not re-declared in this active scale.)

| Role | Size | Weight | Line Height |
|------|------|--------|-------------|
| Body (result snippet) | 14px (`text-sm`) | 400 (regular) | 1.5 (`leading-relaxed`) |
| Label (result meta: entity ¬∑ predicate) | 12px (`text-xs`) | 400 (regular), `font-mono` | 1.4 |
| Label (lens tab / legend / chrome) | 10‚Äì12px (`text-[10px]`/`text-xs`) `uppercase tracking-wide` | 600 (semibold) when active, 400 muted | 1.4 |
| Emphasis (matched term) | 14px (`text-sm`) | 600 (semibold), `text-primary` | 1.5 |

**Weights ‚Äî exactly 2 introduced by this phase:** 400 (regular) for content/snippets/meta, and 600 (semibold) for
the active "Search" lens tab and the matched-term emphasis. The 700/bold page H1 is inherited from Phase 71 ¬ß2.7
and is NOT re-declared here.

**Matched-term emphasis:** the substring that matched the query renders `font-semibold text-primary` inside an
otherwise `text-muted-foreground` snippet ‚Äî never a background highlight (keeps it on-palette, no yellow `<mark>`).

## Color

Inherited Phase 71 palette. The 60/30/10 split below describes THIS phase's two new surfaces only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` (`--background`) / `#141416` (`--card`) | Results panel surface, search-bar background, graph backdrop |
| Secondary (30%) | `#27272a` (`--secondary`/`--border`) | Result-row dividers, search input border, hover `bg-accent/50` |
| Accent (10%) | `#10b981` (`--primary`) | See reserved-for list below |
| Destructive | `oklch(0.704 0.191 22.216)` (`--destructive`) | Not used this phase ‚Äî no destructive actions in KG-08/KG-09 |

**Accent (`#10b981`) reserved for ‚Äî explicit list (never "all interactive elements"):**
- The active "Search" lens tab fill/border (`bg-primary/15 border-primary/50 text-primary` ‚Äî matches existing lens tabs).
- The matched-term emphasis inside result snippets (`text-primary font-semibold`).
- The selected/hovered result-row left-accent (`border-l-2 border-primary`) and the centered node's selection ring (already emerald).
- Search-input focus ring (`--ring`, already emerald via shadcn `Input`).

**Community-cluster palette ‚Äî SEPARATE from accent and from entity-type colors (Q3-C):**
Community color encodes graph topology, not brand. It must NOT reuse `#10b981` (entity-type "person" anchor) or
the entity-type ramp (`ENTITY_TYPE_COLORS`, `src/lib/kg-graph.ts:101`). Declare a fixed **8-color categorical
community palette**, assigned by `community % 8` so a given community id is stable across renders:

| Community slot | Hex | Note |
|----------------|-----|------|
| 0 | `#60a5fa` | blue-400 |
| 1 | `#f472b6` | pink-400 |
| 2 | `#fbbf24` | amber-400 |
| 3 | `#34d399` | emerald-400 (lighter than accent ‚Äî distinct) |
| 4 | `#a78bfa` | violet-400 |
| 5 | `#22d3ee` | cyan-400 |
| 6 | `#fb923c` | orange-400 |
| 7 | `#a3e635` | lime-400 |

- Community color is applied as a **ring/halo** around the node (drawn in `paintNode`), NOT as the node fill ‚Äî
  the node fill stays the entity-type color so both encodings coexist. Halo: `ctx.arc` stroke at `size + 3`,
  `lineWidth 2`, the community color at `globalAlpha 0.7`, soft `shadowBlur 6`.
- Nodes with `community == null` (vault nodes, un-clustered KG nodes) get NO halo ‚Äî they render exactly as today
  (no-regression, SC#4).

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Switch to Search lens | Lens tab `Search` activates (emerald active style). The Entity-lens name input hides; the full-text input shows. Other lens state preserved. |
| Type a query | Debounced (~250ms) call to √Åstr√≠√∞r `/api/kg/search`, scoped by the shared `entityType`/`agentId` filters (CONTEXT D-03). Loading state shows `Searching knowledge graph‚Ä¶`. Empty query ‚Üí "Search the knowledge graph" empty state (no fetch). |
| Click a result row | **Switches to the Entity (ego) lens AND focuses the result's subject entity** (CONTEXT D-02) ‚Äî NOT a center-in-place. Reuses the Phase 85 focus infrastructure (`buildFocusUrl` / `useFocusParam`) to set the entity-name filter, switch `lens` to `entity`, and center the subject entity in the ego graph (`centerNodeWhenReady`, `src/lib/graph-center.ts`). Selects it (`selectNode`), populates `KGDetailsPanel`. The clicked row gets `border-l-2 border-primary` selected accent. Inbound `hops` defaults to 1 (planner discretion, mirrors Phase 85). |
| Search endpoint unreachable | Error banner (red, mirrors existing `KnowledgeGraph.tsx:227`) with the endpoint-named copy + fallback note. Other lenses unaffected. |
| Endpoint 404/501 (not deployed) | Distinct "not available on this √Åstr√≠√∞r build yet" gated copy (cross-repo, SC#2) ‚Äî informational, not an error-red banner. |
| Community halos | Drawn automatically when nodes carry `community`; honor the existing `opts.dimmed` alpha and selection ring (halo sits under the selection ring). |
| Community spatial force | Engaged automatically when community data is present; disabled (force not registered) otherwise ‚Äî no layout change for legacy graphs (SC#4). Honors `prefers-reduced-motion` (no extra animated reflow beyond the standard force settle). |

## Cross-Repo Dependency (SC#2 ‚Äî flag for planner)

KG-08 full-text search depends on a **net-new √Åstr√≠√∞r `/api/kg/search` endpoint** that does not yet exist
(confirmed open blocker, `STATE.md` Blockers/Concerns + ROADMAP "key sequencing constraint" + CONTEXT D-01). The UI MUST:
1. Degrade gracefully when the endpoint is absent (404/501 ‚Üí the "not available on this build yet" gated copy,
   NOT a hard error) so the Search lens never crashes the page.
2. Keep the existing entity-name search (Entity lens, `/api/kg`) fully functional ‚Äî it is the documented fallback.
3. The planner must call out this cross-repo delta as a gated task and decide whether to ship the UI behind the
   gate (recommended ‚Äî the lens + graceful-degrade ship CodePulse-side; live results light up when √Åstr√≠√∞r deploys).
