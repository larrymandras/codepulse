**Cross-volume-safe move primitive to reuse/adapt verbatim** (`promote.ts` lines 59-109):
```typescript
const CROSS_VOLUME_CODES: ReadonlySet<string> = new Set([
  'EXDEV', 'EIO', 'EPERM', 'EACCES', 'ENOTSUP',
]);

function copyTreeReadWrite(src: string, dest: string): void {
  fs.mkdirSync(dest, { recursive: true });
  for (const entry of fs.readdirSync(src, { withFileTypes: true })) {
    const sp = path.join(src, entry.name);
    const dp = path.join(dest, entry.name);
    if (entry.isDirectory()) {
      copyTreeReadWrite(sp, dp);
    } else {
      fs.writeFileSync(dp, fs.readFileSync(sp));
    }
  }
}

export const defaultPromoteFn: PromoteFn = (src, dest) => {
  try {
    fs.renameSync(src, dest);
  } catch (err: unknown) {
    const code = (err as NodeJS.ErrnoException).code ?? '';
    if (CROSS_VOLUME_CODES.has(code)) {
      copyTreeReadWrite(src, dest);
      fs.rmSync(src, { recursive: true, force: true });
    } else {
      throw err;
    }
  }
};
```
**CRITICAL ADAPTATION (do not call verbatim):** `promoteWorkspace()` itself THROWS `ForgePromotionError` on cross-volume ŌĆö that is the opposite of lifecycle move's required behavior (cross-volume must SUCCEED via copy+delete, never refuse). Extract only `CROSS_VOLUME_CODES` + `copyTreeReadWrite`, write a new lifecycle-move function that tries `fs.renameSync` first and falls back to copy+delete on `CROSS_VOLUME_CODES` ŌĆö never throw-and-refuse for that error class. The reparse-point scanner (`defaultReparseScanner`, lines 133-163, `lstatSync`-based) is also directly reusable as a pre-move safety check (Security Domain finding, RESEARCH.md).

**Richer executor shape to copy (not `execute()`'s bare shape)** ŌĆö `executeIntake()`, lines 316-361, is the correct template because it demonstrates the post-success `rescanAndSync` trigger lifecycle also needs:
```typescript
private async executeIntake(cmd: ForgeCommand): Promise<void> {
  let status: 'done' | 'failed' = 'done';
  let error: string | undefined;
  let report: unknown | undefined;

  try {
    if (!this.intakeFn) {
      throw new Error('intake not supported on this host');
    }
    const result = await this.intakeFn(cmd.commandId, cmd.intakePayload!, cmd.downloadUrl ?? null);
    status = result.status;
    error  = result.error;
    report = result.report;

    if (status === 'done' && this.rescanCfg) {
      void rescanAndSync(this.rescanCfg);   // fire-and-forget, never awaited (T-97-08/09)
    }
  } catch (err) {
    status = 'failed';
    error  = err instanceof Error ? err.message : String(err);
  }

  await this.fetchImpl(this.ackUrl, {
    method: 'POST',
    headers: { 'Content-Type': 'application/json', Authorization: `Bearer ${this.apiKey}` },
    body: JSON.stringify({
      commandId: cmd.commandId, status,
      ...(error !== undefined && { error }),
      ...(report !== undefined && { report }),
    }),
  }).catch(() => null);
}
```
Add a new `executeLifecycle(cmd)` following this exact shape (mutable `status`/`error` locals, try/catch, unconditional fire-and-forget ack, conditional `rescanAndSync` on `status === 'done'`), dispatched from `poll()`'s parallel branch (`void this.executeLifecycle(cmd)`), NOT the serial `intakeQueue`.
