# Phase 87 — UI Design Contract
# Saved Views + Temporal Diff (KG-10, KG-11)

> Visual and interaction contract for KG Saved Views (KG-10) and Temporal Diff/Animation (KG-11).
> Generated by gsd-ui-researcher, no CONTEXT.md available — all decisions derived from prior art.
>
> **Scope:** This phase EXTENDS the existing KG Explorer (`src/pages/KnowledgeGraph.tsx`) and its
> control bar (`src/components/kg/KGControls.tsx`). It does NOT introduce a new page or route.
> It does NOT replace the existing Temporal lens (single-point as-of scrubber) — it ADDS two features
> that compose with it. Design system is the locked Phase 71 Matrix-Emerald skin. Every token below
> is referenced, not re-derived.

## Color

Inherited Phase 71 palette. The 60/30/10 split below describes THIS phase's new surfaces only.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` (`--background`) / `#141416` (`--card`) | Saved-views popover surface, diff controls backdrop |
| Secondary (30%) | `#27272a` (`--secondary`/`--border`) | View-row dividers, popover border, date-picker border |
| Accent (10%) | `#10b981` (`--primary`) | See reserved-for list below |
| Destructive | `oklch(0.704 0.191 22.216)` (`--destructive`) | Delete saved view confirmation only |

**Accent (`#10b981`) reserved for — explicit list:**
- The active "Diff" / "Animate" sub-lens tab fill/border (`bg-primary/15 border-primary/50 text-primary` —
  matches the existing lens-tab active style from `KGControls.tsx:57-60`).
- The "Save view" CTA button border/text in its idle state.
- The currently-loaded saved-view row left-accent mark (`border-l-2 border-primary`).
- The animation playback Play/Pause button icon while playing.

**Diff visual encoding — separate semantic palette (NOT accent, NOT entity-type colors):**
The diff encoding uses the existing `--status-*` tokens already in the design system. Nodes and edges
are re-colored in the diff view according to their change state:

| Change state | Color token | Hex | Node/edge treatment |
|--------------|-------------|-----|---------------------|
| Added (present in B, absent in A) | `--status-ok` / `#22c55e` | green-500 | Node fill + glow: green-500; edge stroke: `rgba(34,197,94,0.55)` |
| Removed (present in A, absent in B) | `--status-error` / `#ef4444` | red-500 | Node fill + glow: red-500; edge stroke: `rgba(239,68,68,0.40)`, dashed `[4,3]` |
| Changed (present in both, facts differ) | `--status-warn` / `#eab308` | amber-500 | Node fill + glow: amber-500; edge stroke: `rgba(234,179,8,0.55)` |
| Unchanged (identical in A and B) | Muted — `#a1a1aa` at `globalAlpha 0.35` | zinc-400 dimmed | Standard node fill at 35% alpha; edge stroke `rgba(161,163,170,0.15)` |

Rationale: reusing `--status-*` is the natural read (green=added, red=removed, amber=changed).
These tokens are already used consistently for semantic states across the dashboard. No new tokens needed.

**Diff legend** floats over the canvas in the same `bg-card/70 backdrop-blur border border-border`
absolute-positioned overlay box used by the entity-type legend (`KnowledgeGraph.tsx:379`), APPENDED below the
entity-type legend entries when diff mode is active. Auto-hides when diff mode is off.

Opens/closes `KGViewsPopover`. Also add a "Save view" button (`BookmarkPlus h-3.5 w-3.5`) next to it — always visible, single-click, no modal. Clicking "Save view" calls `onSaveView()` from props. |

Sections: (1) header row `"SAVED VIEWS"` label (`text-[10px] font-mono uppercase tracking-widest text-muted-foreground`) + count badge; (2) scrollable list of `KGViewRow` items (`max-h-[320px] overflow-y-auto custom-scrollbar`); (3) empty state when no views exist. Closes on outside click (Popover default). |
| **`KGViewRow`** | New | One saved view in the list. Layout: `flex items-center gap-2 px-3 py-2 hover:bg-accent/50 cursor-pointer transition-colors`. Left: `Bookmark h-3.5 w-3.5 text-primary/60`. | **`useSavedViews`** | New hook | Manages named view CRUD. Storage: Convex `savedKgViews` table (new). Shape: `{id, name, lens, filters (without searchQuery), createdAt, shareToken}`. Exposes: `views`, `saveView(name, lens, filters)`, `loadView(id)`, `deleteView(id)`, `buildShareUrl(id)`. See Interaction Contract for sharing URL. |
| **`KnowledgeGraph.tsx`** | Modified | Wire `onSaveView` and `onLoadView` callbacks. On "Save view" button click: prompt for a name via an inline `Input` (no modal — see Interaction Contract), then call `useSavedViews.saveView`. On load: apply `setLens` + `setFilter` for each restored field. |

**Save-name prompt (no modal — principled decision):** A brief inline expand of the "Save view" button into
a small `Input + confirm button` row (replaces the button in-place for the interaction). This keeps the save
action in the control bar without a blocking dialog. The name field is required; empty-name submit is a no-op
with a subtle shake animation (`animate-[shake_0.2s_ease-in-out]`). Confirmed by Enter key or the emerald
checkmark button. Escaped or blurred without entry cancels.

Rationale: a modal (shadcn `Dialog`) is heavier than the action warrants. The in-place expand is the
lightest option consistent with existing CodePulse pattern (cf. inline-expand patterns in `ChatInput.tsx`).

### KG-11: Temporal Diff + Animation

| Component | New/Modified | Contract |
|-----------|--------------|----------|
| **`KGControls`** | Modified | The existing Temporal lens row (containing the single `as-of` date Input + "Now" button) GAINS a mode toggle when `lens === "temporal"`: two segmented buttons `Point` | `Diff` | `Animate` (10px mono, same segmented-chip style as Phase 84's `Code`/`Vault`/`Both`). Default: `Point` (preserves existing single-as-of behavior). In `Diff` mode: show a second date Input (the existing one becomes "From" / as-of A; the new one is "To" / as-of B). In `Animate` mode: show the animation scrubber + controls (see below). These three sub-modes are purely UI-local state; the hook / lens value stays `"temporal"`. |
| **`KGDiffControls`** | New | Extracted sub-component for the `Diff` sub-mode row. Two `Input type="date"` pickers side by side: `From` label + input (as-of A) and `To` label + input (as-of B). A `Diff` / `Compare` button (shadcn `Button variant="secondary" size="sm" font-mono text-sm`) triggers the diff computation. The button is disabled until BOTH dates are set AND from < to. |
| **`KGAnimateControls`** | New | Extracted sub-component for the `Animate` sub-mode row. Contains: `Play/Pause` button (`Play`/`Pause` icon, shadcn `Button variant="ghost" size="icon"`), `StepBack`/`StepForward` icon buttons (frame-by-frame), a `Slider` (shadcn, inherits Phase 86 Slider style) spanning the date range (min=earliest snapshot, max=latest), and a `text-xs font-mono text-muted-foreground` readout of the current frame date. Speed: a small `Select` with `0.5×` / `1×` / `2×` options (shadcn `Select`, `w-20 font-mono text-sm`). |
| **`useKgDiff`** | New hook | Fetches the KG at two as-of points (two calls to `fetchOverview({ asOf })`) and computes the diff sets: `{added: Set<nodeId>, removed: Set<nodeId>, changed: Set<nodeId>}`. Returns loading state and the two raw graphs. The diff is ALWAYS node-id-based; edge diffing follows from node-set membership. Edge "changed" means the edge's `current`/`superseded` state changed between snapshots. |
| **`useKgAnimation`** | New hook | Manages the animation frame sequence. Fetches the list of available snapshot dates from Ástríðr (`fetchSnapshotDates()` — new kgApi function, returns `string[]` ISO dates sorted ascending). Exposes `{frames, currentFrameIndex, isPlaying, play, pause, stepForward, stepBack, setFrameIndex, fps}`. Each frame is a `{asOf: string}` descriptor; graph data is fetched lazily per frame (with a 2-frame prefetch lookahead). |
| **`paintNodeDiff`** | New paint fn | Applied instead of the default `paintNode` when diff mode is active. Takes a `diffState: "added" | "removed" | "changed" | "unchanged"` on each node and applies the diff color table (see Color section). For "unchanged" nodes, applies `globalAlpha 0.35`. Selection ring and hover behavior are unchanged — a selected node in diff mode still shows its diff color + a white fill on hover. |
| **`KnowledgeGraph.tsx`** | Modified | When `temporalSubMode === "diff"` and diff is computed: swap `paintNode` → `paintNodeDiff`, swap `linkColorFn` → `linkColorDiffFn`. The diff legend appended to the floating legend box. When `temporalSubMode === "point"`: existing behavior unchanged (SC — no regression on single-point Temporal lens). |

## Layout Contract

The KG Explorer page layout (`grid grid-cols-1 lg:grid-cols-[1fr_320px] gap-4`) is unchanged.
New affordances are confined to the `KGControls` bar and the canvas layer.

### KGControls — Phase 87 additions

```
[Lens tab strip: Overview | Entity (ego) | Temporal | Contradictions | Search]  [Views ▾]  [Save ✓]  [Refresh]
[Filter row: (lens-aware — unchanged for non-temporal lenses)]

When lens === "temporal":
  [Point | Diff | Animate] sub-mode toggle
  → Point mode (default):  [As of: <date input>]  [Now button]   ← unchanged from today
  → Diff mode:             [From: <date A>]  [To: <date B>]  [Compare button]
  → Animate mode:          [◀ step] [▶/⏸ play] [scrubber ─────────────] [date readout]  [speed: 1×]
```

**Sub-mode toggle is NOT a new lens tab** — it is a secondary control row that appears only when
`lens === "temporal"`, positioned between the lens tab strip and the shared entity-type/predicate filter row.
Rationale: the Temporal lens IS the as-of mechanism; diff and animation are modes of that same lens.
Adding them as sibling tabs would fragment the temporal concept and require the user to manage two
orthogonal controls. Sub-modes under Temporal keeps the IA coherent.

### Diff legend append (canvas overlay)

The existing floating legend (`absolute top-3 left-3 z-10`, `KnowledgeGraph.tsx:379`) gains a diff section
appended BELOW the entity-type rows, separated by a `border-t border-border mt-1 pt-1`, when diff mode is
active and diff data is loaded:

```
─────────────
DIFF
● added        (green-500 dot)
● removed      (red-500 dot)
● changed      (amber-500 dot)
○ unchanged    (zinc-400 dimmed dot)
```

Auto-hides when diff mode is off or diff data is not loaded.

## Interaction Contract

### KG-10: Saved Views

| Interaction | Behavior |
|-------------|----------|
| Click "Save view" | The button label becomes an inline `Input` (width `w-48`) + `Check h-3.5 w-3.5 text-primary` confirm button. Focus moves to the input. |
| Enter a name + confirm (Enter / checkmark) | Calls `useSavedViews.saveView(name, lens, filtersWithoutSearchQuery)`. The input collapses back to the "Save view" button. A `sonner` toast: `View "{name}" saved` (2 s). The new view appears at the top of the views list. |
| Empty name submit | No-op with `animate-[shake_0.2s_ease-in-out]` on the input. Does not save. |
| Escape / blur without confirm | Cancels the name entry. Input collapses. Nothing saved. |
| Click "Views" button | `KGViewsPopover` opens (shadcn `Popover`, anchored below the button). |
| Click a saved view row | Closes the popover, calls `useSavedViews.loadView(id)`. Applies `setLens(view.lens)` + `setFilter(k, v)` for each persisted filter field. The URL does NOT update (views are session navigation, not URL state). The loaded view row gets `border-l-2 border-primary` active accent until any filter changes. |
| Hover a saved view row | Reveals the `Copy link` (`Link h-3.5 w-3.5`) and `Delete` (`Trash2 h-3.5 w-3.5`) icon buttons on the right, fading in with `transition-opacity duration-150`. At rest they are hidden (`opacity-0`). |
| Click "Copy link" | Calls `navigator.clipboard.writeText(useSavedViews.buildShareUrl(id))`. Toast: `View link copied`. |
| Click trash icon | Deletes the view immediately (no confirmation). View row disappears. |
| Open shared link | URL carries `?view=<shareToken>`. On page load, `KnowledgeGraph.tsx` reads `?view`, resolves the view from Convex by `shareToken`, applies `setLens` + `setFilter` (one-shot, same as manual load). Same hydration guard as the existing `?focus` param (appliedFocusRef pattern). |
| Shared view not found | Token absent or expired — silent fallback to default state (no error banner, SC#3 parity). |

**Share URL shape:** `/knowledge-graph?view=<shareToken>` where `shareToken` is a short opaque ID stored
alongside the view in Convex. Composable with existing `?focus`/`?lens`/`?hops` params.
Rationale: reusing the `buildFocusUrl` / `?focus` param would require serializing the entire filter set into
the URL (unbounded length). A Convex-backed token is cleaner, supports revocation, and mirrors the
existing `idb` auto-persist pattern at a higher fidelity level.

### KG-11: Temporal Diff + Animation

| Interaction | Behavior |
|-------------|----------|
| Switch to Temporal lens | Sub-mode toggle appears (Point / Diff / Animate). Default sub-mode: `Point` — existing as-of scrubber visible, unchanged. |
| Switch sub-mode to Diff | From/To date pickers appear. Compare button disabled until both dates selected and from < to. |
| Select From + To dates + click Compare | `useKgDiff` fetches the two snapshots. Canvas shows `Diffing knowledge graph…` pulse. When loaded: nodes and edges re-colored per diff palette. Diff legend appended to floating legend box. |
| Switch sub-mode to Animate | Scrubber + controls appear. `useKgAnimation` fetches `fetchSnapshotDates()`. If dates loaded: scrubber range = [earliest, latest]. If no snapshots: empty state copy. |
| Click Play | Animation plays forward: advances `currentFrameIndex` at the configured `fps` (default 1×, approximately 1 frame per second). Each frame fetches / uses the cached snapshot for that `asOf` date. Canvas updates frame-by-frame. Play button icon becomes Pause. |
| Click Pause | Freezes at current frame. Pause button icon becomes Play. |
| Click StepForward / StepBack | Advances or retreats one frame. Pauses if playing. |
| Drag scrubber | Jumps to the dragged date. Pauses if playing. |
| Switch sub-mode back to Point | Diff / animation state is discarded. Canvas reverts to single-as-of behavior with `paintNode` (not `paintNodeDiff`). As-of date defaults to the "To" date if in Diff mode (preserving where the operator was), or the last animated frame if in Animate mode. |
| Switch to a different lens entirely | Diff / animation state is discarded. Sub-mode resets to `Point` when Temporal lens is re-entered. |
| Node click in Diff mode | Standard `selectNode(n.id)` — `KGDetailsPanel` opens with the node's facts at the "To" snapshot. Panel header shows a `Badge` for the change state: `added` (green outline), `removed` (red outline), `changed` (amber outline), no badge for unchanged. |

**Animation data fetching:** each frame is one `fetchOverview({ asOf })` call. Prefetch up to 2 frames ahead.
Cache by `asOf` key (a `Map<string, KgGraphData>` in the hook ref) so frames don't re-fetch when scrubbing backward.
Memory: cap the frame cache at 20 entries (LRU evict oldest). This prevents unbounded memory growth across
long animation ranges.

**Diff computation approach:** client-side set diff over `node.id`. "Changed" = node present in both but
at least one of its `attributes` or the set of incident current edges differs between the two snapshots.
Rationale: keeps the diff logic frontend-only with no new backend endpoint (no Ástríðr delta required,
unlike KG-08's search endpoint).

## Cross-Repo Dependencies

**KG-10 (Saved Views):** Requires a new `savedKgViews` Convex table (CodePulse-side only). No Ástríðr delta.
Share tokens are Convex document IDs (or a short random string stored alongside). Fully self-contained.

**KG-11 (Temporal Diff):** Client-side diff computation over existing `fetchOverview({ asOf })` calls —
no new Ástríðr endpoint required. The diff logic is pure TypeScript over two fetched snapshots.

**KG-11 (Animation):** Requires `fetchSnapshotDates()` — a new function in `src/lib/kgApi.ts` that calls
Ástríðr's existing `/api/kg` endpoint (or a new `/api/kg/snapshots` endpoint) to get the list of available
as-of dates. If this endpoint does not exist on the current Ástríðr build, the animation sub-mode degrades
gracefully to the empty state: `"No snapshot history available."` The Point and Diff sub-modes are unaffected.
Planner should gate `fetchSnapshotDates()` with a 404/501 graceful-degrade, same pattern as KG-08 search.
