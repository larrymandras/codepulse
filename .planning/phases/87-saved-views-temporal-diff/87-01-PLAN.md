<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser → Convex mutation | Operator-supplied view `name` and the client-generated `shareToken` cross into persisted storage |
| shared-link recipient → Convex query | An attacker-controlled `?view=<token>` value reaches `getByShareToken` |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-87-01 | Denial of Service | `save` mutation `name` arg | mitigate | Reject empty (trim length < 1) and over-long (> 100 chars) names in the handler before insert (ASVS V5). |
| T-87-02 | Tampering | `getByShareToken` `shareToken` arg | mitigate | Lookup is an exact index match on `by_shareToken`; a malformed/non-matching token returns null (no reflected content, no error leak). No string interpolation into a query. |
| T-87-03 | Information Disclosure | global-scope view enumeration via `list` | accept | D-02 intentional: single-operator/team deployment, all views are shared by design. No per-user scoping. |
| T-87-04 | Spoofing | client-generated `shareToken` (`crypto.randomUUID()`) | accept | Token is a non-secret opaque 122-bit-random identifier, not a credential; unguessable enough to gate casual link-sharing (D-03). Not a high-severity control. |
</threat_model>
