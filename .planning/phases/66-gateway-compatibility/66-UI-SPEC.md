## Scope of UI Changes

This phase is backend-heavy. The **only** UI surface that changes is `ProviderHealthPanel`. All other UI changes are behind Convex queries (schema extension, new indexes, provider registry) with no visible interaction surface.

**Components modified:**
- `src/components/ProviderHealthPanel.tsx` — dynamic provider list replacing hardcoded array; new fields: availability dot, auth status, billing type, quota bar
- `src/hooks/useProviderHealth.ts` — consume updated `api.providerHealth.latest` that returns all 4 gateway providers

**Components created:**
- None (extend existing `ProviderCard` within `ProviderHealthPanel.tsx`)

**Backend modules (no UI surface):**
- `convex/lib/providers.ts` — provider registry (no UI)
- `convex/otelLogs.ts`, `convex/otelMetrics.ts` — fallback fix (no UI)
- `convex/schema.ts` — field extensions (no UI)
- `astridr-repo: astridr/tools/cli_gateway.py` — telemetry emission (no UI)

## ProviderCard Visual Contract

### Layout

```
┌─────────────────────────────────────────┐
│ ● provider-name          [SUBSCRIPTION] │  ← header row
│ ─────────────────────────────────────── │
│ Availability   ● ONLINE                 │  ← field rows
│ Auth           ● AUTHENTICATED          │
│ Billing        subscription             │
│ Quota          ▓▓▓▓▓▓▓▓░░  99.5%       │  ← quota bar
│ Latency EMA    0.8s                     │
│ Success        98%                      │
└─────────────────────────────────────────┘
```

### Status Dot Color Rules

| State | Dot Color |
|-------|-----------|
| available: true, authenticated: true | `--status-ok` (#22c55e) |
| available: true, authenticated: false | `--status-warn` (#eab308) |
| available: false | `--status-error` (#ef4444) |
| No data in Convex | `bg-gray-600` (muted, no dot pulse) |

### Billing Type Badge

Use existing `StatusBadge` component with inline label mapping:

| billingType value | Display label | Semantic |
|-------------------|---------------|----------|
| `"subscription"` | SUBSCRIPTION | `idle` (muted) |
| `"api"` | API-BILLED | `warn` |
| `null` / missing | omit badge | — |

### Quota Bar

- Full-width progress bar inside card, height 4px (`h-1`)
- Track: `bg-gray-700/50`
- Fill color:
  - `quotaRemaining >= 0.20`: `--chart-bar-accent` (`#10b981`)
  - `quotaRemaining < 0.20`: `--status-warn` (`#eab308`)
  - `quotaRemaining < 0.05`: `--status-error` (`#ef4444`)
- Percentage value rendered in `text-xs font-mono tabular-nums` next to bar
- If `quotaRemaining` is null/missing: omit the bar entirely, do not show 0%

### Grid Layout

4 providers → `grid-cols-1 sm:grid-cols-2 lg:grid-cols-4` (was 3-col for 3 providers).

Source: CONTEXT.md gateway data contract (GET /health response shape), Phase 66 scope in ROADMAP.md.

## Loading and Empty States

| State | What to Show |
|-------|--------------|
| Convex query loading | Skeleton — 4 placeholder cards with `animate-pulse` gray blocks, same grid layout |
| Provider returns no Convex row yet | Show card shell with provider name + gray dot + "No data" in `text-xs text-muted-foreground` |
| All providers have no data | Panel renders with empty cards, not hidden |
| `available: false` | Show card normally with red dot, do not collapse or hide |

## Interaction Contract

- **No interactions** — `ProviderHealthPanel` is a read-only status panel. No click handlers, no modals, no forms.
- Panel updates automatically via Convex `useQuery` subscription when backend data changes.
- `useProviderHealth` hook throttle: preserve existing 5000ms throttle (`useThrottledQuery`).
- No new user-initiated actions introduced by Phase 66 UI work.

## Provider Registry Impact on UI

The new `convex/lib/providers.ts` registry replaces the hardcoded `["anthropic_direct", "openrouter", "ollama"]` array. The updated `api.providerHealth.latest` query returns a dynamic object keyed by all registered provider names.

**Four providers Phase 66 introduces:**

| Registry Key | Display Name | Billing |
|-------------|--------------|---------|
| `claude-cli` | Claude CLI | subscription |
| `codex` | Codex CLI | subscription |
| `antigravity` | Antigravity CLI | subscription |
| `claude-sdk` | Claude SDK | api |

Display name rendering: convert `"claude-cli"` → `"Claude CLI"` via a static display-name map in the component (not dynamic). If a provider key is not in the display map, render the raw key as-is.
