<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| daemon → Convex | Outbound bearer-authed fetch (FORGE_INGEST_API_KEY). Key lives only in the daemon process env. |
| Convex command → local execution | A claimed command becomes a local Forge action (launch/stop). The daemon trusts Convex only after its own bearer auth succeeded server-side. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-07 | Information Disclosure | FORGE_INGEST_API_KEY in logs | mitigate | Mirror emitter discipline — never interpolate the key into console output; only into the Authorization header. |
| T-80-08 | Denial of Service | poll loop throwing / tight retry | mitigate | Fire-and-forget `.catch(()=>null)` on both claim and ack; fixed 7s interval (no exponential hammer); execute() never rethrows. |
| T-80-09 | Tampering | double-execution across two daemon instances on one host | mitigate (server-side) | The atomic claim in 80-01 (claimAndUpsertHost queued→executing) guarantees a command is handed to exactly one poll. Two daemons on one hostId is operationally discouraged. |
| T-80-10 | Elevation of Privilege | dangerous-mode / workspace-create via command | accept (structurally absent) | The only command types are launch + stop; launchPayload has no dangerous flag (stripped in 80-01) and there is no create-workspace command (D-06/D-07). |
| T-80-SC | Tampering | npm installs in forge repo | mitigate | No new packages — built-in Node fetch only (RESEARCH §Standard Stack). |
</threat_model>
