## Component Inventory (net-new affordances)

### 1. "Related across graphs" link section

**Slot:** Bottom of each surface's existing detail panel, separated from the neighbors/facts block by a `<Separator />` with `mt-6 mb-4` spacing.

**Renders only when:** at least one cross-graph match is confirmed by eager resolution (D-04). If zero matches resolve, the entire section is absent — no heading, no empty row.

**Structure:**

```
[Section label: "RELATED ACROSS GRAPHS" — 10px mono uppercase muted]
[SectionErrorBoundary wrapping the list]
  [Link row × N]
    [ArrowRight icon 12px emerald] [prefix text in muted] [entity name in foreground semibold] [ExternalLink icon 12px muted]
```

**Link row anatomy:**
- Height: 28px (`py-1.5 px-2`)
- Layout: `flex items-center gap-2`
- Prefix text: `text-xs text-muted-foreground` — describes the relationship
  - Tool → agent: "Owning agent:"
  - Agent → KG entities: "3 KG entities" (count shown when > 1; singular: "1 KG entity")
- Entity name: `text-xs font-semibold text-foreground`
- Arrow icon: `ArrowRight h-3 w-3 text-primary` (Lucide)
- External indicator: `ExternalLink h-3 w-3 text-muted-foreground/50` (Lucide) — end of row
- Hover state: `hover:bg-primary/5 rounded cursor-pointer transition-colors duration-200`
- Full row is a `<button>` (accessible, keyboard-navigable)
- `SectionErrorBoundary` wraps the full section so a match-resolution error cannot take down the panel

**Agent → KG link behavior:** When N > 1 KG entities match, the link row navigates to `/knowledge-graph?focus=<firstEntityName>&lens=entity&hops=1` and the label reads "N KG entities →". The first entity by alphabetical sort after normalization is the jump target. When N = 1, link uses that entity's name directly.

**Surfaces it appears on:**
- `CodeVaultGraph.tsx` detail panel (agent node selected): shows "N KG entities →"
- `KnowledgeGraph.tsx` KGDetailsPanel (not applicable for forward links — this surface is a destination, not a source for this phase's two forward links)
- `ToolGalaxy.tsx` detail panel (tool node selected): shows "Owning agent: {label} →"

> Source: CONTEXT.md D-03 (two forward links only), D-04 (eager match, silently absent), D-05 (section placement).

### 2. Return chip

**Slot:** Top-left of the destination surface's detail panel header area, rendered only when `from` param is present in the URL.

**Structure:**

```
[ChevronLeft icon 12px muted] [copy: "Back to {Surface}"]
```

**Anatomy:**
- Rendered as `<button>` with `ghost` variant styling (no background, no border by default)
- `text-xs font-mono text-muted-foreground hover:text-foreground`
- Left accent: `border-l-2 border-primary/40 pl-2` — the single emerald touch that signals this is navigation
- Icon: `ChevronLeft h-3 w-3` (Lucide) — 2px gap to copy
- Height: 28px (`py-1.5`)
- Position: inline in the panel header row, left-aligned, before the panel title label
- Hover: `hover:border-primary/70 transition-colors duration-200`
- Copy pattern: "Back to Tool Galaxy" / "Back to Code/Vault Graph" / "Back to KG Explorer"
- On click: `react-router navigate(fromUrl)` where `fromUrl` is the decoded `from` param — restores originating surface with its prior selection encoded in that URL

**Visibility rule:** Return chip renders ONLY when `?from=...` is present in the current URL. It is absent on direct nav or bookmark load without `from`.

> Source: CONTEXT.md D-06 (return chip + origin encoded in URL), SC#4; `CodeVaultGraph.tsx:418-428` (panel header pattern).

### 3. Arrival states (inbound focus param handling)

All three graph pages read their focus param on mount. The states a surface can be in on arrival:

| State | Trigger | Visual |
|-------|---------|--------|
| **Loading** | Query still resolving (node not yet in memory) | `<Skeleton>` placeholder replacing the focused node's natural position; canvas shows spinner overlay using existing `animate-pulse text-primary/70 font-mono text-sm` pattern from `CodeVaultGraph.tsx:541` |
| **Focused** | Node found; `centerAt` + `zoomToFit` executed | Node centered and zoomed; detail panel auto-opens with that node selected; return chip visible (if `from` present) |
| **Not found on arrival** | Node absent in loaded data (data changed since link was built) | Silent fallback to default view (no focus, no error shown); return chip still visible if `from` present; no toast, no error banner |

**Loading implementation:** Wrap the canvas area in `SectionErrorBoundary`. While `useFocusParam` is resolving (query `undefined`), show a skeleton in the detail panel slot: `<Skeleton className="h-full w-full rounded-[var(--radius)]" />`. This reuses the existing `skeleton.tsx` shadcn primitive.

**Not-found fallback:** If the focused node is confirmed absent after the query resolves (not `undefined`, but node missing from the dataset), silently clear the focus target and render the surface in its default state (no node selected). Do NOT display an error state. This is the SC#3 "silently absent" rule applied at the destination.

> Source: CONTEXT.md Claude's Discretion (loading/empty state on arrival); CONTEXT.md D-04 (SC#3); REQUIREMENTS.md GH-04.

## Interaction Contract

### Navigation behavior

- **Outbound jump:** `react-router navigate(buildFocusUrl(target))` — replaces or pushes history entry
  - URL carries `?focus=<id>` (code/vault, tool-galaxy) or `?focus=<entityName>&lens=entity&hops=1` (KG)
  - URL also carries `?from=<encodedOriginUrl>` where `encodedOriginUrl` is the originating surface's URL with its current state (selected node + active filters) serialized into it
- **Return:** clicking the return chip calls `react-router navigate(decodedFromUrl)` — history push, not replace, so browser back also works
- **Focus effect on arrival:** `ForceGraphHandle.centerAt(node.x, node.y, 800)` followed by `ForceGraphHandle.zoom(3, 800)` — smooth 800ms animation matching the existing `ToolGalaxy.tsx:267` pattern

### Match resolution

- Normalization: strip namespace prefix (`graphify:<repo>:` / `vault:`), lowercase, trim whitespace — yielding a normalized key
- Match condition: exact string equality on normalized keys — no fuzzy, no substring
- Resolution is eager: computed over the already-loaded graph dataset; the affordance renders only after a confirmed match
- A non-resolving match produces no affordance (no placeholder, no "no match" row)

### Accessibility

- All link rows and the return chip are `<button>` elements — keyboard-navigable and screen-reader-announced
- Section heading uses `aria-label="Related across graphs navigation links"` on the containing section
- Return chip `aria-label="Return to {Surface}"` (full surface name)
- `SectionErrorBoundary` ensures errors in match resolution are contained and do not propagate to the panel
