## Interaction Contracts

### CommandCatalogPanel — Connection State

The panel content is driven by the WebSocket catalog payload received on connect. Three visual states are required:

**State 1 — Connecting (no catalog received yet)**
- Show a muted spinner or shimmer placeholder inside the panel
- Spinner: single `Loader2` Lucide icon (16px) with `animate-spin` class, centered, `text-muted-foreground`
- No text below spinner

**State 2 — Connected, catalog received**
- Render the full grouped/expandable list of commands
- Group commands by `category` field
- Within each group, use EntityRow pattern: leading category dot (4px circle), command name in `font-mono text-sm`, description truncated to one line

**State 3 — Disconnected or catalog load failed**
- Panel shows error state copy (see Copywriting Contract)
- No spinner — static message only

### CommandCatalogPanel — Expand/Collapse

- Each command row is clickable. Click expands an inline detail panel below the row.
- Expand/collapse uses `ChevronUp` / `ChevronDown` Lucide icons (12px), `text-muted-foreground`, aligned trailing.
- Expanded detail shows: description (full, not truncated), parameters table (name, type, required), source manifest.
- Parameters table columns: Name (`font-mono text-xs`), Type (`font-mono text-xs text-muted-foreground`), Required (boolean — show `text-status-ok` checkmark or `text-muted-foreground` dash).
- Only one command expanded at a time (accordion pattern — expand collapses the previous).
- Transition: none (Phase 1 pattern avoids JS animations except activity feeds).

### CommandCatalogPanel — Category Filter

- Category filter pills render above the command list (same as DiscoveredToolsTable pattern).
- "All (N)" pill is always first.
- Active pill: `bg-(--primary) text-(--primary-foreground)`.
- Inactive pill: `bg-(--muted) text-(--muted-foreground) hover:bg-(--accent)`.
- Pills use `rounded-sm` (2px) — the status badge radius exception. Not `rounded-full`.
- Click a category pill to filter. Click "All" or the active pill again to reset.

### CommandCatalogPanel — Search Integration

- The existing Capabilities page search input (`Search tools, skills, servers...` placeholder) is updated to include commands in its scope.
- Updated placeholder: `Search tools, skills, commands...`
- The `filter` prop passed to `CommandCatalogPanel` is the same lowercased trimmed string passed to all other panels.
- Search matches against: `name`, `description`, `category`, `source`.

### Capabilities Page — MetricCard Row Update

- The Commands MetricCard is appended as the 7th card in the existing MetricCard row.
- Grid class: `grid grid-cols-2 md:grid-cols-4 lg:grid-cols-7 gap-3` (updated from `lg:grid-cols-6`).
- Value: count of commands in received catalog. Shows `0` during connecting state, shows live count once catalog is received.
- No `trend` prop on Commands MetricCard — catalog count does not have a historical baseline.

Source: Phase 1 UI-SPEC (interaction patterns, EntityRow, StatusBadge); objective description; DiscoveredToolsTable as structural reference

## Anti-Patterns (Executor Reference)

All Phase 1 anti-patterns apply. Additional Phase 58-specific prohibitions:

| Prohibited | Use Instead |
|-----------|-------------|
| `rounded-xl`, `rounded-lg` on CommandCatalogPanel or any container | No radius — `--radius: 0rem` globally |
| `rounded-full` on category filter pills | `rounded-sm` (2px) — same as StatusBadge exception |
| Hardcoded `gray-800`, `gray-700`, `gray-500` color classes | `bg-(--muted)`, `text-(--muted-foreground)`, `border-(--border)` |
| Fetching command catalog via Convex query (polling) | Receive catalog over WebSocket on connect event — do not persist to Convex |
| Showing stale catalog while reconnecting | Clear catalog state and show connecting spinner on disconnect |
| Expanding multiple command rows simultaneously | Accordion pattern — one row expanded at a time |
| `purple-400` or other saturated Tailwind colors on category labels | Use `text-muted-foreground` for default; `text-(--primary)` for active only |
| `font-bold` (weight 700) anywhere | `font-semibold` (600) is maximum weight |
| `bg-[var(--token)]` (Tailwind v3 syntax) | `bg-(--token)` (Tailwind v4 syntax) |
| Inline SVG search icon (current codebase pattern) | Replace with `Search` Lucide icon (16px) |
| Unicode triangle arrows (`▲` / `▼`) for expand/collapse | `ChevronUp` / `ChevronDown` Lucide icons (12px) |

Source: Phase 1 UI-SPEC Anti-Patterns; audit of existing `Capabilities.tsx` and `DiscoveredToolsTable.tsx`
