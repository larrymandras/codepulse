# Phase 90 — UI Design Contract

> Visual and interaction contract for the War Room completion (ROOM-01..04): real agent identity,
> bounded room listing, genuine LiveKit operator join, and transcript robustness with deep-link
> support. This phase wires and hardens an existing ~70–75%-built surface; no net-new page is
> created. Four new interaction surfaces (connection-state UI, muted-by-default mic affordance,
> generated-avatar treatment for unknown participants, read-only "room ended" deep-link state)
> require explicit contracts.
>
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding (gap-1, px-1) |
| sm | 8px | Compact element spacing (gap-2, py-2, px-3) |

| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing |

Exceptions:
- Voice control touch targets (mute toggle, leave button): **44px minimum** (`h-11 w-11` =
  44px) — existing `VoiceControlBar` pattern, WCAG 2.5.5 compliant. Do not reduce.
- Agent voice card minimum height: **160px** (`min-h-[160px]`) — existing `AgentVoiceCard`
  pattern. Do not reduce.
- Room sidebar width: **256px** (`w-64`) — existing `WarRoom` split-panel layout. Fixed, not
  responsive in this phase.

## Color

All tokens are CSS custom properties resolved per the active theme. The deployed default skin is
Electric Cyan (`.dark` block, `src/index.css:127`). All new War Room surfaces MUST use token
references (`var(--primary)`, `var(--status-warn)`, etc.), never hardcoded hex values, so the
surface works correctly across all Phase 89 themes.

| Role | Token | Electric Cyan default | Usage |
|------|-------|-----------------------|-------|
| Dominant (60%) | `var(--background)` | `#040405` | Page background, fullscreen surfaces |
| Secondary (30%) | `var(--card)` / `var(--muted)` | `#0a0a0c` / `#1e1e24` | GlassPanel cards, sidebar fills, borders |
| Accent (10%) | `var(--primary)` | `#06b6d4` (Electric Cyan) | See accent reserved-for list below |
| Speaking ring | `var(--speaking-ring)` | `#06b6d4` | Speaking animation ring only |
| Destructive | `var(--destructive)` | `oklch(0.704 0.191 22.216)` | Leave button, failed-connection indicator |

Connection-state semantic colors:

| State | Token | Electric Cyan default |
|-------|-------|-----------------------|
| Connected | `var(--status-ok)` | `#06b6d4` |
| Reconnecting | `var(--status-warn)` | `#eab308` |
| Failed | `var(--status-error)` | `#ef4444` |
| Muted (initial join highlight) | `var(--status-warn)` | `#eab308` |
| Secondary text | `var(--muted-foreground)` | `#94a3b8` |

**Accent reserved for (explicit list):**
- "Join Voice" button (primary variant, enabled state only)
- Speaking ring animation overlay on `AgentVoiceCard`
- Active room ping indicator (pulsing dot in `RoomListItem`)
- Selected room left border in sidebar (`border-l-2 border-(--primary)`)
- Connection-connected status dot

**Not accent:**
- "New Room" launch button uses `bg-red-600` (existing — deliberate urgency signal, not a
  navigation accent; do not convert)
- Disabled "Join Voice" (room ended) uses `bg-muted` / `text-muted-foreground` (standard
  disabled styling)

## Phase 90 Interaction Surfaces

These are the four net-new visual/interaction contracts. Existing War Room components are wired,
not redesigned.

---

### Surface A — Connection-State UI (ROOM-03)

Renders inline inside `VoiceControlBar` when `isJoined=true`. Driven by a `connectionState`
prop: `"connecting" | "connected" | "reconnecting" | "failed"`.

Placement: left side of the control bar (`flex-1` left region), between the left panel edge and
the mute toggle. Mute + leave buttons remain right-anchored.

| State | Icon | Text | Color | Controls |
|-------|------|------|-------|----------|
| `connecting` | `Loader2` + `animate-spin` h-4 w-4 | "Connecting to voice…" | `text-muted-foreground` | Mute + Leave disabled (`opacity-50 pointer-events-none`) |
| `connected` | w-2 h-2 filled dot (`rounded-full bg-(--status-ok)`) | none | `var(--status-ok)` | All enabled |
| `reconnecting` | w-2 h-2 filled dot + `animate-pulse` | "Reconnecting…" | `text-(--status-warn)` dot + `text-muted-foreground` text | All enabled (user can leave) |
| `failed` | `AlertCircle` h-4 w-4 | "Connection failed — check your network and try again" + "Retry" ghost button | `text-(--status-error)` | Retry resets `isJoined → false`; all controls disabled |

`prefers-reduced-motion`: replace `animate-spin` with static `Loader2`; replace `animate-pulse`
dot with static dot.

---

### Surface B — Muted-by-Default Mic Affordance (ROOM-03)

Join happens with `isMuted=true` (D-03). Three visual sub-states:

**Pre-join:**
- `VoiceControlBar` shows "Join Voice" button (primary variant, full-width centered)
- Immediately below: "You'll join muted" in `text-xs text-muted-foreground` centered
- Icon: none on the sub-label (copy alone is sufficient)

**Post-join, first 5 seconds (initial-muted highlight window):**
- Mute toggle button: `MicOff` icon in `text-(--status-warn)` amber tint
- Tooltip on hover: "Click to unmute"
- Purpose: draw attention so operator knows they are muted without confusion
- Timer: `useState<boolean>(true)` initialized on join, auto-set to `false` after 5 000 ms

**Post-join, after 5 s or after first mute-toggle interaction:**
- `isMuted=true`: `MicOff` in default foreground color (operator consciously muted)
- `isMuted=false`: `Mic` in `text-(--status-ok)` tint (live/active signal)

`prefers-reduced-motion`: no animation changes needed; color states are static.

---

### Surface C — Generated Avatar for Unknown Participants (ROOM-01)

For a `participantId` that does not resolve to a `useRosterAgents()` agent:

- `name` prop: `"Agent #" + participantId.slice(-4)` — never the raw participantId string
- `avatar` prop: `{ name: participantId }` — `AgentAvatar.getColor(participantId)` already
  implements deterministic color hashing across 8 brand colors (indigo/violet/pink/orange/
  green/cyan/amber/red). Initials rendered: `participantId[0].toUpperCase()`. No code change
  to `AgentAvatar` is needed; only the `AgentVoiceCard` call site changes.
- `roleBadge` prop: `"Agent"` (fallback)
- No visual difference from known agents in card layout; the deterministic color ensures each
  unknown participant gets a visually distinct avatar, never a generic gray placeholder

For the **operator's own card** (when joined):
- `name`: `"You"`
- `avatar`: `{ name: "You", color: "var(--primary)" }` (accent-colored circle with "Y" initial)
- `roleBadge`: `"Operator"`

---

### Surface D — Room Ended Deep-Link State (ROOM-04)

`/war-room/:roomId` for a closed or non-existent room. Not a redirect; not a 404.
Renders the full archive view.

**Room header** (unchanged layout):
- `StatusBadge status="ended"` renders existing muted "ENDED" badge (legacyMap already handles
  `ended → { semantic: "idle", label: "ENDED" }`)
- Participant count and room name displayed normally

**Room Ended notice bar** (new element, inserted between room header and agent cards grid):
- Full-width horizontal bar: `py-2 px-4 flex items-center gap-2`
- Background: `bg-(--status-warn)/10 border-b border-(--status-warn)/20`
- Icon: `AlertCircle` h-4 w-4 in `text-(--status-warn)`
- Copy: "Room Ended" in `text-sm font-medium` + " — This session has ended. The transcript is
  preserved below." in `text-sm text-muted-foreground`

**Agent cards grid:**
- Cards rendered at `opacity-50`
- `isSpeaking` always `false`; speaking ring never shown
- Cards are non-interactive (pointer-events-none or no onClick)

**VoiceControlBar:**
- Shows disabled "Join Voice" button: `disabled` prop, `cursor-not-allowed`, `opacity-50`
- Wraps button in Radix `Tooltip` with content "This room has ended"
- "You'll join muted" sub-label is absent (join is disabled)
- No connection-state UI rendered (not joined)

**Transcript:**
- `TranscriptPanel` with `live={false}` (no auto-scroll on new chunks)
- On page load: auto-select the room, auto-scroll transcript to newest entry (bottom)

---

### Surface E — Bounded Room List + "Show More" (ROOM-02)

- Active rooms: always fully listed (expected < 20 simultaneous active rooms)
- Closed rooms: initial query returns first 20, ordered by most-recently-closed descending
- If `hasMore: true` returned from Convex query:
  - After last `RoomListItem` in closed section: `Button` variant="ghost" size="sm" with
    `text-muted-foreground w-full justify-center py-2` and label "Show older rooms"
  - On click: increment page / cursor, re-query; replace button with `Loader2 animate-spin h-4
    w-4 mx-auto` + "Loading…" in `text-xs text-muted-foreground`
  - On load complete: append new items and restore button (if still more) or remove it
- If `hasMore: false` or unset: "Show older rooms" button is absent (no empty sentinel)
