## Decisions Made
- **byday day-code convention**: the plan interface only specifies `byday?: string[]` without a format. Adopted iCal's 2-letter codes (`MO`/`TU`/`WE`/`TH`/`FR`/`SA`/`SU`) as the standard "rrule-lite" convention (D-05 names the shape "rrule-lite"), documented in `computeNextDueAt`'s JSDoc. Downstream plans (03 Ástríðr tool, 06 UI) should use these codes.
- **Task 1 skipped a vitest RED gate**: the plan's own `<verify>` block for Task 1 specifies `npx convex codegen && npx tsc --noEmit`, not a test file — a schema-table addition has no failing-behavior to assert beforehand. Tasks 2 and 3 (the actual logic) both followed a full RED (failing test, confirmed via `npx vitest run`) → GREEN (implementation, confirmed passing) cycle per the plan's `tdd="true"` marking.
- **Handler ctx typed `{ db: RemindersDb } | any`**: Convex's generated `GenericMutationCtx`/`GenericQueryCtx` types are not structurally assignable to a hand-written `RemindersDb` interface (the real `ctx.db.insert` is generic over `TableNamesInDataModel`, not a plain `string`), and `QueryCtx.db` is read-only (no `insert`/`patch`/`delete`) while `MutationCtx.db` is read-write — a single shared interface can't satisfy both without the `| any` escape hatch. Followed the existing `evalScores.ts` `storeEvalScoreHandler` precedent exactly rather than inventing a new pattern.

## Issues Encountered

- **TS2345 generic-function mismatch** after widening handler signatures: `ctx.db.insert`'s real generic signature (`<TableName extends TableNamesInDataModel>(table: TableName, ...) => Promise<Id<TableName>>`) is not assignable to a plain `(table: string, doc) => Promise<any>` interface member, and `QueryCtx.db` lacks write methods entirely. Resolved by typing every handler's `ctx` parameter as `{ db: RemindersDb } | any` (Rule 1 — the evalScores.ts precedent already establishes this exact fix in this codebase) rather than maintaining two separate reader/writer interfaces.
- **Unused `Id` import**: added `import type { Id } from "./_generated/dataModel"` anticipating typed ids in the `RemindersDb` interface, then simplified to `any` for id parameters — removed the now-dead import before the final GREEN commit (Rule 1, caught before commit, no separate fix commit needed).
