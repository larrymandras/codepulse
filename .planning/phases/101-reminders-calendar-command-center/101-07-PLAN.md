<objective>
Close the one diagnosed gap from UAT test 8 (severity: major, live-reproduced): reminders with no due date disappear from the list whenever a calendar day is selected.

Root cause (fully diagnosed, file:line in 101-UAT.md Gaps): `dayFiltered` in `src/components/reminders/ReminderList.tsx:472-478` requires `due !== undefined && startOfDaySeconds(due) === selectedDay`, which excludes ALL undated rows the moment `selectedDay !== null`. Undated rows otherwise live under Upcoming (`due === undefined`, ~L493-495), which is why they "show up" with no day filter but vanish on any day click.

Fix (minimal, consistent with the existing Overdue/Today/Upcoming/Done grouping UI): exempt `due === undefined` from `dayFiltered` so undated reminders remain visible — they continue to render under Upcoming — while dated rows still filter to the selected day. No new sub-group, no new UI, no CSS, no Convex change. Purely a client grouping/filter fix.

Purpose: reminders created via QuickAdd with an empty due field (the operator's exact repro) must not appear to "not persist" when a future day is clicked.
Output: a RED-first regression test + the one-line semantic fix.
</objective>
