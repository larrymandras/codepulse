## Voice-Mode State Machine (6 states)

This section is the primary design contract addition for this phase. The executor maps each state to the UI elements below.

### State definitions

| State | Trigger | Duration |
|-------|---------|----------|
| `idle` | Voice mode OFF (toggle not enabled) or wake word engine unstarted | Persistent until user enables toggle |
| `listening` | Wake word detected OR voice mode enabled + engine running (no command yet spoken) | Until utterance starts OR silence timeout (~30s) |
| `transcribing` | Speech recognition receiving audio after wake | While user is speaking |
| `processing` | STT final result sent via `chat.send`; awaiting first `run.text` event | Until first token arrives |
| `speaking` | `run.tts` audio playing back (Ástríðr response TTS) | While `<audio>` element is playing |
| `error-disabled` | ONNX model load failure OR `onnxruntime-web` init failure | Persistent; toggle remains disabled |

### Palette voice-mode layout (inside `CommandDialog`)

When voice mode is active (palette opened by wake word), the palette renders in **voice mode** — the standard command list is replaced by the voice interaction panel. The transition is not animated; the mode switch is instant because the palette opens fresh in voice mode.

**Voice panel structure (top to bottom within `CommandList`):**

```
┌────────────────────────────────────────────┐
│  [State badge]  [State label]    [X close] │  ← 36px row
├────────────────────────────────────────────┤
│                                            │
│  [Live transcript area]                    │  ← min 48px, grows
│   interim text (muted, italic)             │
│   final text (foreground, normal)          │
│                                            │
├────────────────────────────────────────────┤
│  [Streamed reply area]                     │  ← grows with content
│   Ástríðr: [streamed tokens...]            │
│                                            │
├────────────────────────────────────────────┤
│  [End-phrase hint]  [Waveform / spinner]   │  ← 28px row
└────────────────────────────────────────────┘
```

**State → visual rendering table:**

| State | State badge | Transcript area | Reply area | Waveform / indicator | Bottom hint |
|-------|-------------|-----------------|------------|---------------------|-------------|
| `listening` | Emerald/primary dot + "Listening" | Empty; placeholder: "Say a command…" (muted) | Empty | 3× `.eq-bar` in `text-primary`, animated | "Say 'stop' to exit" |
| `transcribing` | Amber/warn dot + "Hearing you" | Interim transcript in `text-muted-foreground italic`; final segments in `text-foreground` | Empty | 3× `.eq-bar` in `text-primary`, faster (0.4s cycle) | "Say 'stop' to exit" |
| `processing` | Primary dot + "Thinking" | Final transcript text (static, `text-foreground`) | Empty; `CommandEmpty`-style "..." spinner (`animate-pulse text-muted-foreground`) | Spinner (`Loader2` Lucide, `animate-spin h-4 w-4 text-primary`) | — |
| `speaking` | Primary dot + "Ástríðr speaking" | Final transcript (static, `text-muted-foreground text-xs`) | Streamed reply text at 14px/1.5lh; new tokens appended | `Volume2 h-4 w-4 text-primary animate-pulse` | "Say 'stop' to exit" |
| `error-disabled` | Red dot + "Voice unavailable" | "Wake-word model could not be loaded." | Error reason (12px muted): specific message from load failure | — | "Mic toggle disabled — see reason above" |

**State badge implementation:**
```tsx
// Status dot — 8px circle, color via Tailwind status class
<span className="w-2 h-2 rounded-full {dotClass}" />
// listening/speaking/processing: bg-primary animate-pulse shadow-[var(--glow-md)]
// transcribing: bg-yellow-400 (--status-warn)
// error-disabled: bg-red-500 (--status-error)
```

**Transcript area:**
- Container: `min-h-[48px] px-4 py-3 font-mono text-[10px] uppercase tracking-wide`
- Interim text: `text-muted-foreground italic` — not yet confirmed by STT
- Final text: `text-foreground not-italic` — confirmed by STT
- aria: `aria-live="polite"` on the transcript container (updates as interim text arrives; final text does not need to re-announce if interim was already announced)

**Reply stream area:**
- Container: `px-4 py-3 border-t border-border`
- Label row: `text-xs text-muted-foreground font-semibold font-mono uppercase tracking-widest mb-2` — "ÁSTRÍÐR"
- Streamed text: `text-sm text-foreground leading-relaxed` (14px / 1.5)
- aria: `aria-live="polite" aria-atomic="false"` — announces tokens as they stream in

**Close (X) button:**
- Position: top-right of the voice panel header row
- Icon: `X h-4 w-4` (Lucide)
- Action: exits voice mode + closes palette; clears transcript + reply
- aria-label: "Close voice mode"

## Top-Bar Mic Toggle + Listening Indicator

Positioned in `DashboardLayout.tsx` header, inserted immediately before `EStopButton` in the right-side control group (`flex items-center gap-1.5`).

### Mic toggle button (always visible when voice mode is supported)

| Property | Value |
|----------|-------|
| Size | 36px × 36px (`w-9 h-9`) |
| Shape | `rounded-md` (matches `ThemeSwitcher` and `CrtToggle` siblings) |
| Icon: OFF state | `Mic h-4 w-4 text-muted-foreground` |
| Icon: ON / listening | `MicVocal h-4 w-4 text-primary` |
| Icon: DISABLED state | `MicOff h-4 w-4 text-muted-foreground opacity-40` (toggle `disabled` attribute set) |
| Background: OFF | `bg-transparent hover:bg-accent/50 transition-colors` |
| Background: ON | `bg-primary/10 border border-primary/30 shadow-[var(--glow-xs)]` |
| Background: DISABLED | `bg-transparent opacity-40 cursor-not-allowed` |
| aria-label: OFF | "Enable voice mode" |
| aria-label: ON | "Disable voice mode" |
| aria-label: DISABLED | "Voice mode unavailable — model failed to load" |
| Tooltip (Radix `Tooltip`) | OFF: "Voice mode (Hey Ástríðr)" / ON: "Voice mode active" / DISABLED: error reason string |

The toggle is the only affordance. No separate "on/off" text label needed — it matches the visual density of adjacent icon buttons.

### Listening indicator pill (visible only when voice mode is ON)

Rendered immediately left of the mic toggle button, inside the same `flex` row. Hidden (not rendered) when voice mode is OFF or DISABLED.

| Property | Value |
|----------|-------|
| Shape | `px-3 py-1 rounded bg-primary/10 border border-primary/20 shadow-[var(--glow-xs)]` (4-point-grid padding) |
| Dot | `w-2 h-2 rounded-full bg-primary animate-pulse shadow-[var(--glow-md)]` |
| Text | `text-[10px] font-semibold font-mono tracking-widest text-primary uppercase` — "VOICE ACTIVE" |
| prefers-reduced-motion | dot `animate-pulse` removed; static dot at 60% opacity |
| aria-label | `aria-live="polite"` on a visually-hidden span: "Voice mode active, listening for Hey Ástríðr" |

Pattern source: based on the existing "Astridr Runtime Telemetry" pill at `DashboardLayout.tsx:651-656`, but with `py-1` instead of the existing pill's off-grid `py-1.5`.

### Error-disabled state (top bar)

When `error-disabled`:
- Mic toggle shows `MicOff` icon, `disabled` attribute, `opacity-40`, `cursor-not-allowed`
- No listening indicator pill rendered
- Tooltip shows the specific failure reason (e.g. "Wake-word model failed to load: network error" or "onnxruntime-web: WASM not supported in this browser")
- No toast on page load — tooltip-on-hover is the only disclosure (silent degradation; no interruption)

## State Transition Affordances

### Wake-word → palette open transition

- The palette opens via the existing `setPaletteOpen(true)` call in `DashboardLayout` (the same path as Cmd+K)
- No additional entry animation — `CommandDialog` already has Radix/shadcn entry animation
- The voice panel replaces the `CommandInput` + `CommandList` content; it does NOT animate in separately
- The palette title/header is NOT shown in voice mode (the state badge row serves as the header)

### State transitions within the palette (listening → transcribing → processing → speaking)

- Dot color transition: `transition-colors duration-200` on the state dot
- State label text: swap instantly (no fade) — screen reader should announce on state change via `aria-live="assertive"` on the state label span
- Transcript area: text appears character-by-character (streaming from STT); no transition needed
- Reply area: text appears token-by-token (streaming from WS); no transition needed
- Processing → speaking: reply area fades in via `opacity-0 → opacity-100 transition-opacity duration-300` when first token arrives

### Silence timeout (30s)

- At 25s: no visual warning (too distracting for a HUD-style interface)
- At 30s: palette closes, voice mode returns to `listening` state (not OFF), top-bar indicator remains
- No countdown timer shown — this matches the "Jarvis feel" where the system quietly resets

### Barge-in (deferred — planner to assess)

If barge-in ships in MVP: when user starts speaking during `speaking` state, audio playback pauses instantly (no visual transition); state shifts to `transcribing` without palette close. The state dot changes from primary to warn (`text-yellow-400`) immediately on mic activation. If barge-in deferred: `speaking` state ignores new mic audio; mic indicator dims to 30% opacity during TTS playback to signal "not listening right now."

## Accessibility Contract

### aria-live regions

| Region | Element | `aria-live` | `aria-atomic` | `aria-label` |
|--------|---------|-------------|---------------|--------------|
| Voice state label | `<span>` next to state dot | `assertive` | `true` | "Voice mode status" |
| Transcript (interim + final) | transcript container `<div>` | `polite` | `false` | "Live transcript" |
| Reply stream | reply container `<div>` | `polite` | `false` | "Ástríðr response" |
| Top-bar listening indicator | visually-hidden `<span>` inside pill | `polite` | `true` | n/a (text content announces state) |

### Focus management

- On wake-word detection: palette opens, focus lands on the voice panel's close button (NOT on the transcript — it is read-only)
- On palette close (X button, Escape, or "stop" end-phrase): focus returns to the element that had focus before the palette opened (`document.activeElement` saved on open)
- Tab order within voice panel: [State close button] → [Voice panel container (focusable for scroll)] — transcript and reply areas are read-only, not in tab order

### Keyboard

- `Escape` while palette is open in voice mode: exits voice mode + closes palette (same as X button)
- `Escape` while voice mode is ON but palette is closed: does NOT disable voice mode (Escape is already used for sidebar close; voice toggle needs deliberate mic-button click)

### `prefers-reduced-motion`

The project already has a global `prefers-reduced-motion` block at `src/index.css:376`. Voice UI must honor it:

- Listening indicator dot: `animate-pulse` disabled → static dot at full opacity
- State dot color transitions: reduced to instant (remove `transition-colors`)
- `.eq-bar-*` waveform animation: disabled → static bars at fixed height (4px each)
- Mic toggle `shadow-[var(--glow-xs)]` on hover: retained (it is a CSS property, not an animation)
- Reply stream fade-in (`transition-opacity`): disabled → instant opacity-100

Implementation: add `@media (prefers-reduced-motion: reduce)` overrides in `src/index.css` for the two new keyframes added this phase (`voice-listen-pulse` for the expanded eq-bar version if used, `voice-processing-spin` if a custom spinner is added). Standard `animate-spin` already respects the media query via Tailwind.
