<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Operator toggle → mic engine start | The only path that opens the always-on mic; gated on explicit, persisted operator consent |
| Persisted preference → reload | localStorage `codepulse-voice-mode` controls whether listening auto-resumes |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-92-14 | Information Disclosure | silent always-on mic | mitigate | OFF by default; `start()` gated on `voiceModeEnabled && status !== 'error-disabled'`; persistent ListeningIndicatorPill renders whenever listening is active — listening is never silent or hidden (VOX-04) |
| T-92-15 | Denial of Service / crash | ONNX/runtime load failure | mitigate | `useWakeWord` surfaces `error-disabled`; toggle disables with reason tooltip; engine never starts on failure — no crash, no orphaned mic (VOX-04) |
| T-92-16 | Repudiation | unclear listening state | mitigate | The pill + the toggle's ON visual (MicVocal + glow) make the listening state continuously observable; tooltip discloses the disabled reason on hover |
| T-92-17 | Tampering | localStorage preference | accept | A tampered `codepulse-voice-mode` value can only enable a feature the same user controls via the visible toggle; bounded to that browser profile; no privilege gained |
</threat_model>
