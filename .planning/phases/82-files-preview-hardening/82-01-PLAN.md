| Boundary | Description |

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-82-01 | Spoofing | POST /forge-file-ingest | mitigate | `validateForgeIngestAuth` (fail-closed, reuse verbatim) → 401 via `unauthorizedResponse()`; bearer is `FORGE_INGEST_API_KEY`, server-to-server only |
| T-82-02 | Tampering | request body shape | mitigate | type/hostId/forgeJobId/files validation → 400; `v.` validators on internalMutation args reject malformed entries |
| T-82-03 | DoS | oversized doc write | mitigate | textContent capped daemon-side ≤1,000,000 bytes (82-04); query/mutation never construct oversized values; PER_FILE_BYTE_CAP constant documents the ceiling |

| T-82-06 | Info Disclosure | base64 in doc | mitigate | httpAction sets `imageBase64: undefined` post-store; base64 transport string is never persisted (Pitfall 3) |
| T-82-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages this plan (82-RESEARCH Package Legitimacy Audit: zero installs); no `[ASSUMED]`/`[SUS]` packages |
