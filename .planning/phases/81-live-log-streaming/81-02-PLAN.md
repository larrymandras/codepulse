  <behavior>
    - TTL: chunks with `_creationTime < Date.now() - SEVEN_DAYS_MS` are deleted; newer chunks survive.
    - Per-job byte cap: after TTL, for a single (hostId,forgeJobId) whose surviving line bytes exceed the cap, oldest chunks (lowest seq / earliest _creationTime) are deleted first until total ≤ cap; newest chunks always survive.

    - Empty table / under-cap job: sweep is a no-op (deletes nothing).
