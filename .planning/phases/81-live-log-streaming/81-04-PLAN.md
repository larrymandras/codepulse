| T-6-KEYLEAK | Information disclosure | `cfg.apiKey` / Authorization header in makeLogSink | mitigate | Bearer sent only in the fetch header; NEVER referenced in any console/error/log string — error logging cites only status code + batch line count. | T-81-13 | Spoofing | Outbound POST authentication | mitigate | `Authorization: Bearer ${FORGE_INGEST_API_KEY}` (D-3 shared key); receiver rejects bad/no bearer with 401 (plan 01). |
| T-81-14 | Denial of service (self) | Log-flood from a runaway job | mitigate | Best-effort/no-retry/lossy-under-pressure delivery (Forge drops, never blocks); receiver-side retention cap (plan 02) bounds storage. |
| T-81-15 | Repudiation/ordering | Out-of-order / duplicate delivery | mitigate | Monotonic per-job `seq` (D-1) added here → CodePulse dedups on `(hostId,forgeJobId,seq)` and orders by seq. |
| T-81-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed — uses the Forge runtime's existing `fetch`. No `[ASSUMED]`/`[SUS]` packages → no install checkpoint required. |
