## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Forge daemon → CodePulse httpAction | Untrusted HTTP POST crosses the cloud boundary at `/forge-log-ingest` |
| httpAction → Convex DB | internalMutation write (no Clerk identity present) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-01 | Spoofing | `/forge-log-ingest` httpAction (unauthenticated by default) | mitigate | Reuse `validateForgeIngestAuth` + `FORGE_INGEST_API_KEY` (D-3); bad/no bearer → `unauthorizedResponse()` 401. Verified by Section-1 auth tests. |
| T-81-02 | Tampering | Request body / envelope | mitigate | Strict field validation (`type==="log"`, hostId, forgeJobId, `Array.isArray(lines)`, `seq != null`) → 400 on any miss. JSON parse guarded by try/catch. |
| T-81-03 | Information disclosure (T-6-KEYLEAK) | Bearer key in handler | mitigate | Never log `apiKey`/Authorization in `forgeLogIngest.ts`. Grep gate in acceptance criteria. `unauthorizedResponse()` omits CORS headers on 401. |
| T-81-04 | Elevation of privilege | DB write path | mitigate | `appendLogChunk` is `internalMutation` (not `mutation`) — reachable only via `ctx.runMutation` from the authed httpAction, never client-callable (81-SPEC §Locked design 3). |
| T-81-05 | Tampering (replay/dup) | Re-delivered chunks | mitigate | D-1 idempotency: `by_host_job_seq` `.unique()` check; existing (hostId,forgeJobId,seq) → no-op. Best-effort/no-retry delivery handled deterministically. |
| T-81-06 | Information disclosure | CORS exposure | mitigate | Reuse `getCorsHeaders` verbatim on all non-401 responses; OPTIONS preflight returns CORS-only 200. |
| T-81-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed in this plan (reuse-only). No `[ASSUMED]`/`[SUS]` packages → no install checkpoint required. |
</threat_model>
