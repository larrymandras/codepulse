<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Convex cloud → React UI | ForgePage subscribes to forgeJobs via useForge; untrusted agent-emitted strings flow into the rendered list/detail (rendered by 79-02 components). |
| Router → page | /forge is a client route inside the existing AuthGuard-wrapped DashboardLayout; no new auth surface, no new network endpoint. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-79-06 | Elevation of privilege (unauthenticated access to /forge) | App.tsx route | accept | /forge is nested under the same `<AuthGuard>` + `<DashboardLayout>` as every existing route (App.tsx:71-73); it inherits the app's existing auth gating. No new gate introduced or bypassed. Read-only query, no mutation. |
| T-79-07 | Tampering (XSS via rendered job data) | ForgePage → ForgeJobList/Detail | mitigate | Page passes adapted rows as typed props; all untrusted-string rendering is delegated to the 79-02 components which rely on React default-escaping (no dangerouslySetInnerHTML). Page itself renders only static chrome (title) + child components. |
| T-79-08 | Information disclosure (nav reveals a feature) | DashboardLayout nav entry | accept | The nav entry is a static label/route; it discloses nothing beyond the feature's existence to an already-authenticated operator. |
| T-79-SC | Tampering (supply chain) | npm installs | accept | No new packages — `Flame` is an existing export of the already-installed `lucide-react ^1.8.0`; react-router/convex already present. No legitimacy gate required. |

No high-severity threats: the route inherits existing AuthGuard auth; no new endpoint, mutation, or dependency; untrusted-string escaping is handled by React in the child components.
</threat_model>
