## ForgeMetadataPanel вҖ” Fields Layout (D-02)

Rendered as a two-column CSS grid (`grid-cols-[auto_1fr]` or `grid-cols-2`) inside a `GlassPanel` below the job header. Fields ordered: identity group first, then execution group, then workspace/model/capabilities.

| Group | Fields (display order) | Format |
|-------|------------------------|--------|
| Identity | agent, mode, status | `agent`: capitalize; `mode`: capitalize; `status`: `ForgeStatusBadge` component |
| Execution | pid, exitCode, startedAt, finishedAt | `pid`/`exitCode`: numeric or "вҖ”" if null; `startedAt`/`finishedAt`: ISO вҶ’ `toLocaleString()` (full date + time, local timezone); null вҶ’ "вҖ”" |
| Resources | workspaceId, artifactCount | `workspaceId`: truncated to 12 chars + "вҖҰ" if longer; `artifactCount`: numeric |
| Configuration | model, capabilities | `model`: string or "default" if null; `capabilities`: parse JSON string вҶ’ pretty-print as `key: value` lines (one line per capability key); font-mono for the value block |
| Audit | createdAt, updatedAt | ISO вҶ’ `toLocaleString()` |

**Field key typography:** `text-[10px] font-mono uppercase tracking-widest text-muted-foreground` (CodePulse operational chrome style per DashboardLayout.tsx:179).
**Field value typography:** `text-xs text-foreground` for scalars; `text-xs font-mono text-muted-foreground` for the capabilities block.
**Row padding:** `py-2` between rows. Section group dividers: a 1px `border-t border-border` with a `text-[10px] font-mono uppercase tracking-widest text-muted-foreground` group label above each group (matches SectionHeader convention, lighter weight).

## Interaction Contract

### Job Selection
- Click a job card вҶ’ set `selectedKey: { hostId, forgeJobId }` вҶ’ render that job's detail pane.
- Selected card: `bg-accent` background + emerald left border `border-l-2 border-primary` to indicate selection (CodePulse active-state convention).
- Hover (non-selected): `hover:bg-accent/50` (Forge convention, compatible with CodePulse tokens).
- `aria-selected={isSelected}` on the card button element.

### Loading State
- `ForgeJobList` loading: render 3 skeleton rows (matching Forge's pattern) using CodePulse `Skeleton` component (`animate-pulse rounded-md bg-accent`). Three rows, each with:
  - Row 1: `Skeleton h-4 w-2/3` (badge + agent name area)
  - Row 2: `Skeleton h-3 w-1/2` (prompt preview)
  - Row 3: `Skeleton h-3 w-1/3` (timestamp)
  - Padding: `p-3 rounded-md flex flex-col gap-1.5`
- `ForgeMetadataPanel` loading: not applicable вҖ” detail renders from the already-loaded `listJobs` row; no second round-trip, no second loading state.

### Relative Timestamps
Use **`relativeTime()` from `src/lib/formatters.ts`** (Claude's Discretion resolved). The CodePulse formatter takes epoch seconds; the Convex `createdAt`/`updatedAt` fields are Convex `_creationTime` (milliseconds) or ISO strings вҖ” the adapter converts to epoch seconds before calling. Forge's own `relativeTime()` takes ISO strings and has slightly different formatting ("s ago" vs "just now" for <60s). Use CodePulse's version for consistency; accept the minor formatting difference ("just now" instead of "5s ago").

Full datetime display (startedAt/finishedAt/createdAt/updatedAt in the metadata panel): `new Date(isoString).toLocaleString()` вҖ” full local date + time, no relative conversion, since exact timestamps matter for execution audit.

### Transitions
- Card hover/select: `transition-colors` (inherits CodePulse standard, no duration override).
- No entry animations on job cards (list is a data list, not a feature showcase). `GlassPanel` motion for the detail panel: standard `opacity 0вҶ’1, y 8вҶ’0, duration 0.2 easeOut` via `GlassPanel`'s existing `animate` prop.
