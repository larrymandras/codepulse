### Status Heartbeat Grid (D-02, D-03, D-04)

- Tile grid: `grid grid-cols-[repeat(auto-fill,minmax(80px,1fr))] gap-3`
- Tile size: minimum 80px × 80px, aspect-ratio 1/1
- Tile contents (top to bottom): state pulse ring around AgentAvatar (md=40px), agent name (12px label), active counter or "idle" caption
- Pulse ring: `animate-pulse` via Tailwind for active/recent states; no animation for waiting/idle
- Click → inline detail panel expands BELOW the full grid row containing the selected tile. Detail shows: last 5 heartbeats (EntityRow list), current task string, error count badge. Close on second click or clicking another tile.
- Idle timeout: 5 minutes (300 000ms) computed by `setInterval(1000)` comparing `Date.now()` vs last event timestamp. No server-side flag.

### Cron Calendar (D-05, D-06, D-07)

- 7-day view: today ± 3 days (Mon–Sun window preferred)
- Hour slots: 24 rows × 7 columns. Slot height: 48px. Column width: equal flex.
- Current-time indicator: `1px solid var(--sidebar-primary)` horizontal line at current hour offset within day column
- Next-up countdown: small label beneath calendar header showing "Next: {action} in {mm:ss}" for the soonest upcoming entry
- Toggle "Hide system crons": shadcn `Switch` component, right-aligned in calendar header row
- Multiple entries per slot: stacked vertically as colored badge chips (12px label, truncated to 8 chars). Show "+N" overflow badge if > 3 entries in one slot.
- Click slot → popover (shadcn Popover) showing last execution result, duration, errors. Popover preferred over inline expand for calendar (compact grid, no room for inline).

### Pipeline Flow Diagram (D-08, D-09, D-10)

- Container: `<div style={{ height: 400 }}>` — explicit pixel height mandatory
- Node layout: horizontal left→right chain, nodes equally spaced
- Node size: 120px wide × 60px tall
- Node contents: stage name (14px body), status indicator dot (8px circle), duration label if completed (12px mono)
- Edge style: `animated: true` on running edges, `style={{ stroke: 'var(--status-ok)' }}` on completed, `style={{ stroke: 'var(--status-error)' }}` on failed, `style={{ stroke: 'var(--border)' }}` on pending
- Dropdown selector: shadcn `Select` in diagram header, "Live" option as default + past executionIds from query
- Click node → inline detail panel BELOW diagram showing: step name, status badge, durationMs (mono), inputSize/outputSize (mono), error string if failed

## Interaction States

### Agent Tile States

| Trigger | Visual Change | Animation |
|---------|--------------|-----------|
| New `agent_status` WS event | Tile immediately re-renders with new state color | `live-update-flash` CSS animation (existing, src/index.css) |
| Tile idle timeout (5min) | Tile fades to gray idle state | CSS transition-colors 300ms |
| Tile click | Inline detail expands below grid | Expand: `animate-in slide-in-from-top-2 duration-200` |
| Second tile click / tile re-click | Detail collapses | `animate-out slide-out-to-top-2 duration-150` |
| Hover | `hover:brightness-110` on tile | 150ms CSS transition |

### Calendar Slot States

| Trigger | Visual Change |
|---------|--------------|
| Hover | `hover:brightness-110` on slot chips |
| Click occupied slot | shadcn Popover appears anchored to slot |
| Click empty slot | No action (no interaction) |
| Current time passes slot | Time indicator line moves (1-second interval) |

## Accessibility Contract

- All interactive tiles: `role="button"` + `tabIndex={0}` + `onKeyDown` Enter/Space handler
- Status color is never the only differentiator — each state also has a text label inside the tile
- Calendar slot popover: triggered by click and keyboard (Radix Popover handles keyboard natively)
- React Flow: `proOptions={{ hideAttribution: true }}`, `showInteractive={false}` on Controls (existing pattern from AgentTopology)
- All section error boundary fallbacks have visible text (not just icon)
- Icon-only buttons (if any in diagram controls): `aria-label` required
