## Spacing Scale

Declared values (multiples of 4, standard 8-point scale):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding (`py-1`), tight label spacing |
| sm | 8px | Compact element spacing, badge horizontal padding (`px-2`), badge gap from parent |
| md | 16px | Default element spacing, card internal padding |
| lg | 24px | Section padding, stat row gaps |
| xl | 32px | Layout gaps between major panels |
| 2xl | 48px | Major section breaks (e.g., between Analytics sections) |
| 3xl | 64px | Page-level top padding |

Exceptions:
- Tab bar hit targets: 44px min-height (accessibility)
- AnomalyBadge inline: `py-1` (4px vertical) / `px-2` (8px horizontal) — compact badge using standard scale values

## Typography

All sizes use Geist Variable. Mono strings (dates, cron times, session IDs) use JetBrains Mono.

| Role | Size | Weight | Line Height | Notes |
|------|------|--------|-------------|-------|
| Label / Caption | 12px (text-xs) | 400 regular | 1.5 | MetricCard labels, table column headers, badge text, muted supplementary info |
| Body / UI | 14px (text-sm) | 400 regular | 1.5 | List items, tab labels, form fields, briefing feed body text |
| Sub-heading | 16px (text-base) | 600 semibold | 1.4 | Section headers, stat titles, panel headings within a page |
| Metric value | 24px (text-2xl) | 600 semibold | 1.2 | MetricCard values (tabular-nums), StatCard values on Memory/Briefings pages |

Font weights used: 400 (regular) and 600 (semibold) only.

Source: `src/components/MetricCard.tsx` (`text-xs` label, `text-2xl font-semibold tabular-nums` value), `src/pages/Memory.tsx` — existing patterns confirmed.

## Color

All colors defined as oklch CSS variables in `src/index.css`. Phase 7 uses the established palette without additions.

| Role | Light Mode Value | Dark Mode Value | Usage |
|------|-----------------|-----------------|-------|
| Dominant (60%) — background | `oklch(1 0 0)` | `oklch(0.145 0 0)` | Page backgrounds, main surfaces |
| Secondary (30%) — surfaces | `oklch(1 0 0)` card / `oklch(0.985 0 0)` sidebar | `oklch(0.205 0 0)` card / `oklch(0.205 0 0)` sidebar | Cards, sidebar, nav, panel backgrounds |
| Accent (10%) — interactive | `oklch(0.205 0 0)` primary | `oklch(0.922 0 0)` primary | See reserved-for list below |
| Semantic: OK / green | `oklch(0.65 0.15 142)` `--status-ok` | `oklch(0.72 0.17 142)` | Budget within threshold, positive trends |
| Semantic: Warning / yellow | `oklch(0.75 0.12 85)` `--status-warn` | `oklch(0.80 0.14 85)` | 2σ anomaly badge, budget at 80–99% |
| Semantic: Error / red | `oklch(0.65 0.18 27)` `--status-error` | `oklch(0.72 0.20 27)` | 3σ anomaly badge, budget exceeded |
| Destructive | `oklch(0.577 0.245 27.325)` `--destructive` | `oklch(0.704 0.191 22.216)` | Destructive actions (delete config) |
| Muted foreground | `oklch(0.556 0 0)` | `oklch(0.708 0 0)` | `--muted-foreground`: secondary labels, captions |

**Accent reserved for:** active tab indicator (border-bottom), primary CTA buttons ("Generate Digest", "Save Config"), focused input rings, and the budget progress bar fill when status is OK.

**Anomaly badge color rules (D-11, from CONTEXT.md):**
- `--status-warn` (yellow): 2σ warning anomaly badge background at 20% opacity, text at full opacity
- `--status-error` (red): 3σ critical anomaly badge background at 20% opacity, text at full opacity
- Pattern: `bg-[--status-warn]/20 text-[--status-warn]` / `bg-[--status-error]/20 text-[--status-error]`

**Budget progress bar gradient:** green (`--status-ok`) → yellow (`--status-warn`) → red (`--status-error`) at 80% and 100% thresholds.

Source: `src/index.css` oklch palette, CONTEXT.md D-09/D-11, RESEARCH.md AnomalyBadge pattern.

## Interaction Patterns

### Cost Forecast Panel (Analytics page — D-03)
- Three stat boxes in a row: Projected Daily | Projected Weekly | Projected Monthly
- Budget progress bar below stats: fills left-to-right, color transitions green→yellow→red at 80%/100% of budget cap
- Trend sparkline (CSS flex bars using existing `FlexBarChart` pattern) showing last 7 days of actual spend
- Forecast updates reactively with Convex query (no manual refresh)
- If fewer than 3 days of data: show "Insufficient data for forecast" in place of projections

### Anomaly Badges (inline on MetricCards — D-11)
- Rendered as a small tag appended to the MetricCard value row (right-aligned)
- "WARN" label for 2σ, "ANOMALY" label for 3σ
- Padding: `py-1 px-2` (4px vertical, 8px horizontal — standard scale)
- Hover/focus shows shadcn Tooltip with: metric name, current value, expected range, z-score
- Clicking the badge navigates to the triggered alert in the Alerts page
- At most one badge per MetricCard (highest severity wins if multiple anomalies)

### Briefings Page (D-08 — reverse-chronological timeline feed)
- Two entry types visually distinct: daily digest (full-width row with date header) vs session briefing (indented row with session ID and agent name)
- Collapsed state: type pill + date + one-line summary + expand chevron
- Expanded state: full narrative text in `text-sm` with `font-mono` for session IDs, timestamps, and cost figures
- Date range filter: two `Input` fields (from / to) in a filter bar above the feed
- `LoadMoreButton` at bottom (cursor pagination — established Phase 5 pattern)
- Empty state: no digests generated yet (see Copywriting Contract)
- Generating state: spinner row with "Generating briefing…" while LLM action is in-flight

### Memory Quality Tab (Memory page — D-15)
- New "Quality" tab added to the existing tab row: Timeline | Tiers | Reflections | **Quality**
- Stats row (3 StatCards): Dedup Rate (%) | Stale Memories (count) | Contradictions (count)
- Below stats: three accordion sections — "Duplicate Flags", "Stale Memories", "Contradictions"
- Each accordion row uses EntityRow pattern (established Phase 1): leading icon, memory summary, age/flag date
- Empty accordion: "No issues detected" caption in `text-xs text-muted-foreground`

### Settings — Intelligence Config (new section)
- Section header: "INTELLIGENCE" in uppercase tracking-wide muted-foreground (established SectionHeader pattern)
- Budget Cap: labeled Input + currency prefix "$" + "Save Budget Settings" button inline
- LLM Provider: two stacked config rows (Primary, Backup) — each row: provider Select + model Input + API key Input (masked) + "Save Provider Config" button
- General intelligence config (staleness threshold, digest cron time): "Save Intelligence Config" button
- All saves use optimistic UI: button goes to "Saving…" state then "Saved" for 2 seconds

## Phase-Specific Constraints

These constraints flow from CONTEXT.md locked decisions and must be respected by the executor:

1. **No rounded corners** — `--radius: 0rem` is global. AnomalyBadge uses `rounded` class only for the small inline badge (exception approved in RESEARCH.md code example). All panels, cards, tabs, and inputs: zero radius.
2. **Monochromatic oklch palette** — no blue/purple/indigo accent colors (Memory.tsx currently uses `indigo-600` — this is pre-Phase-7 debt, do not propagate it to new components).
3. **MetricCard pattern enforced** — all numeric stat displays (forecast values, quality stats) must use `text-2xl font-semibold tabular-nums` value + `text-xs text-muted-foreground uppercase tracking-wide` label.
4. **Severity color system consistent with Phase 6** — warning=`--status-warn` (yellow oklch), critical=`--status-error` (red oklch). Do not use Tailwind `yellow-*` or `red-*` utility classes for severity; always use the CSS variable.
5. **Activity changelog is rule-based, not LLM** — no LLM cost for INT-05. Use event grouping template pattern (tool call counts by type, not narrative prose).
