<decisions>
## Implementation Decisions

### Cost Forecasting
- **D-01:** Simple moving average (7-day or 14-day) projected forward for daily/weekly/monthly spend predictions. Computed from existing daily aggregates in the `aggregates` table (Phase 5).
- **D-02:** Dual budget system — manual budget cap set by operator on Settings page AND auto-calculated baseline from historical data. Manual cap triggers alerts when projected spend exceeds budget. Auto-baseline flags unexpected deviations even within budget.
- **D-03:** Cost Forecast panel displayed on Analytics page showing current spend, projected daily/weekly/monthly, budget progress bar (green→yellow→red), and trend sparkline.
- **D-04:** All three time horizons shown: daily, weekly, monthly projections. Matches INT-01 requirement.

### Session Briefings & Daily Digest
- **D-05:** LLM provider configurable in Settings with primary + backup provider. If primary fails, automatically falls back to backup. Supports OpenAI and Anthropic. API keys stored in Convex config.
- **D-06:** Session briefings generated on session end via Convex action. Each completed session gets an LLM-generated narrative summarizing what happened, key decisions, and anomalies. Available immediately on Briefings page.
- **D-07:** Daily digest auto-generated by Convex cron, synthesizing: activity summary (what Ástríðr accomplished — SC-5), total spend vs budget, anomalies flagged, and ideation scan findings (SC-6). Stored in Convex for browsing.
- **D-08:** Briefings page restructured as reverse-chronological timeline feed of daily digests and session briefings. Each entry expandable with full narrative. Date range filter. Replaces current static template data.

### Anomaly Detection
- **D-09:** Z-score / standard deviation method. Compare current values against rolling mean ± N standard deviations from daily aggregates. Flag at 2σ (warning) and 3σ (critical).
- **D-10:** Three monitored metrics: cost spikes, error clusters, latency degradation. All already tracked in the Phase 5 aggregates table.
- **D-11:** Anomaly indicators displayed as inline badges on affected MetricCards/widgets. Tooltip shows anomaly detail. Uses Phase 6 severity color system (warning=yellow, critical=red).
- **D-12:** Anomalies auto-create alerts using Phase 6 alert infrastructure. Severity mapped from z-score (2σ=warning, 3σ=critical). Gets webhook delivery, mute, and lifecycle management for free.

### Memory Quality Metrics
- **D-13:** Three quality metrics: deduplication rate (% flagged as duplicates), staleness indicators (memories not accessed/updated beyond threshold), contradiction count (memories with conflicting content).
- **D-14:** Contradiction detection uses LLM-based comparison — periodic Convex action sends related memory pairs to the configured primary/backup LLM to check for contradictions. Same LLM config as briefings (D-05).
- **D-15:** Quality metrics displayed in top stats row (dedup rate, staleness count, contradiction count as StatCards) plus new "Quality" tab alongside existing timeline/tiers/reflections tabs showing detailed breakdowns and flagged items.

### Claude's Discretion
- Moving average window size (7-day vs 14-day) — pick based on data density
- Exact z-score rolling window size for anomaly detection
- Session briefing prompt design and context window strategy
- Daily digest cron timing (e.g., midnight, 6am, configurable)
- Memory contradiction comparison batch size and scheduling frequency
- Staleness threshold default (e.g., 30 days, 60 days)
- Activity changelog event grouping and narrative structure

</decisions>
