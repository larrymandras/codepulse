<objective>
Harden production CORS for all CodePulse ingest endpoints to a fail-closed origin allowlist (OPS-01, D-06/D-07). Replace the static `corsHeaders` constant in `convex/ingestAuth.ts` ŌĆö which currently emits `Access-Control-Allow-Origin: CODEPULSE_ALLOWED_ORIGIN ?? "*"` (fail-open) ŌĆö with a per-request `getCorsHeaders(request)` function that echoes the request Origin only when it is in a parsed allowlist, omits ACAO otherwise, and falls back to permissive `"*"` only when the env var is unset (dev).

Purpose: The current `?? "*"` is a fail-open CORS control. A comma-separated env value cannot be emitted as a single ACAO header (browsers reject multi-origin ACAO), so the static constant must become a per-request matcher. This closes a cross-origin data-exfiltration surface on the ingest endpoints.

Output: Refactored `convex/ingestAuth.ts`, all 8 handler files updated to call `getCorsHeaders(request)`, and expanded unit tests in `convex/__tests__/ingestAuth.test.ts`.
</objective>
