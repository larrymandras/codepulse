Build EVAL-02: a nightly Convex `internalAction` that samples completed sessions per active persona, judges each against a code-defined 4-dimension rubric via an isolated LLM config slot, and writes version-stamped `llm_judge` rows into `evalScores`.

Purpose: Give every persona's KPI trend a steady stream of quality signal from observable session behavior (errors, retries, tool churn, cost) without ever reaching the local Ástríðr API (D-06).

Output: judge machinery (dual-provider caller, zod validation, digest builder, idempotent store), the sampling orchestration action with `Promise.allSettled` isolation and a summary count, and the 05:00 UTC cron.
</objective>
