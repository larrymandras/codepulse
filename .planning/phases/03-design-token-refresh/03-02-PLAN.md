  <action>
**1. Add `accent` field to KpiDef interface:**
```typescript
accent?: "cost" | "health" | "activity" | "memory" | "alerts";
```

**2. Assign accent to each KPI in the `kpis` array:**
```typescript
{ label: "Sessions", ..., accent: "activity" },
{ label: "Error Rate", ..., accent: "alerts" },
{ label: "Alerts", ..., accent: "alerts" },
{ label: "Security", ..., accent: "alerts" },
{ label: "Memory Hit Rate", ..., accent: "memory" },
{ label: "Durable Facts", ..., accent: "memory" },
{ label: "Advisor Savings", ..., accent: "cost" },
{ label: "Startup Time", ..., accent: "health" },
```

**3. Apply data-accent and lift-on-hover to each tile div in the render map:**

Change the tile `<div>` from:
```tsx
<div
  key={kpi.label}
  onClick={kpi.onClick}
  className="group flex flex-col gap-1 cursor-pointer rounded-lg px-2 py-1.5 -mx-2 -my-1.5 transition-colors hover:bg-gray-800/70"
>
```
To:
```tsx
<div
  key={kpi.label}
  onClick={kpi.onClick}
  data-accent={kpi.accent}
  className="group flex flex-col gap-1 cursor-pointer rounded-lg px-2 py-1.5 -mx-2 -my-1.5 lift-on-hover"
>
```

Note: Remove `transition-colors hover:bg-gray-800/70` — the lift-on-hover utility handles the hover transition, and the radial gradient from data-accent provides the visual differentiation. The old hover:bg-gray-800/70 would fight with the gradient.

**4. Replace hardcoded hex fallback colors in the KPI definitions:**

Replace these hardcoded hex color values with accent token CSS custom properties:
- `color: "#60a5fa"` (Sessions) → `color: "var(--accent-activity)"`
- `color: "#f87171"` / `"#fb923c"` / `"#34d399"` (Alerts) — these are threshold-based, leave the threshold logic as-is since it uses thresholdColor. Only replace the static `color` prop on KPIs that don't use threshold. For Alerts/Security which compute color conditionally, replace hex values:
  - `"#f87171"` → `"var(--accent-alerts)"` (critical)
  - `"#fb923c"` → `"var(--status-warn)"` (errors)
  - `"#34d399"` → `"var(--accent-health)"` (all clear)
- `color: "#60a5fa"` (Sessions) → `"var(--accent-activity)"`
- Security color: `"#fb923c"` → `"var(--status-warn)"`, `"#34d399"` → `"var(--accent-health)"`

**5. Replace hardcoded hex in Sparkline fallback color:**
In the Sparkline usage `color={color ?? "#6366f1"}`, change to `color={color ?? "var(--accent-activity)"}`.
