**2. `data-accent` radial gradient selectors per D-05:**

The gradient pattern is `radial-gradient(120% 60% at 0% 50%, ${accent}10, transparent 55%)` where the accent color uses 10% opacity. In CSS, apply accent at 10% opacity using the `/` syntax in oklch:

```css
/* Phase 03: Category accent radial gradient backgrounds (D-05) */
[data-accent="cost"] {
  background-image: radial-gradient(120% 60% at 0% 50%, oklch(0.70 0.15 80 / 0.10), transparent 55%);
}
[data-accent="health"] {
  background-image: radial-gradient(120% 60% at 0% 50%, oklch(0.70 0.15 142 / 0.10), transparent 55%);
}
[data-accent="activity"] {
  background-image: radial-gradient(120% 60% at 0% 50%, oklch(0.70 0.12 230 / 0.10), transparent 55%);
}
[data-accent="memory"] {
  background-image: radial-gradient(120% 60% at 0% 50%, oklch(0.70 0.12 290 / 0.10), transparent 55%);
}
[data-accent="alerts"] {
  background-image: radial-gradient(120% 60% at 0% 50%, oklch(0.70 0.18 27 / 0.10), transparent 55%);
}
```

**3. Update the existing `prefers-reduced-motion` media query** (around line 280) — it already has a universal `animation-duration: 0ms` and `transition-duration: 0ms` rule. .lift-on-hover {

This ensures lift-on-hover is fully disabled per DT-06. The existing universal `transition-duration: 0ms !important` rule already covers transition timing, but explicitly nullifying the transform ensures no visual movement at all.
