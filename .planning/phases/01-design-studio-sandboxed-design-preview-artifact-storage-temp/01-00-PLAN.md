**open-design/Dockerfile** — Per D-07 (Docker sidecar) and RESEARCH.md Pitfall 5 (no published image) and Pitfall 7 (Node version mismatch):

```dockerfile
# Open Design daemon — Docker sidecar for CodePulse Design Studio
# Requires: git clone https://github.com/nexu-io/open-design.git ./open-design
# RESEARCH.md Pitfall 7: MUST use node:24 — better-sqlite3 native binding requires matching ABI
FROM node:24-alpine AS base

# Install curl for healthcheck and git for potential deps
RUN apk add --no-cache curl git

# Enable corepack for pnpm (RESEARCH.md: pnpm 10.33.x via corepack)
RUN corepack enable

WORKDIR /app

# Copy package files first for layer caching
COPY package.json pnpm-lock.yaml pnpm-workspace.yaml ./
COPY packages/ ./packages/
COPY apps/ ./apps/

# Install dependencies
RUN pnpm install --frozen-lockfile

# Copy remaining source
COPY . .

# Expose daemon default port (RESEARCH.md: 17456)
EXPOSE 17456

# Set data directory (RESEARCH.md: OD_DATA_DIR env var)
ENV OD_DATA_DIR=/app/.od

# Start the daemon (RESEARCH.md: verified start command)
CMD ["pnpm", "tools-dev", "start", "web"]
```

**Key decisions per RESEARCH.md:**
- `node:24-alpine` (not `node:22` or `node:lts`) — Pitfall 7 explicitly flags ABI mismatch with `better-sqlite3`
- `corepack enable` — Open Design uses pnpm via corepack
- `pnpm install --frozen-lockfile` — deterministic installs
- `OD_DATA_DIR=/app/.od` — persist SQLite data in the volume mount
- `pnpm tools-dev start web` — RESEARCH.md verified daemon start command
- `curl` installed for Docker healthcheck (`docker-compose.yml` uses `curl -f http://localhost:17456/api/health`)
