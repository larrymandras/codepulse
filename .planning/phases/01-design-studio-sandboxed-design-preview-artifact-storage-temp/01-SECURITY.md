## Audit Notes

### T-01-05 — IframeEmbed Sandbox (CR-05 Change)

Plan 02 declared the mitigation as `sandbox="allow-scripts allow-same-origin allow-forms"` to allow the full Open Design web app to make same-origin API calls to the daemon.

The code review fix pass (CR-05) removed `allow-same-origin`, resulting in the implemented value `sandbox="allow-scripts allow-forms allow-popups"` at IframeEmbed.tsx:113.

This is a **stronger security posture** than declared in the plan: removing `allow-same-origin` prevents the embedded Open Design web app from accessing CodePulse's origin context even if the daemon were compromised. The tradeoff is that the embedded app may not be able to make credentialed calls to its own origin, but since it communicates with the daemon (a different origin) this is acceptable. The plan's stated rationale for `allow-same-origin` ("full Open Design web app needs same-origin for its own API calls to the daemon") does not hold — daemon calls are cross-origin regardless of this flag.

Verdict: CR-05 change is a security improvement over the plan's declared mitigation. Status: closed.

### T-01-14 — SSE Abort on Unmount (CR-03)

Two cleanup paths verified:
1. `StreamingPreview.tsx` line 86: `return () => controller.abort()` — aborts main generation stream on unmount.
2. `NativeWorkflow.tsx` lines 90-95: `directionStreamCleanupRef` with `useEffect(() => { return () => directionStreamCleanupRef.current?.(); }, [])` — aborts direction-generation stream on unmount (CR-03 fix).

### T-01-02 — Convex Validator Scope (CR status check)

`convex/designProjects.ts` uses `v.string()` for `odProjectId`, `name`, `status` and optional strings for others. `status` accepts any string value — there is no allowlist enum enforcement in the validator. This is consistent with the plan's declared mitigation ("validate odProjectId is non-empty string via Convex v.string() validator") which does not mention status allowlist. Status closed per plan's declared scope.

### T-01-15 (Plan 05) — ZIP File Validation (WR-06 Fix)

`ZipImportDialog.tsx` line 39 implements both extension check (`file.name.endsWith(".zip")`) and MIME type check (`file.type !== "application/zip"`) as a client-side gate. The `accept=".zip"` attribute on the file input provides UI-layer filtering. Daemon receives the file for server-side validation. WR-06 fix confirmed present.

### T-01-08 — Brief Injection Cap (CR-02 Fix)

`NativeWorkflow.tsx` lines 151-152: `const MAX_BRIEF_LENGTH = 2000; const sanitizedBrief = brief.slice(0, MAX_BRIEF_LENGTH)` in `generateDirections()`. Also at lines 244/247 and 279/282 in `handleRegenerate()` and `handleNext()`. CR-02 brief cap confirmed across all three prompt-building paths.

### T-01-18 — TOCTOU Sync Race (CR-04 Fix)

`convex/designProjects.ts` line 95: `const actionStartedAt = Date.now()` recorded before listing existing IDs. Lines 129-135: `getSyncedAt` query checks if record was synced after `actionStartedAt - REMOVAL_GRACE_MS (60s)` before removing. CR-04 TOCTOU guard confirmed. Same pattern in `convex/designTemplates.ts` lines 86-87, 112-117.
