## Spacing Scale

Declared values (multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-label gaps, badge padding |
| sm | 8px | Compact element spacing, tag pills |
| md | 16px | Default card padding, form field spacing |
| lg | 24px | Section padding, card-to-card gaps |
| xl | 32px | Page section breaks |
| 2xl | 48px | Major layout divisions |
| 3xl | 64px | Full-height iframe area inset (top/bottom) |

Exceptions:
- Touch targets for icon-only buttons: 44px minimum (WCAG 2.5.5)
- Streaming log line height: 20px (tight, monospace readability)
- Wizard step connector: 2px height (decorative separator, not spacing)
- iframe embed content area: 100% width / calc(100vh - 56px) height (full bleed below topbar)

## Color

All values reference existing oklch design tokens from `src/index.css`. No new colors introduced.

| Role | Token / Value | Usage |
|------|---------------|-------|
| Dominant (60%) | `--background` = `oklch(0.145 0 0)` dark | Page background, iframe container |
| Secondary (30%) | `--card` = `oklch(0.205 0 0)` dark | Template cards, project list items, step panels, sidebar |

| Status: Error | `--status-error` = `oklch(0.72 0.20 27)` | Daemon unreachable, generation failed |

**Accent reserved for:**
- Active/selected tab underline in mode switcher (iframe / native)
- Active step indicator in native UI wizard
- "Generate Design" primary CTA button
- "Export" action button
- Daemon status indicator dot (when connected = status-ok, not accent)

**Accent NOT used for:** hover states, icons, borders, cards, skeleton loaders, streaming text.

## Interaction Contracts

### Mode Tab Switcher
- Default tab: "Embedded Studio" (fastest path to value)
- Tab switch is instant — no loading state
- Switching from Native UI mid-workflow: show Dialog "Abandon current design?" with "Keep Working" / "Switch Anyway"

### IframeEmbed
- On mount: show skeleton overlay, poll `GET /health` on `VITE_OPEN_DESIGN_URL` every 2s
- On success: fade skeleton out (200ms), render iframe
- On failure after 10s: replace skeleton with error overlay (status-error color, message, retry button)
- Retry button: re-triggers mount polling sequence
- iframe height: `calc(100vh - 56px)` where 56px is topbar height
- No border radius (consistent with `--radius: 0`)

### Skill Picker (Step 1)
- Grid: 3 columns on desktop, 2 on tablet, 1 on mobile
- Each card: skill name (text-sm semibold), one-line description (text-xs muted), category badge
- Hover: `bg-accent/10` card surface tint
- Selected: `border-sidebar-primary` (accent border, 1px), `bg-accent/20` surface
- Search input at top (text-sm, placeholder "Search 31 skills...")
- No multi-select — exactly one skill selected before advancing

### Design System Picker (Step 2)
- Same grid pattern as Skill Picker
- 129 items — virtual scroll or paginated (50 per page)
- Filters: style category (minimal, expressive, enterprise), framework (any, React, Vue)
- Selected: same accent border pattern as Skill Picker
- "Use Claude Design ZIP" link at bottom — triggers import Dialog (D-12)

### Discovery Form (Step 3)
- Single `<textarea>` (shadcn Textarea), 6 rows
- Label: "Describe your project" (text-sm semibold)
- Placeholder: "What are you building? Who is the audience? What tone or feeling?"
- Character count: `text-xs muted` bottom-right, no hard limit enforced in UI
- "Generate Directions" button (primary CTA, accent color) advances to Step 4
- Disabled state: button disabled if textarea is empty

### Direction Picker (Step 4)
- 3 direction cards displayed side-by-side (or stacked mobile)
- Each card: direction title (text-base semibold), 2-line rationale (text-xs muted), style keywords as Badge pills
- Directions fetched from daemon — show 3 Skeleton cards during fetch
- Selected: accent border
- "Generate Design" button (primary CTA) — disabled until one direction selected

### StreamingPreview (Step 5)
- Split layout: left 50% = streaming log panel, right 50% = srcdoc iframe preview
- Log panel: dark surface `bg-gray-900`, JetBrains Mono 13px, ScrollArea with auto-scroll to bottom
- Preview iframe: white background, no border (full bleed within its column)
- Progress bar above split: 0–100%, accent color fill, labeled "Generating design..."
- On stream complete: Progress bar reaches 100% and fades out; "Preview Ready" badge (status-ok) appears
- "Regenerate" button (secondary, muted): only visible after stream completes
- Quality score display: 5 dimension scores (Philosophy / Hierarchy / Execution / Specificity / Restraint) shown as 5 small badges with values after stream completes; if any score < 3, amber badge + "Re-pass triggered automatically" note

### ExportPanel (Step 6)
- Format buttons: HTML, PDF, PPTX, ZIP, Markdown — as toggle group (single select)
- "Download" primary CTA button
- Download triggers `GET /api/export/{projectId}?format={format}` → browser download
- Loading state on Download button while request in-flight (spinner icon)
- Success: brief "Downloaded" toast (sonner, 2s, bottom-right)
- Error: inline error below button (text-xs, status-error color)

### ProjectGallery
- Below wizard, separated by `<Separator />` and "My Projects" section heading
- EntityRow per project: icon (Palette Lucide), primary = project name, secondary = "skill / design-system · date", trailing = status badge
- Status badges: "Complete" (status-ok), "In Progress" (status-warn), "Failed" (status-error)
- Click row: opens Sheet (right side panel, 480px) with project details, preview thumbnail, re-export options
- "Delete" in Sheet: opens AlertDialog with destructive confirmation
- Empty state: shown when no projects exist (see Copywriting below)

### DaemonStatusBadge
- Positioned: top-right of page header
- States: connecting (spinner + "Connecting"), online (green dot + "Online"), offline (red dot + "Offline")
- Polls `GET {VITE_OPEN_DESIGN_URL}/health` every 10s
- Click: expands tooltip with URL and last-check timestamp (shadcn Tooltip)

## Accessibility

- All interactive elements: visible focus ring using `outline-ring/50` (enforced globally)
- iframe embed: `title="Design Studio"` attribute required
- srcdoc preview iframe: `title="Design Preview"` attribute required
- Wizard step indicator: `aria-current="step"` on active step, `aria-label="Step N of 6: {name}"` on each step button
- Skill/Design System Picker cards: `role="radio"` + `aria-checked` for single-select behavior
- Direction Picker cards: same `role="radio"` pattern
- DaemonStatusBadge: `aria-live="polite"` for status changes
- Streaming log: `aria-live="off"` (too noisy for screen readers; use landmark instead)
- All destructive actions: AlertDialog (not confirm()) — meets WCAG 3.3.4
- Color: status indicators never rely on color alone — icon + text label always present
