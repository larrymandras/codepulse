<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User file upload -> daemon | ZIP file from user sent to daemon via POST |
| Convex sync action -> daemon | Server-side action fetches from daemon (cloud deployment risk per A7) |
| Convex mutation -> UI | Project deletion propagates to all connected clients |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-15 | Tampering | ZipImportDialog.tsx | mitigate | File input restricted to `.zip` accept type; daemon handles ZIP validation via POST /api/import/claude-design; browser adds no additional parsing |
| T-01-16 | Tampering | ProjectGallery.tsx delete | mitigate | Destructive action guarded by AlertDialog confirmation (WCAG 3.3.4); deletion only removes Convex mirror row, not daemon source files |
| T-01-17 | Repudiation | DesignStudio.tsx sync | accept | Sync is read-only (upsert from daemon data); no audit trail needed for mirror sync operations |
| T-01-18 | Denial of Service | DesignStudio.tsx auto-sync | mitigate | Sync runs once on mount (not continuous polling); manual refresh button has no debounce but is user-initiated |
</threat_model>
