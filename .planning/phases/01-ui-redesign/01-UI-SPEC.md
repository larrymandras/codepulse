## Spacing Scale

Declared values (multiples of 4 only). Information-dense layout — Paperclip pattern uses `gap-1`/`gap-2` for tight operational displays.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, chart bar gaps |
| sm | 8px | Compact element spacing, EntityRow icon-to-text gap |
| md | 16px | Card padding, default element spacing, sidebar item padding |
| lg | 24px | Section padding, content area top/bottom padding |
| xl | 32px | Layout column gaps, major widget separation |
| 2xl | 48px | Page-level section breaks |
| 3xl | 64px | Reserved — not used in this phase (information-dense) |

Exceptions:
- Sidebar width: 240px fixed. Source: CONTEXT.md D-09
- Touch targets (sidebar nav items): minimum 44px height for accessibility

Note: FlexBarChart inter-bar gap uses the `xs` token (4px). No sub-4px exceptions.

## Typography

All sizes from Paperclip UI patterns analysis. Font: Geist. Weights: 400 (regular) and 600 (semibold) only. Maximum 4 distinct sizes in use.

| Role | Size | Weight | Line Height | Classes | Usage |
|------|------|--------|-------------|---------|-------|
| Display stat | 24px (1.5rem) | 600 | 1.2 | `text-2xl font-semibold tabular-nums` | MetricCard values, large KPIs |
| Page title / Section title | 20px (1.25rem) | 600 | 1.2 | `text-xl font-semibold` | Page `<h1>` and widget headings — context distinguishes role |
| Body | 14px (0.875rem) | 400 | 1.5 | `text-sm` | Default text, EntityRow primary |
| Label / muted | 12px (0.75rem) | 400 | 1.4 | `text-xs text-muted-foreground` | MetricCard labels, chart axis labels, tiny identifiers |
| Section header | 14px (0.875rem) | 600 | 1.3 | `text-sm font-semibold uppercase tracking-wide text-muted-foreground` | SectionHeader component (UI-03) |
| Code / log | 12px (0.75rem) | 400 | 1.6 | `font-mono text-xs` | Log lines, execution IDs, mono identifiers |

Notes:
- Active sizes: 12, 14, 20, 24px. Four sizes — within the 4-size maximum.
- Active weights: 400 and 600 only. `font-bold` (700) is not used anywhere in this phase.
- Page `<h1>` and widget section headings share 20px/600. Page titles are distinguished by position and DOM context (`<h1>` vs `<h2>`), not by a separate weight or size.
- `tabular-nums` class required on ALL numeric displays. Source: REQUIREMENTS.md UI-02, Paperclip patterns
- Section headers always uppercase with wide tracking — never sentence case. Source: REQUIREMENTS.md UI-03
- Cinzel font is completely removed — no headings use it. Source: CONTEXT.md D-02

### 60/30/10 Split

| Role | Token | Usage |
|------|-------|-------|
| Dominant (60%) | `--background` / `--card` | Page background, main content area, card surfaces |
| Secondary (30%) | `--muted` / `--accent` | Sidebar background, hover states, secondary panels, divider regions |
| Accent (10%) | `--primary` | Reserved for specific elements only (see below) |
| Destructive | `--destructive` | Destructive actions only |

Accent (`--primary`) is reserved exclusively for:
- Sidebar active nav item indicator (left border or background)
- Activity feed new-entry left stripe (`inset 2px 0 0 var(--primary)`)
- Primary button fill
- Focus ring on interactive elements

Operational status colors (`--status-ok`, `--status-error`, `--status-warn`) are reserved exclusively for:
- Status badge backgrounds
- Trend indicator icons in MetricCard
- Chart bar fill for OK/error/warn breakdown charts
- Animated ping indicators on active agent cards

Source: CONTEXT.md D-03, D-05; RESEARCH.md Pattern 1; Paperclip patterns

## Interaction Contracts

### FlexBarChart Interactions

- **Hover on bar segment**: Show tooltip with `{label}: {value}`. Tooltip uses `bg-popover border-border` styling. No JavaScript library — CSS `group-hover` only.
- **Click on bar segment**: Calls `onSegmentClick?(label, value)`. Caller decides navigation or filter behavior. Chart itself has no opinion on drill-down target.
- **Bar transition**: `transition-colors` on hover fill change (`--chart-bar` → `--chart-bar-accent`). Duration: Tailwind default (150ms).
- **Inter-bar gap**: 4px (`xs` token). No sub-4px exceptions.

Source: CONTEXT.md D-08, RESEARCH.md Pattern 4

### EntityRow Interactions

- **Hover**: `bg-accent/50` on rows with `onClick` prop. No hover on non-clickable rows.
- **Click**: Calls `onClick?()`. Border-b divider on all rows except last (`last:border-b-0`).
- **Cursor**: `cursor-pointer` only when `onClick` is provided. Default cursor otherwise.

Source: CONTEXT.md D-13, RESEARCH.md Pattern 5

### Activity Feed Animations

- **New entry animation**: `translateY(-14px) scale(0.985) blur(4px)` → `translateY(0) scale(1) blur(0)` over 520ms `ease-out`.
- **New entry accent**: `box-shadow: inset 2px 0 0 var(--primary)` left border stripe on newly added rows.
- **CSS keyframe name**: `slide-in-entry`. Applied via class `.activity-entry-new`.
- **No JS animation library**: Pure CSS `@keyframes`.

Source: CONTEXT.md D-14, REQUIREMENTS.md UI-07, RESEARCH.md Pattern 6

### Sidebar Collapse

- **Default (≥ md breakpoint)**: Full 240px width with labels, section headers, and count badges visible.
- **Collapsed (< md breakpoint)**: Icon-only mode — `w-12` width, only Lucide icons visible, no labels.
- **Mobile (< sm breakpoint)**: Hamburger toggle reveals full sidebar as overlay.
- **Collapse toggle**: Single button in sidebar footer. Uses `ChevronLeft`/`ChevronRight` Lucide icons.
- **Accessibility — collapsed icon buttons**: Each icon-only nav item MUST include `aria-label="{nav item name}"` and be wrapped in a `<Tooltip>` (shadcn) that displays the item label on hover. This is the only permitted label mechanism in collapsed state.

Source: CONTEXT.md D-11; accessibility requirement added by checker review

### Status Badge Rounding Exception

Status badges and pills use `rounded-sm` (2px) — the only non-zero radius beside `rounded-full` for avatars. This is the only place `rounded-*` utilities are permitted other than `rounded-full`.

Source: CONTEXT.md D-04

## Anti-Patterns (Executor Reference)

These patterns are explicitly prohibited. Source: RESEARCH.md Common Pitfalls + Anti-Patterns.

| Prohibited | Use Instead |
|-----------|-------------|
| `rounded-*` on cards, panels, inputs | No class — sharp corners by default |
| `indigo-*`, `purple-*`, arbitrary hex colors | `var(--primary)`, `var(--accent)`, `var(--muted-foreground)` |
| `recharts` imports | `FlexBarChart` component |
| `bg-[var(--token)]` (Tailwind v3 syntax) | `bg-(--token)` (Tailwind v4 syntax) |
| Changing MetricCard prop API | Keep `{ label, value, trend }` — 9 page imports depend on it |
| Placing shadcn primitives outside `src/components/ui/` | Always target `src/components/ui/` |
| Direct hex values in any utility class | Use CSS custom properties exclusively |
| `<hr>` or `<div className="border-b">` for section dividers | `<Separator />` from shadcn |
| CRT overlay CSS (`.crt-overlay`) | Remove entirely — do not migrate |
| Cinzel font declarations | Remove `@font-face` and all usages |
| `font-bold` / weight 700 anywhere | `font-semibold` (600) is the maximum weight |
| Inter-bar gap below 4px in FlexBarChart | Use `gap-1` (4px / `xs` token) |
| Icon-only sidebar nav items without `aria-label` | Always add `aria-label` + `<Tooltip>` wrapper in collapsed state |
