<objective>
Implement the D-05 cross-repo handoff from astridr phase 178.1: when a `run.blocks` event carries an approval block whose `requestId` matches an approval block already rendered in the Chat conversation, update that block IN PLACE (status flip) instead of appending a second card. CodePulse must render the resolution as an update to the existing card, and `ApprovalBlock` must render an externally-supplied resolved status with the same visual states as its local click-resolved state.

Purpose: Close the consumer side of the astridr↔CodePulse approval-block contract so operators see a single card that flips status, not a duplicate.
Output: Updated wire type, external-status-aware `ApprovalBlock`, update-by-requestId merge in Chat, plus regression tests.
