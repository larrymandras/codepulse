## Goal

Make Forge (the local coding-agent runner) a first-class CodePulse module so all coding-agent work happens in one application — **without moving Forge's execution engine off the local machine.** Forge runs as a local daemon emitting state UP via `/ingest`-style httpActions (the Surface-Substrate pattern Ástríðr already uses); CodePulse sends commands DOWN via a Convex command queue the daemon long-polls. Clerk-gated. *Rejected: a cloud tab calling `http://localhost` directly (mixed-content blocked).*

## Key Accomplishments

1. **Surface-Substrate bridge** — Forge's local engine stays local; state flows UP through bearer-authed Convex httpActions and commands flow DOWN through a Convex `forgeCommands` queue the daemon polls. No localhost/mixed-content path.
2. **Full Forge surface, Clerk-gated** — read-only job/workspace render (78/79) → launch/stop command bridge (80) → live log streaming (81) → files + artifact preview (82), end-to-end auth-correct.
3. **Convex-reactivity-as-stream** — live logs ship to `forgeLogChunks` and render live via a reactive query, retiring the original HIGH-risk direct daemon→browser SSE/WebSocket spike (risk HIGH→LOW).
4. **Files + artifact preview with hard security invariants (FI-14)** — sandboxed `data:` iframe with **no `allow-same-origin`**, zero `dangerouslySetInnerHTML`, filenames/text as React nodes; bounded daemon→Convex→cloud byte transport (per-file + per-job caps, retention sweeps).
5. **Live cross-repo round-trips each phase** — the Phase 82 round-trip caught and fixed **two real Forge daemon bugs**: `a31dca4` (enumerate dropped every file on a `realpath` EPERM → `lstat` fallback) and `dbfad91` (codex-sandbox files unreadable by the daemon → promotion `icacls /grant`).
6. **Hardening + ops** — retention crons (logs + files), OPS-01 deploy checklist, ingest key rotated off the `<new-strong-secret>` placeholder, and the Forge daemon + CodePulse UI automated via Windows scheduled tasks (hands-off day-to-day).
