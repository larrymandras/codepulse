## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, card inner gaps |
| sm | 8px | Card footer row spacing, button padding, compact list items |
| md | 16px | Card body padding, section gutter, form field spacing |
| lg | 24px | Column header padding, panel section breaks |
| xl | 32px | Board horizontal gap between columns |
| 2xl | 48px | Page-level vertical spacing |
| 3xl | 64px | Full-page layout padding (not used in Kanban fullscreen) |

Exceptions:
- Kanban column minimum collapsed width: 40px (thin strip, label rotated vertically) — not a spacing token, a layout dimension
- KanbanCard drag handle touch target: 44px height minimum (accessibility)
- Cron job row height: 44px minimum (inline controls — touch target)

## Color

All values from the established oklch palette in `src/index.css`. No new colors added.

| Role | CSS Token | Hex Approx | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `--background` | oklch(1 0 0) light / oklch(0.145 0 0) dark | Page background, Kanban board surface |
| Secondary (30%) | `--card` / `--muted` | oklch(1 0 0) light card / oklch(0.205 0 0) dark card | Kanban column backgrounds, KanbanCard surfaces, slide-out Sheet, ConfigEditor surface |
| Accent (10%) | `--primary` | oklch(0.205 0 0) light / oklch(0.922 0 0) dark | Reserved-for list below |
| Destructive | `--destructive` | oklch(0.577 0.245 27.325) | Cancel column left-stripe, dismiss action button border, irreversible-action confirmations |

Accent (`--primary`) reserved for:
- KanbanCard left-border priority stripe — HIGH priority only (see priority stripe rules below)
- Primary CTA buttons (Create Task, Apply Config, Trigger Cron)
- Active column highlight when a card is dragged over a valid drop target
- "Convert Selected" bulk action button background
- Drag overlay ghost border

Priority stripe colors (left-border-2 pattern, matches InboxCard's risk-level stripe):
- HIGH priority: `border-l-2 border-l-(--status-error)` — oklch(0.65 0.18 27) red
- MEDIUM priority: `border-l-2 border-l-(--status-warn)` — oklch(0.75 0.12 85) amber
- LOW priority: `border-l-2 border-l-(--status-ok)` — oklch(0.65 0.15 142) green

Finding severity badge colors (override Ideation.tsx legacy Tailwind grays — align to oklch tokens):
- CRITICAL: `bg-(--status-error) text-white`
- HIGH: `bg-(--status-error)/70 text-white`
- MEDIUM: `bg-(--status-warn) text-(--foreground)`
- LOW: `bg-(--status-ok) text-white`

Config diff view colors:
- Addition line: `bg-(--status-ok)/15` background, `text-(--status-ok)` for `+` prefix
- Removal line: `bg-(--status-error)/15` background, `text-(--status-error)` for `-` prefix
- Unchanged line: no background, `text-(--muted-foreground)`

Hot-reload status bar colors:
- pending: `text-(--muted-foreground)` with spinner
- validating: `text-(--status-warn)` with spinner
- applied: `text-(--status-ok)` static
- confirmed: `text-(--status-ok)` with checkmark
- error: `text-(--status-error)` with error icon + message

## Component Inventory

### KanbanCard (upgrade existing)

**Layout:** Vertical flex card, `bg-(--card) border border-(--border) p-3 rounded-none`

Left-border priority stripe (new — was dot badge):
```
border-l-2 border-l-(--status-error|warn|ok)  — 2px, flush left edge of card
```

Card body content (top to bottom):
1. Title — `text-sm font-medium line-clamp-2 leading-snug`
2. Label/tag chips row — `flex flex-wrap gap-1`, chips: `text-[10px] px-1.5 py-0.5 bg-(--muted) text-(--muted-foreground) rounded-none`
3. Footer row: agent avatar + due date + time-in-column + linked finding badge
   - Agent avatar: 16x16 circle, `bg-(--primary)`, first-char initial, `text-[8px] font-bold`
   - Due date: `text-xs text-(--muted-foreground)` — only shown if set
   - Time-in-column: `text-xs text-(--muted-foreground)` — `Xd Xh` format
   - Linked finding badge: `text-[10px] px-1 py-0.5 bg-(--status-warn)/20 text-(--status-warn) rounded-none` — shown only when finding_id present, clicking navigates to finding

Card drag state: `opacity-40` on source, solid at 100% on DragOverlay ghost

### KanbanColumn (upgrade existing)

**Normal state:** `flex flex-col w-[260px] flex-shrink-0 bg-(--muted)/40 border border-(--border)`

Column header: `px-3 py-2 border-b border-(--border) flex items-center justify-between`
- Label: `text-xs font-semibold uppercase tracking-wide text-(--muted-foreground)`
- Count badge: `text-xs text-(--muted-foreground)` parenthetical
- Plus button: `w-6 h-6` Lucide `Plus` icon button, `hover:bg-(--accent)`

**Collapsed state** (empty column):
- Width: 40px — `w-10 flex-shrink-0 border border-(--border) bg-(--muted)/20`
- Label: rotated 90deg, `text-[10px] uppercase tracking-wider text-(--muted-foreground) whitespace-nowrap`
- Expand on hover or when tasks arrive — animate via CSS `transition-[width] duration-200`
- Collapse threshold: auto-collapse when card count reaches 0 after initial render (not on first load)

**Drop target highlight:** `bg-(--accent)/30 border-dashed border-(--primary)`

### Ideation Findings Row (upgrade existing)

Replace current legacy-gray styles with oklch tokens. Row layout using `EntityRow` base pattern:

```
flex items-center gap-3 px-3 py-2.5 border-b border-(--border) last:border-b-0
```

Row columns (left to right):
1. Checkbox — 16x16, multi-select (D-06)
2. Severity badge — oklch status colors per severity table above
3. Scan type — `text-[10px] font-mono text-(--muted-foreground)`
4. Category — `text-xs text-(--muted-foreground)`
5. Title/description — `text-sm text-(--foreground) flex-1 truncate`
6. Status badge — `StatusBadge` reuse: open/acknowledged/converted/dismissed workflow
7. Actions trailing — "Create Task" `Plus` icon button + "Acknowledge" text button + "Dismiss" text button

**Ideation status → StatusBadge mapping:**
- open → `idle` semantic, label "OPEN"
- acknowledged → `warn` semantic, label "ACK'D"
- converted → `ok` semantic, label "CONVERTED"
- dismissed → `idle` semantic, label "DISMISSED", row opacity 60%

### Config Diff View (new)

Rendered inline below the CodeMirror editor after "Review Changes" click. No modal.

```
border border-(--border) bg-(--muted)/30 font-mono text-xs
```

Line format:
- Line number gutter: `w-10 text-right pr-2 text-(--muted-foreground) select-none border-r border-(--border)`
- Change type prefix: `w-4 text-center font-bold` — `+` green, `-` red, ` ` muted
- Line content: `flex-1 px-2`

**Library:** Use CodeMirror diff extension (`@codemirror/merge`) — already in the CodeMirror ecosystem, no new dependency.

### Hot-Reload Status Bar (new)

Slim bar below the Apply button. Height: 24px. `flex items-center gap-2 text-xs`.

States (left to right transition):
`pending` → `validating` → `applied` → `confirmed` (terminal), or `error` (terminal)

Animation: status text fades in (`transition-opacity duration-200`). Spinner (`Loader2 animate-spin w-3 h-3`) for pending/validating. Checkmark (`CheckCircle2`) for confirmed. `XCircle` for error.

### Cron Slide-Out Panel (new — shadcn Sheet)

Sheet variant: `side="right"`, `className="w-[400px]"`.

Content layout:
- Header: `SheetHeader` with title ("Create Cron Job" / "Edit Cron Job") and close button
- Frequency selector: `Select` component — options: Every minute / Every hour / Every day / Every week / Custom
- Conditional dropdowns based on selection: hour (`Select`, 0-23), minute (`Select`, 0-59), day-of-week (`Select`, Sun-Sat) — rendered as needed
- Expression output: `font-mono text-xs bg-(--muted) px-2 py-1 border border-(--border)` — live-updating cron string
- Human-readable preview: `text-sm text-(--muted-foreground)` italic — updates instantly on dropdown change ("Every day at 3:00 AM")
- Name field: `Input` with `Label`
- CTA: "Save Cron Job" `Button variant="default"` full-width at sheet bottom

### Cron Job Row (upgrade — inline controls)

Base: `EntityRow` pattern + inline controls appended to trailing slot.

Trailing slot contents: `flex items-center gap-2`
1. Play button: `Button variant="ghost" size="icon" w-7 h-7` — Lucide `Play` icon. Shows `Loader2 animate-spin` while waiting for WS ack
2. Toggle: `Switch` component — enable/disable. Sends `cron.toggle` WS command on change
3. Status badge: `StatusBadge` — ok (active) / idle (disabled)

## Interaction Contracts

### Kanban Drag-and-Drop

- Library: `@dnd-kit` (installed) — `DndContext` + `DragOverlay` + `closestCenter`
- Drag activation: pointer sensor with `activationConstraint: { distance: 8 }` (prevents accidental drags on click)
- Cross-column move to action columns (running, cancelled): fires confirmation toast BEFORE sending WS command (D-04)
- Confirmation toast timeout: 5s auto-cancel. If confirmed, sends `task.move` WS command
- Drag overlay: card rendered at 95% scale, `shadow-lg`, `opacity-90`
- Invalid drop zones: columns do not highlight if drag source is already in that column

### Kanban Column Collapse

- Auto-collapse trigger: column reaches 0 tasks (after mount — not on initial render)
- Auto-expand trigger: task arrives in column OR user hovers over collapsed strip
- Animation: `transition-[width] duration-200 ease-in-out`
- Collapsed width: 40px. Expanded width: 260px
- Hover expand: expands on `mouseenter`, collapses on `mouseleave` ONLY if still empty

### Ideation Multi-Select

- Checkboxes visible on row hover; always visible when any are checked
- "Convert Selected" button appears in page header toolbar only when count > 0: "Convert Selected (3)"
- Batch conversion: opens single TaskCreateForm pre-populated with first selected finding's data; additional findings create individual tasks silently
- Deselect all: ESC key or clicking "Convert Selected" header button after conversion completes

### Config Editor

- "Review Changes" button: enabled only when editor content differs from last-applied config
- Diff panel: appears below editor (not replacing it), scrollable, max-height 300px
- "Revert to Saved" button: always visible when there are unsaved changes; triggers inline confirm row (no modal)
- Apply button: disabled while hot-reload status is pending/validating
- CodeMirror undo (Ctrl+Z / Ctrl+Y): native, no custom handler

### Cron Builder

- Dropdown changes update cron expression and human-readable preview synchronously (no debounce)
- Custom expression mode: reveals a text `Input` for raw cron string; validates on blur against cron regex
- Invalid cron expression: input border `border-(--destructive)`, tooltip "Invalid cron expression"
- Save button: disabled if name field is empty or cron expression is invalid
