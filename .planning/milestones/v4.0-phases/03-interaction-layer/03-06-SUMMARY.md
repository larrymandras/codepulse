# Phase 03 Plan 06: Insights Chat Summary

LLM-powered Q&A page over CodePulse operational data with OpenAI function-calling, pre-scoped Convex tool execution, and GenerativeBlock rendering.

## What Was Built

### Task 1: Convex insightsChat Action (commit 5f3134a)

`convex/insightsChat.ts` implements a Convex action that:
- Defines 5 tools in OpenAI function-calling format: `cost_summary`, `error_counts`, `agent_status`, `session_list`, `alert_summary`
- Makes a real `fetch()` call to an OpenAI-compatible API (`OPENAI_API_KEY` required, `OPENAI_BASE_URL` optional for LiteLLM)
- Uses a two-round LLM pattern: LLM selects tools → Convex executes pre-scoped queries → LLM summarizes results
- Executes only named Convex queries via `ctx.runQuery()` — no raw DB access, no dynamic query building
- Assembles `GenerativeBlock`-shaped results (metric blocks for cost/errors/alerts, table blocks for agent/session lists)

`convex/__tests__/insightsChat.test.ts` has 11 passing tests covering:
- TOOLS array schema shape (5 tools, each with `type: "function"`, valid `function.name`, valid JSON Schema parameters)
- `assembleBlocks()` output for all 5 tool types plus unknown-tool fallback

### Task 2: InsightsChat Page + Routing (commit 51dc850)

`src/pages/InsightsChat.tsx`:
- Uses `useAction(api.insightsChat.ask)` to call the Convex action
- Renders responses via `ChatBubble` with `blocks` prop — same component as Agent Chat (D-12)
- Empty state: "Ask a question to get started."
- Loading state: `Loader2` spinner + "Querying your data..."
- Error state: "Query failed. Try rephrasing your question."
- Auto-scroll to bottom on new messages

`src/App.tsx`: lazy import + `/insights` route added

`src/layouts/DashboardLayout.tsx`: "Insights" nav item added to `overviewNavItems` with `insights` icon

`convex/_generated/api.d.ts`: manually updated to include `insightsChat` module for type safety (normally regenerated by `npx convex dev`)

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] process.env not available in Convex TypeScript context**
- **Found during:** Task 1 (type check)
- **Issue:** Convex tsconfig uses `lib: ["ES2021", "dom"]` without `@types/node`, so `process` is not typed. Three `process.env.*` references caused TS2580 errors.
- **Fix:** Added `const env = (globalThis as any).process?.env ?? {}` accessor and used typed casts. Avoids adding node types to convex tsconfig which could affect other Convex files.
- **Files modified:** `convex/insightsChat.ts`
- **Commit:** 51dc850

**2. [Rule 1 - Bug] api.insightsChat not in generated types**
- **Found during:** Task 2 (type check)
- **Issue:** `convex/_generated/api.d.ts` did not include `insightsChat` module — Convex codegen requires a live dev server (`npx convex dev`) to regenerate. InsightsChat.tsx's `api.insightsChat.ask` caused TS2339.
- **Fix:** Manually added the import and module entry to `api.d.ts`. The `api.js` runtime uses `anyApi` (dynamic proxy) so runtime is unaffected.
- **Files modified:** `convex/_generated/api.d.ts`
- **Commit:** 51dc850

**3. [Rule 1 - Adaptation] DashboardLayout nav structure differs from plan interfaces**
- **Found during:** Task 2 (reading DashboardLayout.tsx)
- **Issue:** Plan interfaces showed a typed `NavGroup[]` with an INSIGHTS group. Actual layout uses flat `commandNavItems` and `overviewNavItems` string-icon arrays with no grouping.
- **Fix:** Added Insights nav item to `overviewNavItems` and added `insights: "??"` to `iconMap`. Nav item renders correctly in the sidebar.
- **Files modified:** `src/layouts/DashboardLayout.tsx`
- **Commit:** 51dc850

**4. [Rule 1 - Adaptation] Convex query names differ from plan template**
- **Found during:** Task 1 (reading agents.ts, alerts.ts, sessions.ts)
- **Issue:** Plan template used `api.agents.list`, `api.alerts.list`, `api.sessions.list`. Actual exports are `listAll` (with optional limit arg for sessions) and `listActive`.
- **Fix:** Used correct export names: `api.agents.listAll`, `api.alerts.listAll`, `api.sessions.listAll` with `{ limit: 20 }` arg.
- **Files modified:** `convex/insightsChat.ts`
- **Commit:** 5f3134a

**5. [Rule 1 - Adaptation] ChatInput does not support placeholder prop**
- **Found during:** Task 2 (reading ChatInput.tsx)
- **Issue:** Plan specified `placeholder="Ask about cost, errors, sessions..."` but ChatInput has a hardcoded "Message Ástríðr..." placeholder with no prop support.
- **Fix:** Omitted placeholder prop; added descriptive subtitle in InsightsChat header instead ("Ask about cost, errors, sessions, agents, or alerts.").
- **Files modified:** `src/pages/InsightsChat.tsx`
- **Commit:** 51dc850
