### Live Update Pulse Animation

Widget pulse on incoming WebSocket event: brief background highlight using `--primary` at 8% opacity, fading over 600ms. Implementation: CSS keyframe extending `src/index.css`. No "LIVE" badges. No persistent color change.

```css
/* Add to src/index.css — live-update-pulse animation */
@keyframes live-update-pulse {
  0%   { background-color: oklch(from var(--primary) l c h / 0.08); }
  100% { background-color: transparent; }
}
.live-update-flash {
  animation: live-update-pulse 600ms ease-out forwards;
}
```

Apply `.live-update-flash` class transiently (add, then remove after 620ms) on the widget's container element whenever a WebSocket update is received.

## ConnectionPopover Spec

Trigger: clicking the sidebar footer WSStatusIndicator opens this popover. Popover closes on outside click.

Layout: `w-[280px]` fixed-width. No scroll. All content fits.

| Row | Label | Value |
|-----|-------|-------|
| 1 | URL | WebSocket endpoint URL (e.g. `ws://localhost:8765`) |
| 2 | Status | Connected dot + "Connected" / "Disconnected" / "Reconnecting..." |
| 3 | Uptime | "Connected 4m 32s ago" or "—" when disconnected |
| 4 | Latency | Last round-trip time in ms, or "—" |
| 5 | Topics | Comma-separated list of active subscriptions |
| 6 | Last event | Relative timestamp of most recent received event |

Footer: "Reconnect" button — visible only when disconnected. Uses `variant="outline" size="sm"` Button. Full width of popover.

Popover header: `text-xs font-semibold uppercase tracking-wide text-muted-foreground` — "CONNECTION DETAILS"

Row styling: flex row with label left (`text-xs text-muted-foreground`, 96px fixed width) and value right (`text-xs font-mono` for URL/latency, `text-xs` for others). Separator between header and rows.

## useLiveState Hook Spec

Single hook with topic-based selectors. Manages ALL transient real-time operational state.

```typescript
type LiveStateSlice = {
  agentStatus: "idle" | "running" | "paused" | null;
  activeRunId: string | null;
  activeRunProgress: number | null;  // 0-100
  liveMetricDeltas: Record<string, number>;  // metric key → delta value
  connectionHealth: WSStatus;
};
```

Behavior:
- On WebSocket disconnect: all fields reset to null/empty immediately (D-05 — no stale data)
- Internal state management: `useReducer` with typed action union
- Topic selectors: hook accepts `topics: string[]` — subscribes only to requested topics
- Returns: `{ state: LiveStateSlice, isLive: boolean }`
- `isLive` is `true` only when `connectionHealth === "connected"`

## Merge Strategy Per Widget Type

Per D-02, per-widget merge strategy based on data freshness requirements:

| Widget Category | Strategy | Rationale |
|----------------|----------|-----------|
| Agent status (idle/running/paused) | WebSocket-only via `useLiveState` | Must reflect current state, not Convex-cached state |
| Active run progress | WebSocket-only via `useLiveState` | Transient — not persisted to Convex |
| Live metric counters (events/sec, active sessions count) | WebSocket overlay on Convex base | Convex provides historical, WS provides live delta |
| Historical metrics (cost totals, session counts) | Convex-primary, no WS | Not time-sensitive, accuracy over freshness |
| Security events list | Convex-primary + WS prepend | WS prepends new events, Convex holds history |
| Connection health data | WebSocket-only | By definition not available when Convex can substitute |

## Interaction Contract

### Status Indicator Behavior

- Sidebar footer (expanded): 8px dot + label text. Clickable → opens `ConnectionPopover` anchored to the indicator.
- Sidebar footer (collapsed): 8px dot only. No click interaction (popover requires label context for usability; in collapsed mode, tooltip shows status on hover).
- Header: 8px dot only. Not clickable in Phase 2 (Phase 3 may upgrade).
- Reconnecting state: dot uses `animate-pulse` (Tailwind built-in) — no additional CSS needed.

### Live Update Flash

- Trigger: WebSocket event received and state updated in a widget
- Action: add `.live-update-flash` to widget container, remove after 620ms
- Scope: entire MetricCard container (not just the value span)
- No repeat-flash within 1 second of previous flash (debounce)

### Popover Keyboard

- Opens on Enter/Space when trigger has focus
- Closes on Escape
- Tab navigation through rows and reconnect button
- Focus returns to trigger on close
