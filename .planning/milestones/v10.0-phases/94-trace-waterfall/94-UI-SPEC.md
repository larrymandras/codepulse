# Phase 94 — UI Design Contract

> Visual and interaction contract for the Trace Waterfall (TRACE-01/02). Generated by gsd-ui-researcher, verified by gsd-ui-checker. This is an ADDITIVE surface inside the existing `SessionDetail` page — it inherits the shipped design system wholesale and only specifies what's NEW (the `TraceWaterfall` component, its Trace tab, and the `LangfuseTraceLink` removal). No new tokens, no new theme, no new component library primitives.

| Role | Size | Weight | Line Height | Usage |

## Color

All values are **token roles**, resolved per active theme (`cyan` / `emerald` / `readable` / `aubergine`) — never fixed hex in the component.

| Role | Value | Usage |
|------|-------|--------|
| Dominant (60%) | `var(--background)` / `var(--card)` | Page background, trace-group card surfaces |
| Secondary (30%) | `var(--muted)` / `var(--border)` | Trace-group header background, row dividers, the flat "Untraced calls" group surface (visually recessed vs. traced groups) |
| Accent (10%) | `var(--primary)` | RESERVED FOR: the expand/collapse chevron active state on a trace-group header, the "View Trace" cross-nav link/button (Analytics → session deep-link, D-08), and the current-time/live-update indicator on the time axis. Not used for call bars themselves (see below). |
| Destructive | `var(--destructive)` | Reserved for a failed/error LLM-call bar's border or icon accent only — not the whole bar fill (bar fill uses the status scale below) |

**Call-bar and badge color scale** (extends the existing `--status-*`/`--chart-*` tokens already declared in `src/index.css`, no new tokens needed):

| Element | Token | Condition |
|---------|-------|-----------|
| Normal call bar fill | `var(--chart-1)` | Default LLM call, no error |
| Failed/error call bar fill | `var(--status-error)` | Call's associated event is an error/tool-error |
| Cache badge — HIT | `var(--status-ok)` | `cacheReadInputTokens > 0` (D-13) — badge text e.g. "92% cached" |
| Cache badge — MISS | `var(--status-warn)` | Cache fields present, reads = 0 (D-13) |
| Cache badge — NO DATA | *(no badge rendered)* | Legacy rows lacking cache fields — per D-13, never render a fake MISS |
| Cost dash / "n/a" | `var(--muted-foreground)` | Missing `cost` field (D-14) — never estimated |
| Untraced fallback group | `var(--muted-foreground)` on `var(--muted)` | Visually dimmed vs. traced groups, signaling "no trace data," not an error |

Accent reserved for: expand/collapse chevron active state, "View Trace" cross-nav affordance, live-update/current-time marker on the time axis. Never used for call-bar fill color (bars use the status/chart scale so cost/error signal isn't diluted by decorative accent use).

## Component & Interaction Notes (for planner/executor — supplements the tables above)

- **Visual focal point:** the call bars within the expanded trace groups are the primary visual anchor of the Trace tab — the summary strip and group headers are supporting context and must not compete with them for visual weight.
- **Chevron accessibility:** the expand/collapse chevron is icon-only; it MUST carry an `aria-label` (e.g., `aria-label="Expand trace group"` / `"Collapse trace group"`) or be paired with the group-header text as a single labeled trigger (Radix `Collapsible.Trigger` wrapping the whole header row satisfies this).
- **New component:** `src/components/TraceWaterfall.tsx` — Gantt-styled, mirrors `GanttTimeline.tsx`'s time-axis/lane structure (props, tick generation, `toPercent` mapping) but does NOT import or extend `GanttTimeline`'s `agents`/`events` prop shape (D-09 — separate component, parallel structure only).
- **Vertical structure (D-10):** collapsible trace groups (one per `traceId`, in chronological order) → each group header shows total duration + total cost + call count → expanding reveals one row per `llmMetrics` call, bar positioned on the shared session time axis (`start = timestamp − latencyMs`, `width = latencyMs`). The "Untraced calls" group (rows with no `traceId`) always renders last, flat, un-collapsible (no per-turn grouping possible for it).
- **Bar label (D-11):** inline text `{model} · {cost or "n/a"} · {cache badge or omitted}`; hover or click reveals a detail panel/tooltip with tokens in/out, `cacheReadInputTokens`/`cacheCreationInputTokens`, latency, provider, `toolName`, `billingType`. Exact hover-vs-click mechanism is Claude's discretion (CONTEXT.md) — default to Radix `Tooltip` on hover (already installed, zero new dependency) unless the executor finds hover insufficient for touch/click-through needs.
- **Live updates (D-12):** Trace tab's `useQuery(api.llm.*, {sessionId, ...})` on `llmMetrics` `by_session` — no polling. New rows append and the time axis right-edge extends; no manual refresh affordance should be shown (would contradict the reactive empty-state copy above).
- **Tab placement (D-06):** add `{ key: "trace", label: "Trace" }` to `SessionDetail.tsx`'s existing `TABS` array — no new route, no new nav entry. Deep-link via `?tab=trace` URL param (D-08), following the same pattern as the existing `activeTab` state (initialize from `useSearchParams` if not already wired for other tabs — check before adding a second mechanism).
- **Deletion scope (D-07):** remove `src/components/LangfuseTraceLink.tsx` entirely and its two usage sites in `src/pages/Analytics.tsx` (L30, L83) — no replacement affordance in the Analytics header.
- **Summary strip (D-15):** render 4 `<MetricCard>` instances above the waterfall — total LLM cost, call count, total tokens, overall cache-read ratio — literal reuse, no new props needed unless cache-ratio formatting requires a `format` callback (already supported by `MetricCard`).
