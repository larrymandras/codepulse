## Spacing Scale

Declared values (multiples of 4 only):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, status dot |
| sm | 8px | Compact element spacing, button icon gap |
| md | 16px | Default element spacing, form field gap, panel internal padding |
| lg | 24px | Section padding, inter-section gap |
| xl | 32px | Layout column gap |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (rare) |

Exceptions:
- Call graph SVG node sizes: Agent node 48x48px, Tool node 32x32px (both multiples of 8, acceptable)
- Touch targets for icon-only buttons: 44px min (accessibility requirement)
- Sheet panel width: 480px (matches existing AlertRuleForm — do not change)

## Typography

Matches established project patterns from AlertRuleForm, SectionHeader, NotificationChannels:

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body | 14px (text-sm) | 400 (font-normal) | 1.5 | Form field labels in collapsed state, help text, table cell content |
| Label | 14px (text-sm) | 600 (font-semibold) | 1.4 | Form field labels (active/expanded), collapsible section headers — distinguished from Body by weight only; size is the same 14px |
| Heading | 16px (text-base) | 600 (font-semibold) | 1.3 | Sheet title, panel section headings |
| Section label | 12px (text-xs) | 600 (font-semibold) | 1.2 | SectionHeader uppercase tracking-wide pattern, delivery log timestamps |

Two weights only: 400 (font-normal) and 600 (font-semibold). Weight 500 (font-medium) is not used in this phase.

All section labels use `uppercase tracking-wide text-muted-foreground` per established `SectionHeader` component pattern.

Mono font (JetBrains Mono) used for: routing key input, email address display when masked.

## Color

Dark mode values (primary runtime). Project uses oklch + named CSS variables exclusively.

| Role | CSS Token | Dark Value | Usage |
|------|-----------|------------|-------|
| Dominant (60%) | `--background` | `#09090b` | Page background |
| Secondary (30%) | `--card` / `--secondary` | `#141416` / `#27272a` | GlassPanel cards, Sheet content, collapsible section backgrounds |
| Accent (10%) | `--primary` | `#10b981` (Matrix Emerald) | Reserved list below |
| Status OK | `--status-ok` | `#22c55e` | Healthy node fill, delivery success indicator, enabled toggle |
| Status Error | `--status-error` | `#ef4444` | Errored node fill, error propagation edges, delivery failure indicator |
| Status Warn | `--status-warn` | `#eab308` | Pending/degraded node fill |
| Destructive | `--destructive` | `oklch(0.704 0.191 22.216)` | Delete rule button, remove webhook inline confirm |
| Muted | `--muted-foreground` | `#a1a1aa` | Disabled text, collapsed section header text, empty states |

**Accent (`#10b981`) reserved for:**
- Toggle thumb (enabled state) — matches existing `bg-indigo-600` pattern but use `--primary` per current dark theme
- "Save Rule" primary button (existing pattern)
- Active sidebar nav item indicator
- Delivery log success status dot
- Call graph: no accent — use status-ok green for healthy nodes instead

**Node color contract (call graph SVG):**
- Healthy agent node: `--card` fill (`#141416`), `--border` stroke (`#27272a`), foreground label
- Healthy tool node: `--muted` fill (`#27272a`), `--border` stroke, muted-foreground label
- Errored node (any type): `--status-error` fill (`#ef4444`), white label
- Pending node: `--muted` fill, `--status-warn` stroke dashed
- Error propagation edge: `--status-error` stroke, 2px width
- Normal edge: `--border` stroke (`#27272a`), 1px width
- Arrow heads: same color as their edge stroke

**Toggle (enabled/disabled):**
- Enabled: `bg-indigo-600` track, white thumb — matches existing Settings Toggle component exactly
- Disabled: `bg-gray-700` track, white thumb

## Call Graph SVG Specification

### Node Sizing

| Type | Width | Height | Border Radius | Font |
|------|-------|--------|---------------|------|
| Agent node | 120px | 48px | 0px (radius-0) | 12px Geist, weight 600 (font-semibold) |
| Tool node | 96px | 32px | 0px (radius-0) | 10px Geist, weight 400 (font-normal) |

Agent nodes use weight 600 to match the Label role (active/important elements). Tool nodes use 10px at weight 400 to create clear visual separation from agent nodes — a 2px size difference combined with weight difference ensures both size and contrast distinguish the two tiers.

### Node States

| State | Fill | Stroke | Stroke Width | Label color |
|-------|------|--------|--------------|-------------|
| Healthy (agent) | `#141416` (--card) | `#27272a` (--border) | 1px | `#ffffff` (foreground) |
| Healthy (tool) | `#27272a` (--muted) | `#27272a` (--border) | 1px | `#a1a1aa` (muted-foreground) |
| Errored (any) | `#ef4444` (--status-error) | `#ef4444` | 2px | `#ffffff` |
| Pending (any) | `#27272a` (--muted) | `#eab308` (--status-warn) | 1.5px dashed | `#a1a1aa` |

### Edge States

| State | Stroke | Width | Arrow |
|-------|--------|-------|-------|
| Normal | `#27272a` (--border) | 1px | filled triangle, same color |
| Error path | `#ef4444` (--status-error) | 2px | filled triangle, same color |

### Dagre Config

```typescript
const g = new dagre.graphlib.Graph();
g.setGraph({ rankdir: "TB", nodesep: 40, ranksep: 48, marginx: 16, marginy: 16 });
```

### Legend

Inline row, bottom of panel, `gap-4 text-xs text-muted-foreground`:
```
● Healthy   ● Errored   ● Pending
```
Dots use 8px circles with status fill colors.

## Animation

Follows established `GlassPanel` pattern (motion/react):

| Element | Animation | Duration | Easing |
|---------|-----------|----------|--------|
| GlassPanel entry | `opacity 0→1, y 8→0` | 200ms | easeOut |
| Collapsible expand/collapse | `height auto` (Radix default) | 150ms | easeOut |
| Call graph initial render | Fade in SVG `opacity 0→1` | 250ms | easeOut |
| Node hover | No animation — CSS `opacity: 0.8` on hover | — | — |
| New edge on reactive update | No animation — re-render SVG in place | — | — |
| Delivery log new row | No slide-in (table, not feed) | — | — |

Respect `useReducedMotion` — skip all animations when set (matches GlassPanel pattern).

## Interaction States

### Routing Key Input

- Displayed as `type="password"` when not focused (mask characters)
- On focus: reveal plaintext
- On blur with new value: save immediately (same pattern as webhook URL)
- Masking display after save: show only last 6 characters prefixed with `••••` (not last 8 — routing keys are shorter tokens)

### Collapsible Sections (AlertRuleForm)

- Default: collapsed
- Chevron rotates 90deg when expanded (CSS `transition-transform duration-150`)
- Status summary text in trigger: computed from current saved state, not live form state
- Only one section may be open at a time — no constraint enforced (standard UX, both can be open)

### Call Graph Interactions

- Node hover: show Tooltip with node detail (agent id / tool name, call count, error count)
- Node click: no action (phase 70 scope — detail drill-down is future)
- Scroll: SVG contained in scrollable div if height exceeds 600px
- Empty state: shown when `callGraphEdges` returns empty array

### Delivery History Table

- Load: `useQuery` reactive — no manual refresh button
- Rows: newest first
- Pagination: show last 50 entries (no UI pagination control — operator dashboard, single user)
- Status badge: `variant="outline"` green border for success, red for failure
