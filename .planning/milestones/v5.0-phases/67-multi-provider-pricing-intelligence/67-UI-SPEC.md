## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline label padding |
| sm | 8px | Row item spacing, badge gaps |
| md | 16px | Default card/panel internal padding |
| lg | 24px | Section padding, panel headers |
| xl | 32px | Layout column gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level top padding |

Exceptions: SDK spend cap gauge touch-target minimum height uses `48px` (`min-h-[48px]`), the nearest standard token above the 44px accessibility floor. This satisfies WCAG 2.5.8 touch-target minimums.

Source: Default 8-point scale (no upstream override); gauge exception per D-04, revised from 44px to 48px to comply with standard token set.

- Metric values (API Spend, Subscription Usage counts) use `font-mono tabular-nums`.

| Dominant (60%) | `--background`: `#09090b` (dark) | Page background, panel fills |

Accent reserved for:

- Primary CTA button background
- Ring/focus indicator

Status semantic colors (already established tokens — use directly):

- Warning (80% cap): `--status-warn` `#eab308`
- Exceeded: `--status-error` `#ef4444`

Source: D-09 (provider family colors), `src/index.css` dark mode tokens.

## Interaction Contract

### TokenWaterfall Provider Grouping

- Provider header row: `text-xs font-semibold uppercase text-muted-foreground` with provider family color dot (8px circle, `inline-block`, `rounded-full`)
- Models nested under provider header: 8px left indent (`pl-2`)
- Sort order: provider groups sorted by total tokens desc; models within group sorted by total desc
- No collapse/expand — static grouped list

### SDK Spend Cap Gauge

- Gauge bar: `<Progress value={pct} />` from shadcn ui, color override via inline style or className swap on status
- Gauge container: `min-h-[48px]` to meet touch-target minimum (48px standard token)
- 80% marker: absolutely positioned vertical line `w-px h-full bg-[--status-warn] opacity-70` at `left: 80%` within relative container
- Auto-alert badge: appears inline right of gauge when `currentDaySpend >= 4.00`; uses `<Badge variant="outline">` with `text-[--status-warn]` text

### Analytics Split View Tabs (none needed)

- No tabs — render both "API Spend" and "Subscription Usage" MetricCards simultaneously in a 2-column grid
- Clear visual separation by MetricCard boundary (existing card border pattern)
