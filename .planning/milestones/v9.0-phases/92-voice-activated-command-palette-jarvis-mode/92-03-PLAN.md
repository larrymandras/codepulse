<objective>
Build the browser wake-word engine: an AudioWorklet that captures and resamples mic audio to 16kHz / 1280-sample (80ms) chunks and posts them to a separate Web Worker that runs the three ONNX sessions (melspectrogram → embedding → classifier), which posts a `wake` message back to the main thread. The `useWakeWord` hook orchestrates the Worker lifecycle, exposes `start`/`stop`, fires `onWake`, and degrades to `status:'error-disabled'` on any failure — never throwing, never opening a silent mic.

Purpose: VOX-01 (local in-browser wake detection in a Worker/AudioWorklet, no audio leaves the machine). VOX-04 graceful degradation (ONNX/runtime failure → disabled state, no crash, no silent mic). The architecture split is mandatory — onnxruntime-web cannot run inside an AudioWorklet (RESEARCH Pitfall 2: `self is not defined`).

Output: `src/worklets/micCapture.worklet.ts`, `src/workers/wakeWordWorker.ts` (+test), `src/hooks/useWakeWord.ts` (+test).
</objective>
