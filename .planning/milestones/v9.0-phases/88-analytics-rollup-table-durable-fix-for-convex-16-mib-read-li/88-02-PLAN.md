## Trust Boundaries

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-88-02 | Spoofing | idempotencyKey reuse to suppress real events | accept | A malicious/buggy producer that reuses an idempotencyKey across DISTINCT events would under-count (the second is treated as a dup and dropped). Accepted because: (1) this is a single-tenant internal telemetry pipe from the trusted Ástríðr emitter, not public ingress; (2) D-05 makes the key OPTIONAL — un-keyed events are always counted, so a producer cannot force under-counting without deliberately stamping colliding keys; (3) the alternative (no dedup) reintroduces double-count on legitimate retries. Documented residual risk; tightened automatically when the emitter stamps a stable per-event UUID (deferred cross-repo follow-up). |
| T-88-03 | Tampering | incrementBatch exposed as a public mutation | mitigate | incrementBatch is declared `internalMutation`, callable only via internal.analyticsRollup.incrementBatch from the backfill action — never a public unauthenticated increment endpoint. Asserted in this plan's acceptance criteria (grep gate). |
| T-88-04 | Repudiation/Integrity | Double-count during the deploy transition window | mitigate | The event-count/error-count branch removal ships in the SAME wave/deploy as the ingest-time increments (Convex per-deploy atomicity). No cron tick can run with both paths active. This is the phase's highest-risk correctness threat (Pitfall 1) and is mitigated structurally by wave co-location, not runtime logic. |
| T-88-05 | DoS | Unbounded cron llmMetrics read at high volume | mitigate | computeHourly cost reads are cursor-paginated (numItems 500) so a high-volume hour can never trip the 16 MiB/exec read limit and silently fail the cron (D-03). |
| T-88-SC | Tampering | npm/pip/cargo installs | accept | No package installs (RESEARCH Package Legitimacy Audit: zero new packages). No legitimacy checkpoint required. |
</threat_model>
