## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser UI → enqueueStop mutation | Operator stop action; auth enforced server-side (80-01). Stop is destructive (hard-kill, work discard) so it is double-gated by a confirm dialog before the mutation fires. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-15 | Elevation of Privilege | unauthenticated stop | mitigate | enqueueStop is Clerk fail-closed server-side (80-01 T-80-01). The Stop button only appears inside the authenticated /forge surface. |
| T-80-16 | Tampering | accidental destructive stop (work loss) | mitigate | Two-step AlertDialog confirm (D-03); copy explicitly states hard-kill + work-discard + irreversibility (D-01/D-02). No one-click stop. |
| T-80-17 | Repudiation | UI claims "Stopped" before the daemon actually kills | mitigate | No optimistic terminal flip (D-04, Pitfall 2): only the button shows local "Stopping…"; the status badge updates solely from the reactive query's real terminal status reflected back via /forge-ingest. |
| T-80-SC | Tampering | npm installs | mitigate | No new packages (AlertDialog already installed per UI-SPEC §Registry Safety). |
</threat_model>
