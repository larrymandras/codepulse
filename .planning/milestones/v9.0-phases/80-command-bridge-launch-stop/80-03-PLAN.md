## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser UI → enqueueLaunch mutation | Operator action. The mutation enforces auth server-side (80-01); the UI additionally hides the Launch affordance when Clerk identity is absent (defense-in-depth, not the security boundary). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-80-11 | Elevation of Privilege | Launch button visible without auth | mitigate | Launch button rendered only when `isAuthenticated` (Clerk user present); Clerk-unset → button absent. The real gate is the server mutation (80-01 T-80-01); this is UI defense-in-depth. |
| T-80-12 | Tampering | dangerous capability injected from the modal | mitigate | Modal never builds a `dangerous` capability (D-06); 80-01 `stripDangerousCapability` strips it server-side regardless. Two layers. |
| T-80-13 | Spoofing | duplicate/ghost rows confusing the operator | mitigate | Pending rows reconcile by `resolvedForgeJobId` vs the real forgeJobs list (Pitfall 3); reconciled rows are filtered out — no duplicates. |
| T-80-14 | Repudiation | silent command disappearance | mitigate | Failed/expired commands flip the pending row to a destructive-bordered Failed/Expired state with error copy (D-11); never silently removed. |
| T-80-SC | Tampering | npm installs | mitigate | No new packages (all shadcn primitives already installed per UI-SPEC §Registry Safety). |
</threat_model>
