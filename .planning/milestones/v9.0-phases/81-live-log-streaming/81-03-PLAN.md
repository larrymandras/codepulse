| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-81-10 | Information disclosure | Rendering log lines in the browser | accept | Lines arrive pre-scrubbed (Forge is authoritative on scrubbing, 81-SPEC §Locked design 5). CodePulse renders text only; client-side scrubbing is out of scope per CONTEXT. Trusted upstream scrub boundary. |
| T-81-11 | Tampering (XSS) | Log line render | mitigate | Lines rendered as JSX text children / `whitespace-pre` only — no `dangerouslySetInnerHTML`, no HTML interpolation. React escapes by default. |
| T-81-12 | Denial of service | Unbounded render under live churn | mitigate | `listJobLogs` is bounded (LOG_CHUNK_LIMIT) + retention caps the set (~1 MB/job, plan 02); `useMemo([raw])` prevents render-loop churn (Phase 80 lesson). |
| T-81-SC | Tampering | npm/pip/cargo installs | mitigate | No new packages installed (uses existing React/shadcn/JumpToLatestPill). No `[ASSUMED]`/`[SUS]` packages → no install checkpoint required. |
