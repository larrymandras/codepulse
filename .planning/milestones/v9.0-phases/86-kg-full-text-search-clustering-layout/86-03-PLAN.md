## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-86-06 | Tampering (XSS) | KGSearchResults snippet/predicate render | mitigate | Snippet + matched term rendered as React text / `<span>` (auto-escaped); NO `dangerouslySetInnerHTML` (grep-gated to 0 in Task 2). HIGH-severity to avoid per planning threat model. |
| T-86-07 | Information Disclosure | error-state UI copy | mitigate | Error copy is the fixed UI-SPEC string (names the endpoint + fallback) — never interpolates raw exception/stack into the banner. The AstridrApiError detail shown is the server `detail` field, not a stack trace. |
| T-86-08 | Spoofing / Elevation | result-click `from` / focus URL | mitigate | buildFocusUrl constructs in-app URLs only; the `?from` param is same-origin-guarded by Phase 85 `decodeFromParam` (shipped + tested). No external redirect path. |
| T-86-09 | Tampering | search query injected into URL | mitigate | kgGet uses `url.searchParams.set(k, String(v))` — values are URL-encoded by URLSearchParams, no raw string concatenation into the URL. |
| T-86-10 | Denial of Service | unbounded search calls | mitigate | ~250ms debounce + empty-query no-fetch + monotonic token drop of stale responses; Ástríðr enforces server-side limits when live. |
| T-86-SC | Tampering | npm/pip/cargo installs | mitigate | No package installs this plan (all primitives already in src/components/ui from Phase 71 — UI-SPEC § Registry Safety). |
