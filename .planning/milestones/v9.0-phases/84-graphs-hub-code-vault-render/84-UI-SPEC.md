## Color

Source: `src/index.css` `.dark` block, confirmed live.

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `#09090b` (`--background`, zinc-950) | Page background, canvas backdrop |
| Secondary (30%) | `#141416` (`--card`) / `#27272a` (`--secondary`, zinc-800) | GlassPanel surfaces, tile cards, detail panel, legend pill |
| Code accent (10%) | `#10b981` (`--primary`, Matrix Emerald) | Code-source nodes, canvas glow, active filter chip border, freshness badge dot, primary CTA text |
| Vault accent (10%) | `#8b5cf6` (violet-500) | Vault-source nodes only |
| Stale / warn | `#eab308` (`--status-warn`) | Stale badge, truncation badge |
| Error / destructive | `#ef4444` (`--status-error`) | Integrity warning text |
| Info | `#3b82f6` (`--status-info`) | Informational banners (dangling-link drop notice) |
| Muted foreground | `#a1a1aa` (`--muted-foreground`) | Secondary text in panel, legend labels |

### Vault accent justification (D-05, Claude's Discretion)

Violet `#8b5cf6` is selected over cyan `#06b6d4` for these reasons:

1. **Contrast against emerald.** On the hue wheel, violet (270°) is ~170° from emerald (160°) — maximum angular separation. Cyan (180°) is nearly opposite emerald on the green side, but emerald and cyan share the same blue-green quadrant and will cluster perceptually in low-light.
2. **Colorblind safety.** The emerald (#10b981) vs violet (#8b5cf6) pairing survives deuteranopia simulation: the two hues read as approximately yellow-green vs blue/purple, remaining distinguishable. Emerald vs cyan both fall in the blue-green range for deuteranopes — unsafe.
3. **Existing precedent.** `ToolGalaxy.tsx:27` already uses `#a78bfa` (violet-400) for MCP server nodes. Violet is an established accent in the palette. Using `#8b5cf6` (violet-500, one step richer) for vault nodes signals "graph relationship" without introducing a new hue family.
4. **Dark background legibility.** At #09090b, `#8b5cf6` achieves ~4.6:1 contrast ratio, meeting WCAG AA for large text; canvas nodes are always ≥ 8px radius with glow, so legible.

### Code/Vault glow shadow colors (canvas shadowColor)
- Code node: `#10b981` (emerald) — matches DEFAULT_COLOR.
- Vault node: `#8b5cf6` (violet-500).

### Edge colors (canvas, passed as `linkColorFn`)
- Code-to-code or cross-source: `rgba(16, 185, 129, 0.18)` — ForceGraphCanvas default.
- Vault-to-vault: `rgba(139, 92, 246, 0.18)`.
- Mixed (cross-source relation): `rgba(255, 255, 255, 0.08)` — neutral dim white.

### Code/Vault accent reserved-for list
- `#10b981` is reserved for: code-source graph nodes, active "Code" filter chip border/text, freshness pill dot, primary interactive text links, canvas edge default color.
- `#8b5cf6` is reserved for: vault-source graph nodes only. Not used on any other UI surface this phase.

### Fullscreen affordance (D-03)

Trigger: `Button variant="ghost" size="icon"` with `Maximize2` Lucide icon, positioned in the hero header right cluster. Tooltip: "Expand graph".

Behavior: toggle `fullscreen` boolean state. When true:
- The `CodeVaultGraph` container gains `fixed inset-0 z-50 bg-[#09090b]` — overlays the full viewport.
- Canvas switches to `h-screen` minus a 48px toolbar strip at top.
- The toolbar strip shows the same header row (truncation + filter chips) plus a `Minimize2` icon button to exit. Tooltip: "Exit fullscreen".
- No route change; no separate `/graphs/code-vault` URL this phase.
- ESC key exits fullscreen (standard browser escape hatch convention).

## Copywriting Contract

### Page header
- H1 label: `GRAPHS HUB`
- Subtitle / InfoTooltip body: `"Ástríðr's code, vault, and tool graphs — unified. The hero below shows the nightly graphify + Obsidian snapshot."`

### Summary tile copy (D-02)

| Tile | Label | Metric line |
|------|-------|-------------|
| Tool Galaxy | `TOOL GALAXY` | `{toolCount} tools · {orphanCount} orphans` |
| MCP Inventory | `MCP INVENTORY` | `{serverCount} servers · {errorCount} errors` |
| KG Explorer | `KG EXPLORER` | `{entityCount} entities · {tripleCount} triples` |

Each MetricCard: `onClick` navigates to the respective route (`/tool-galaxy`, `/mcp-inventory`, `/knowledge-graph`). Cursor pointer. No `trend` or `severity` prop this phase — neutral display.

### Truncation header (D-07)

Text: `"Showing {displayedNodeCount} of {totalNodeCount} nodes"`

Per-source chips (one per `sources[]` entry):
- Default: `"{source.label}: {source.displayedCount} / {source.nodeCount}"` — e.g. `"codepulse: 847 / 847"` or `"vault: 500 / 1,243"`
- When `source.truncated === true`: append a `Badge variant="outline"` with text `"truncated"` in `--status-warn` amber color (`text-amber-400 border-amber-400/30`).

The `source.label` is derived from the node's `source` prefix: strip `graphify:` → use the repo name portion; strip `vault:` → `"vault"`.

### Freshness badge (D-09)

Stale threshold: **36 hours** from `generatedAt`. This is one missed nightly cron (24h cycle) plus a 12h grace window. If the threshold needs tuning at runtime, change the constant `STALE_THRESHOLD_MS = 36 * 60 * 60 * 1000` in `CodeVaultGraph.tsx`.

- Fresh (< 36h): `"Updated {relativeTime}"` — e.g. `"Updated 6h ago"`. Plain `text-xs font-mono text-muted-foreground`.
- Stale (≥ 36h): prepend a `Badge` with text `"stale"` in amber: `bg-amber-500/10 text-amber-400 border-amber-400/30`. Then the relative time string.

### Integrity warning (D-08)

Shown only when `storedNodeCount < nodeCount` OR `storedLinkCount < linkCount`:

```
"N links dropped as dangling (source emitted X, stored Y)"
```

Rendered as a banner using the KG Explorer pattern:
```
border border-red-500/30 bg-red-500/5 px-4 py-3
```
`AlertTriangle h-4 w-4 text-red-500` + `text-xs font-mono`.

Silent when counts match — do not render the banner at all.

### Source filter chips (D-06)

Three segmented toggle buttons: `Code` | `Vault` | `Both`

Active chip: `bg-primary/10 text-primary border border-primary/40`.
Inactive chip: `bg-transparent text-muted-foreground border border-border`.
All chips: `text-xs font-mono px-3 py-1 rounded-[var(--radius-sm)] cursor-pointer`.

Default state: `Both` active (show all nodes, no client-side filtering).

### Hover tooltip (D-11)

`labelFn` returns: `"{node.label} · {node.type} · {node.source}"`

Example: `"src/pages/GraphsHub.tsx · file · graphify:codepulse"`

### Detail panel (D-10)

Opens in `grid grid-cols-1 lg:grid-cols-[1fr_320px]` layout, right column, same as KGDetailsPanel. Close: X button top-right (`aria-label="Close node details"`) + background click.

**Field order (locked):**
1. **id** — full namespaced id, `font-mono text-xs text-muted-foreground`, truncated with `truncate` class, full id in native title attribute.
2. **label** — `font-bold text-sm text-foreground`
3. **type** — `Badge variant="outline"` `text-xs`
4. **source** — source family chip: emerald `Code` or violet `Vault` pill matching node color
5. **community** — `text-xs text-muted-foreground` prefixed `"community:"` — shown even if `null` ("community: —"), to surface the field for later Phase 86 use
6. **Direct neighbors** — `SectionHeader title="Neighbors"` then a list of neighbor node labels (clickable to select that node), each as an `EntityRow`. If > 8 neighbors, wrap in `ScrollArea h-[200px]`.

Panel state when no node selected: render the panel column with a placeholder `"Select a node to inspect"` in `text-xs font-mono text-muted-foreground` centered vertically.

The close (`X h-4 w-4`) button is icon-only and MUST carry `aria-label="Close node details"`.

### Empty / null state (D-12)

When `useProjectGraph` returns `null` (no snapshot ingested):

Hero canvas area shows the standard empty shell:
```
h-[600px] flex flex-col items-center justify-center gap-3
border border-primary/20 rounded-[var(--radius)] bg-[#09090b]
```

Content:
- `Network className="h-8 w-8 text-primary/40"` icon
- Heading: `"No graph snapshot received yet"` — `text-sm font-mono text-muted-foreground`
- Body: `"Ástríðr's nightly graph_snapshot cron (graphify + Obsidian vault) has not pushed a snapshot to this deployment yet. Summary tiles above are independent and update on their own."` — `text-xs text-muted-foreground/70 max-w-md text-center`

The summary tile row renders normally above — it uses independent hooks and is unaffected by the null snapshot state.

### Loading state (Claude's Discretion)

Hero canvas while `useProjectGraph` is loading (Convex `undefined` before first response):
```
h-[600px] flex items-center justify-center
rounded-[var(--radius)] border border-primary/20 bg-card/50
```
Content: `<p className="text-primary/70 font-mono text-sm animate-pulse">Loading graph snapshot…</p>`

This matches the KG Explorer loading pattern (`KnowledgeGraph.tsx:218`).

Summary tiles: MetricCard renders with a numeric `0` while its hook is loading (Convex undefined → hook returns `0`). No skeleton needed; the MetricCard already handles this gracefully.

### Error states

If `getProjectGraph` throws (Convex error — unlikely for a query, but handled by SectionErrorBoundary):
- The SectionErrorBoundary fallback renders its standard error UI.
- No custom error copy needed — the boundary handles it.
