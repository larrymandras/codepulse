<objective>
Add the three persistence tables for the graph-snapshot receiver (GH-01): a `graphSnapshots` meta doc (one row per `snapshotId`, holds the `activeVersion` pointer + producer/receiver counts) plus `graphSnapshotNodes` and `graphSnapshotLinks` entity tables keyed by `(snapshotId, version)`. Row-based storage (D-01) is required because a single-blob doc would exceed Convex's 8,192-element array-field limit (~9,000 links worst case) and the ~1 MiB document-size limit.

Purpose: This is the schema foundation. The receiver module (plan 02) references these table names and the `by_snapshot_version` index, so the schema MUST land first and pass `npx convex dev`'s schema push before that module compiles against `_generated`.
Output: Three new tables in `convex/schema.ts` with correct nullable/float types and compound indexes.
</objective>
