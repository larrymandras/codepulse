## Page Layout: Agent Console (evolved from LiveRun.tsx)

```
┌─────────────────────────────────────────────────────────────┐
│  HEADER  "Agent Console"   [WS status]   [■ E-STOP ALL]     │
├──────────────────┬──────────────────────────────────────────┤
│  RUN LIST        │  RUN DETAIL (selected run)               │
│  ┌─ RunCard ──┐  │  ┌── Tab bar: Timeline | Summary ──────┐ │
│  │ run-id     │  │  │                                      │ │
│  │ engine     │  │  │  RunTimeline (streaming / history)   │ │
│  │ status pill│  │  │                                      │ │
│  │ elapsed    │  │  └──────────────────────────────────────┘ │
│  │ [Stop]     │  │                                           │
│  └────────────┘  │  [↓ Latest] scroll-to-bottom pill        │
│  (+ more cards)  │                                           │
│                  │                                           │
├──────────────────┴──────────────────────────────────────────┤
│  HISTORY  RunHistorySelector  (completed runs from Convex)   │
└─────────────────────────────────────────────────────────────┘
                        [+ New Run]  (floating or header CTA)
```

**Layout mechanics:**
- Two-column: run list (fixed `w-72`, `shrink-0`) left; run detail (`flex-1`) right. Stacks to single column below `md` breakpoint.
- Run list is independently scrollable (`overflow-y-auto`).
- Run detail uses `flex flex-col h-full` with the tab bar pinned at top, timeline scrollable in the middle, history selector pinned at bottom.
- "New Run" CTA: a `Button` with `+` icon in the page header (right side), opens `NewRunModal`.
- `[■ E-STOP ALL]` button: destructive variant, placed in the header right of the WS status dot. Disabled when no active runs exist.

## Stop Controls — Interaction Contract

### Per-run Stop (on each RunCard)

- Control: `Button variant="destructive" size="sm"` with `Square` Lucide icon, label "Stop".
- Trigger: single click, no confirmation dialog (D-10, graceful cancel — work persists).
- Action: `DELETE /tasks/{task_id}`.
- State machine:
  - `running` → click → button becomes disabled, label changes to "Stopping…" with `animate-pulse`.
  - Remains `Stopping…` until gateway emits cancel-ack event (research flag per D-09).
  - On cancel-ack (or `TaskStatus.CANCELLED` in stream): status pill → `stopped` (`--muted-foreground`), Stop button hidden.
- Enabled only when run status is `running`.

### Global E-Stop (page header)

- Control: `Button variant="destructive"` with `OctagonX` Lucide icon (or `ShieldOff`), label "E-Stop All".
- Trigger: single click opens an `AlertDialog` (shadcn) with:
  - Title: "Emergency Stop All Runs"
  - Body: "This will cancel all {N} active run(s). Agents will stop at their next safe checkpoint."
  - Buttons: "Cancel" (outline) | "Stop All Runs" (destructive fill).
- Action on confirm: iterate `DELETE /tasks/{id}` for every active task_id, optionally also trip `estop.py`.
- Disabled when zero active runs.

## Run State Machine — Visual States

| State | Status pill | Stop button | Run card border | Notes |
|-------|-------------|-------------|-----------------|-------|
| `queued` | `StatusBadge status="idle"` (zinc) | Hidden | `border-border` | Task submitted, not yet started |
| `running` | `StatusBadge status="warn"` (amber) + animate-pulse | Visible, enabled | `border-primary/40` emerald tint | Streaming active |
| `stopping` | amber pulsing + "Stopping…" text `text-xs font-mono` | Disabled (spinner) | `border-primary/20` | Awaiting cancel-ack |
| `completed` | `StatusBadge status="ok"` (green) | Hidden | `border-border` | Terminal |
| `error` | `StatusBadge status="error"` (red) | Hidden | `border-status-error/30` | Terminal |
| `stopped` | zinc muted | Hidden | `border-border` | Terminal (cancelled) |
