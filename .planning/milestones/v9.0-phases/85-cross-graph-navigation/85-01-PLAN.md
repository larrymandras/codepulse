<objective>
Build the shared, framework-pure deep-link plumbing that every graph surface in this phase consumes: the `buildFocusUrl` URL-builder, the normalized-exact match utilities (`normalizeFocusKey` / `focusKeysMatch`), the `from`-param encode/decode helpers (same-origin-constrained), and the generic `useFocusParam` hook that reads `?focus`/`?from` on mount and applies the focus once data resolves.

Building it generically here lets Phases 86 (search-to-focus) and 87 (saved/shareable views) reuse it.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL bar / shared link → app router | The `focus` and `from` query params are attacker-controllable (anyone can craft a CodePulse URL with arbitrary params and share it). They cross into client-side navigation and DOM. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-01 | Tampering / Elevation (open-redirect) | `decodeFromParam` in focus-url.ts (consumed by the return chip's navigate target) | mitigate | `decodeFromParam` rejects to null any value that is not a same-origin in-app path: must start with single `/`, must not start with `//`, must not contain `://`, must not match a `scheme:` prefix (blocks `javascript:`/`http(s):`/protocol-relative). Unit-tested in Task 1. The return chip (plans 02/03/04) navigates ONLY to the guarded value. |
| T-85-02 | Information disclosure / Injection (XSS) | `focus`/`from` param values rendered or matched in the UI | mitigate | Param values are only ever used as (a) react-router navigate targets — never `window.location =`/`href` with raw input, and (b) string-equality match keys via normalizeFocusKey. They are never passed to `dangerouslySetInnerHTML` and are rendered as React text children (auto-escaped). No raw HTML sink touches a param. |
| T-85-03 | Spoofing (wrong-target jump) | `focusKeysMatch` exact-match gate | mitigate | Exact equality on normalized keys only — no fuzzy/substring/Levenshtein (D-04). A non-match yields no affordance (SC#3), so a crafted near-miss id cannot produce a misleading "matched" link. Unit-tested. |
| T-85-SC | Tampering | npm/pip/cargo installs | accept | No new package installs in this plan — all utilities are hand-written over existing react-router-dom + Vitest. No supply-chain surface added. |
</threat_model>
