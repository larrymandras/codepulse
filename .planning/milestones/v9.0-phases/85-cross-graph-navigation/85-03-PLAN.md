<objective>
Wire the CodeVaultGraph surface end-to-end for cross-graph navigation: (a) when an agent node is selected, eagerly resolve which KG entities relate to that agent and, only when ≥1 resolves, render an "N KG entities →" link that jumps to the KG entity lens carrying the CodeVaultGraph's own state in `?from`; (b) read `?focus` on mount to select+center a target node; (c) render the "← Back to {Surface}" return chip in the panel header when `?from` is present.

Purpose: This is forward link #2 of GH-04 (SC#2 — agent → related KG entities). It fills the exact slot Phase 84 D-10 left open in the CodeVaultGraph detail panel. It is also a focus DESTINATION — the Tool Galaxy tool→agent jump (Plan 02) lands here on `/graphs?focus=<id>`.

DESIGN NOTE — the one real decision: CodeVaultGraph currently loads NO KG data, so the eager agent→KG match (D-04 requires confirming the destination BEFORE rendering the link) must consult the KG. The faithful, no-new-backend path is to reuse the EXISTING `useKnowledgeGraph` hook in overview lens scoped by the agent (its `agentId` filter is the agent→KG join). This is not new fetch code and not a new Convex query — it is the same hook KnowledgeGraph.tsx already uses; CodePulse-frontend-only is preserved. When KG data is undefined/empty/errored, the link simply does not render (SC#3-safe degrade). This is the only way to satisfy SC#2 ("agent → related KG entities WHERE a relationship exists") — you cannot know the relationship exists without consulting the KG.

Output: `src/components/graph/CodeVaultGraph.tsx` modified only.
</objective>
