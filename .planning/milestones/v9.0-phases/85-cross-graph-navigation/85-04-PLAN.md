| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|

| T-85-08 | Tampering (filter injection) | `?focus`/`?hops` → setFilter("entityName")/setFilter("hops") | mitigate | `entityName` is passed to the existing name-driven KG fetch (a query param to Ástríðr `/api/kg/entity` via authHeaders) — same path the KG search box already uses; no new sink. `hops` is coerced via `Number(...)` and defaults to 1 on NaN/absent (bounded by the existing hook's hops handling). No string is interpolated into a DOM/HTML sink. |
| T-85-05 | Injection (XSS via entity-name render) | KG entity name / focus param rendered in panel/chip | mitigate | Entity names and the focus param are React text children (auto-escaped); none reach `dangerouslySetInnerHTML`. |
| T-85-SC | Tampering | npm/pip/cargo installs | accept | No new package installs — existing react-router-dom, lucide-react, shadcn primitives, Plan 01 helpers, and existing hooks only. |
