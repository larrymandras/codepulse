<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| URL bar / shared link → ToolGalaxy router | `?focus` and `?from` are attacker-controllable and drive node selection + the return-chip navigate target. |
| Tool Galaxy → /graphs outbound navigate | The built focus URL embeds the current Galaxy state as `from` — must not be a vector for embedding a hostile return target. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-85-01 | Tampering (open-redirect) | Return chip navigate target (`fromParam`) | mitigate | `fromParam` is the output of `useFocusParam`, which routes the raw `?from` through `decodeFromParam` (Plan 01, same-origin-path-only guard). The chip navigates to that guarded value only — never to raw `searchParams.get('from')`. |
| T-85-04 | Spoofing (misleading cross-graph link) | Eager owning-agent match | mitigate | Link renders only when `focusKeysMatch` confirms an exact normalized match against a real code/vault node (D-04). A near-miss agent name yields no link (SC#3), so no dead/misleading jump. |
| T-85-05 | Injection (XSS via param/label render) | Detail panel + link row text | mitigate | Node names, labels, and the `focus` param are rendered as React text children (auto-escaped); none reach `dangerouslySetInnerHTML`. The outbound `from` value is `encodeURIComponent`-wrapped before being placed in the URL. |
| T-85-SC | Tampering | npm/pip/cargo installs | accept | No new package installs — only existing react-router-dom, lucide-react, shadcn primitives, and Plan 01 helpers. |
</threat_model>
