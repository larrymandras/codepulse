## Overview

A sleek, profile-organized Reminders command center where Larry tracks personal / business / consulting reminders — creatable and editable from both CodePulse and an Ástríðr conversation, always in sync — with recurrence, proactive due-nudges, and a read-only Google Calendar overlay per profile.

One Convex-backed `reminders` store is the single source of truth, written by both the CodePulse UI and an Ástríðr chat tool, each row origin-tagged (`dashboard` / `astridr`). Ástríðr crons drive proactive due-nudges (once-only via `notifiedAt` dedupe, on the reminder's own profile channel) and a per-profile read-only Google Calendar cache. Nothing is ever written back to Google.

> **Non-linear note:** v12.0 was executed as an interleaved side-quest while **v11.0 (Skills Command Center, Phases 97–100) remained in progress** (Phases 97/98 complete; 99/100 not started). Closing v12.0 did not change the active milestone — v11.0 resumes at Phase 99. REQUIREMENTS.md was therefore not deleted at close; only its v12.0 section was extracted here.

## Milestone Summary

**Key Decisions:**

- One Convex `reminders` store is the single source of truth (D-01); `complete()` spawns the next open occurrence for recurring reminders via `computeNextDueAt`; one-offs and past-`until` recurrences spawn nothing. `source` records origin only and never gates writes.
- Recurrence `byday` uses iCal 2-letter day codes (MO/TU/WE/TH/FR/SA/SU); Convex CRUD handlers extracted as plain `*Handler(ctx,...)` functions to unit-test without `convex-test`.
- Reminders are the Convex source of truth; the Ástríðr `reminders` tool writes the same store via `ConvexHandler.send_to`. Snooze is implemented as an `op:"update"` `dueAt` shift.
- Read-only Google Calendar overlay — **no Google write path exists**; per-profile cron caches a bounded forward window, upserted by `googleEventId` with stale rows pruned; one account's auth failure never blanks another profile's cache.
- Proactive nudges fire exactly once per due occurrence on the reminder's own profile channel (deduped via `notifiedAt`); a business reminder never nudges the personal channel.
- Ástríðr calendar/nudge crons register via the real scheduler (`cron_builders.py` + `cron_dispatcher.py`), not `jobs.py` (which is execution-tracking only).

**Issues Resolved:**
