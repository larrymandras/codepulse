<threat_model>
## Trust Boundaries
| Boundary | Description |
|----------|-------------|
| Ástríðr → /reminders-ingest, /reminders-read, /calendar-ingest | external HTTP writes/reads crossing into the Convex store |
| Google event text → /calendar-ingest → browser | event title/location (attacker-influenceable via a shared/invited event) is cached and later rendered in the page |

## STRIDE Threat Register
| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-101-01 | Spoofing/EoP | all three endpoints | mitigate | validateIngestAuth fail-closed (Bearer ASTRIDR_INGEST_API_KEY); a missing key does NOT open the endpoint. Unit-pinned (401 without auth) in Tasks 2/3. |
| T-101-02 | Info disclosure | /reminders-read | mitigate | Read is authed (POST + Bearer), never a public GET — reminders are personal data and never anonymously readable (D-07). |
| T-101-03 | Tampering (XSS) | calendar event text -> page | mitigate-here + downstream | Store event text as DATA only; never build HTML from it. The page (plan 06) renders via React text nodes (no dangerouslySetInnerHTML). No mitigation removed here. |
| T-101-04 | Tampering | /calendar-ingest prune | accept-scoped | Prune is scoped to (profileId, calendarAccount) from the authed push; it cannot delete another account's or the reminders table's rows. |
| T-101-SC | Supply chain | package installs | accept | No new deps; reuses ingestAuth + convex/server only. |
</threat_model>
