Architecture Approaches

How should CodePulse's HR section communicate with Ástríðr's agent factory?

A

Direct API — CodePulse → Ástríðr REST

CodePulse calls Ástríðr's existing HTTP endpoints directly. New endpoints added to Ástríðr for catalog browse, agent create, approval actions, and team management. Convex stores UI state (wizard drafts, view preferences). Ástríðr owns agent lifecycle.

Pros

  • Leverages existing Ástríðr agent factory pipeline
  • Single source of truth for agent config (agent-types.yaml)
  • Real-time agent status via existing WebSocket
  • Approval gate already built — just add HTTP trigger

Cons

  • Requires new Ástríðr API endpoints (catalog, teams, approvals)
  • CodePulse depends on Ástríðr being online for HR features
  • Two data stores to coordinate (Convex + Supabase)
B

Convex-First — Full CRUD in Convex, Ástríðr Syncs

All HR data (catalog, teams, agent configs, approvals) lives in Convex. CodePulse does full CRUD. Ástríðr polls or subscribes to Convex for changes and reconciles with its registry. Agent creation triggers a Convex mutation → Ástríðr webhook picks it up.

Pros

  • HR section works even if Ástríðr is offline
  • Convex real-time means instant UI updates
  • All CodePulse data in one store

Cons

  • Dual source of truth — agent-types.yaml vs Convex
  • Sync complexity: reconciling two registries
  • Duplicates existing Ástríðr factory logic in Convex
  • Risk of config drift between systems
C

Hybrid — Convex for UI State, Ástríðr for Lifecycle ★ Recommended

Convex owns UI-specific state: wizard drafts, team presets, view preferences, approval queue display. Ástríðr owns agent lifecycle: catalog, config validation, registry, activation. CodePulse sends create/approve commands to Ástríðr API, which emits telemetry events back to Convex for real-time display.

Pros

  • Each system owns what it's good at
  • No dual source of truth — Ástríðr is canonical for agents
  • Convex real-time for UI responsiveness
  • Existing telemetry pipeline carries lifecycle events to CodePulse
  • Team presets are purely a UI concept — Convex is natural home
  • Wizard drafts save locally, only submit to Ástríðr on "Deploy"

Cons

  • Still needs new Ástríðr API endpoints
  • Agent creation requires Ástríðr online
