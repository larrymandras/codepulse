Route Structure

New Routes
/hr/roster — Agent directory (org chart / cards / table views)
/hr/roster/:agentId — Agent detail panel (config, runtime, security)
/hr/catalog — Browse agent archetypes/templates
/hr/onboarding — Creation wizard (or redirects to catalog)
/hr/onboarding/:catalogId — Wizard pre-filled from template
/hr/teams — Team presets manager
/hr/teams/:teamId — Team detail/edit
Removed: /agents, /profiles (redirects to /hr/roster for backwards compat)

Page Responsibilities

📋 Roster

  • Switchable views: org chart, card grid, table
  • Agent status (active/idle/pending approval)
  • Runtime monitoring (from old Agents page)
  • Topology visualization (from old Agents page)
  • Security scan results (from old Agents page)
  • Click-through to agent detail
  • Approval queue badge/indicator

📚 Catalog

  • Browse archetypes (FTS5 search)
  • Preview template details
  • Filter by tier, capability, domain
  • "Onboard This Agent" → wizard pre-filled
  • Sourced from Ástríðr catalog.db

🚀 Onboarding

  • Step 1: Template selection (catalog embed)
  • Step 2: Identity (name, tier, description)
  • Step 3: Personality (soul variant editor)
  • Step 4: Tools & Capabilities
  • Step 5: Review & Deploy
  • Each step: "Advanced Options" toggle
  • Wizard drafts auto-save to Convex
  • Deploy → Ástríðr API → approval if permanent

👥 Teams

  • Create named team presets
  • Drag agents from roster into teams
  • Team description and purpose
  • "Launch War Room" → opens room with team
  • Ad-hoc team creation from war room too
  • Stored in Convex (UI concept only)
