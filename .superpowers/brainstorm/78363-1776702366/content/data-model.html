New Ástríðr API Endpoints

REST API additions to Ástríðr (FastAPI)
── Catalog ──
GET /api/catalog — List/search archetypes (query, tier, capability filters)
GET /api/catalog/:id — Archetype detail with default config
── Agent Lifecycle ──
POST /api/agents — Create agent (AgentTypeConfig JSON body)
GET /api/agents — List all registered agents (registry snapshot)
GET /api/agents/:id — Agent detail (config + runtime status)
PUT /api/agents/:id — Update agent config
DEL /api/agents/:id — Deregister agent
POST /api/agents/:id/validate — Validate config without creating
── Approvals ──
GET /api/approvals — List pending approvals
POST /api/approvals/:id/approve — Approve agent creation
POST /api/approvals/:id/reject — Reject agent creation
── War Room ──
POST /api/war-room — Create room with dynamic participant list
DEL /api/war-room/:roomName — Close room

Data Flow: Agent Onboarding

End-to-end flow from wizard to active agent
1. User fills wizard steps in CodePulse
↓ Auto-saves each step to Convex wizardDrafts
2. User clicks "Deploy" on review step
↓ CodePulse validates client-side, then POST /api/agents
3. Ástríðr receives AgentTypeConfig
↓ Server-side validation (config schema, tool availability)
↓ If ephemeral → activate immediately via EphemeralLifecycleManager
↓ If permanent → create approval request
4. Approval request
↓ Ástríðr emits agent_approval_requested telemetry → Convex approvalQueue
↓ CodePulse shows pending approval in Roster + notification bell
5. User approves in CodePulse UI
↓ POST /api/approvals/:id/approve
↓ Ástríðr activates via AgentOrchestrator
↓ Updates agent-types.yaml (permanent persistence)
↓ Registers in AgentTypeRegistry (runtime)
6. Agent active
↓ Ástríðr emits agent_created telemetry → Convex updates
↓ Agent appears in Roster as active, eligible for Teams/War Room

Data Flow: War Room with Dynamic Teams

Team preset → War room launch
1. User selects team preset (or picks agents ad-hoc) in Teams page
↓ Reads agentIds[] from Convex teamPresets
2. User clicks "Launch War Room"
↓ POST /api/war-room { participants: ["astridr", "hervor", "new-agent-x"] }
3. Ástríðr dispatcher
↓ Creates LiveKit room
↓ For each participant: dispatch LiveKit agent (dynamic, not hard-coded)
↓ New agents need NorseAgent subclass OR generic war room agent class
4. War room active with custom team composition
